# 项目文档索引

本文是项目文档唯一总入口。没有出现在这里的文档，不作为当前开发依据。

## 可信入口

| 入口 | 用途 |
| --- | --- |
| `docs/项目配置和代码风格/README.md` | 前后端代码规范、权限、i18n、UTC、代码生成器、部署约束、文档治理 |
| `docs/产品配置知识库/README.md` | 基础信息、配方管理操作手册、接口契约、体验问题、AI 代办能力说明 |
| `docs/商家体系/README.md` | 商家中心、商家管理操作手册、接口契约、体验问题、AI 代办边界 |

## 业务域状态

| 业务域 | 当前状态 | 文档策略 |
| --- | --- | --- |
| 基础信息 | 当前完成基线 | 以 `docs/产品配置知识库/基础信息/` 为准 |
| 配方管理 | 配方档案、原料、选项、模拟和审核快照基线 | 以 `docs/产品配置知识库/配方管理/` 为准 |
| 商家体系 | 商家中心、商家管理、客户管理当前实现基线 | 以 `docs/商家体系/` 为准 |
| 订单 / 系统基础能力 | 待确认是否纳入 | 暂不和产品配置知识库混放 |

## 工作流

1. 先读本索引。
2. 按任务领域读取对应规范或知识库。
3. 复杂变更先在对话中给计划。
4. 只有长期有效的规则、业务契约、操作手册和体验问题写入 `docs/`。
5. 修改后执行能覆盖本次变更的最小验证，并说明实际结果。

文档放置、删除和维护规则见 `docs/项目配置和代码风格/documentation-governance.md`。
