# 共享产品能力中心数据库真实库执行计划

## 目标

在真实开发库或后续测试库中受控执行 `sql/postgresql/product_capability.sql`，完成共享产品能力中心基础表、菜单、按钮权限和角色授权初始化。

本计划不面向生产直接执行。生产执行必须先转成正式 migration 或发布脚本，并经过备份、评审和窗口确认。

## 前置条件

- 已确认目标库是开发库或测试库，不是生产库。
- 已完成数据库备份或快照。
- 已确认应用版本包含 `bocoo.tenant.ignore-tables` 中的 23 张 `pc_*` 表。
- 已确认 `base.sql` 已在目标库执行过，至少存在 `sys_menu`、`sys_role_menu`、`sys_role`。
- 已确认目标库时区策略符合项目 UTC 规则。

## Preflight 查询

执行前先检查目标库冲突。

```sql
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
  AND table_name LIKE 'pc_%'
ORDER BY table_name;
```

```sql
SELECT menu_id, parent_id, menu_name, path, component, perms, visible, status
FROM sys_menu
WHERE menu_id BETWEEN 24000 AND 24099
   OR menu_id BETWEEN 24100 AND 24199
   OR menu_id BETWEEN 24200 AND 24499
ORDER BY menu_id;
```

```sql
SELECT menu_id, menu_name, perms
FROM sys_menu
WHERE perms LIKE 'product:%'
ORDER BY menu_id;
```

```sql
SELECT role_id, role_name, role_key
FROM sys_role
WHERE role_id = 1;
```

如果以上查询发现已有 `pc_*` 表、菜单 ID 或权限码，需要先做差异评审，不能直接覆盖。

## 执行步骤

建议在事务外执行初始化脚本，因为脚本包含 `CREATE TABLE IF NOT EXISTS`、`CREATE INDEX IF NOT EXISTS`、`COMMENT`、`INSERT ... ON CONFLICT` 等 DDL/DML 混合语句。

命令模板：

```bash
psql "$PG_DSN" -v ON_ERROR_STOP=1 -f sql/postgresql/product_capability.sql
```

执行过程中必须保留完整日志，但不要把数据库连接串、密码或 Token 写入提交记录和文档。

## Post-check 查询

执行后核对表数量。

```sql
SELECT count(*) AS pc_table_count
FROM information_schema.tables
WHERE table_schema = 'public'
  AND table_name LIKE 'pc_%';
```

预期：`23`，包含通用快照实例和导入中心基础表。

核对菜单数量。

```sql
SELECT count(*) AS product_menu_count
FROM sys_menu
WHERE menu_id BETWEEN 24200 AND 24499
  AND status = '1';
```

预期：`86`。其中 `24200/24300/24400` 是正式三个一级菜单；`24000` 旧 `Product Capability` 父菜单保留但隐藏，不计入正式菜单数量。

核对三个一级菜单。

```sql
SELECT menu_id, menu_name, path
FROM sys_menu
WHERE menu_id IN (24200, 24300, 24400)
ORDER BY order_num;
```

预期：`Product Master Data`、`Product Configuration Pricing`、`Product Release Application`。

核对 P0 二级入口。

```sql
SELECT parent_id, menu_name, path, component, perms
FROM sys_menu
WHERE menu_id IN (
  24201, 24202, 24203, 24204, 24205, 24206,
  24301, 24302, 24303, 24304, 24305, 24306,
  24401, 24402, 24403, 24404, 24405, 24406, 24407
)
ORDER BY parent_id, order_num;
```

预期：入口文档第 4 节列出的全部 P0 二级入口都存在，且 `component` 不为空。

核对管理员角色授权。

```sql
SELECT count(*) AS role1_product_menu_count
FROM sys_role_menu
WHERE role_id = 1
  AND menu_id BETWEEN 24200 AND 24499;
```

预期：`86`。

核对产品能力字典。

```sql
SELECT dict_type, count(*) AS item_count
FROM sys_dict_data
WHERE dict_type IN ('product_material_type', 'product_component_type', 'product_business_type', 'product_unit')
GROUP BY dict_type
ORDER BY dict_type;
```

预期：四类字典均有数据，物料类型、辅材类型包含面料、电机、遥控、下杆等常用值。

核对 `pc_*` 表不含租户字段。

```sql
SELECT table_name, column_name
FROM information_schema.columns
WHERE table_schema = 'public'
  AND table_name LIKE 'pc_%'
  AND column_name = 'tenant_id';
```

预期：`0` 行。

核对注释缺失。

```sql
SELECT c.relname AS table_name
FROM pg_class c
JOIN pg_namespace n ON n.oid = c.relnamespace
WHERE n.nspname = 'public'
  AND c.relkind = 'r'
  AND c.relname LIKE 'pc_%'
  AND obj_description(c.oid, 'pg_class') IS NULL
ORDER BY c.relname;
```

预期：`0` 行。

## 应用侧核验

执行 SQL 后，至少重启或刷新后端配置，并确认：

- `bocoo.tenant.ignore-tables` 包含全部 23 张 `pc_*` 表。
- 登录后能看到 `基础资料`、`配置与价格`、`发布与应用` 三个一级菜单。
- 侧边栏最多两级：三个一级菜单下直接挂入口文档第 4 节全部 P0 二级入口。
- 普通用户没有按钮权限时，不显示新增、编辑、删除、发布等操作。
- 产品能力接口查询不报 `column tenant_id does not exist`。

## 回退策略

开发库回退可以按以下顺序处理：

1. 删除角色授权：

```sql
DELETE FROM sys_role_menu
WHERE menu_id BETWEEN 24200 AND 24499;
```

2. 删除产品能力菜单：

```sql
DELETE FROM sys_menu
WHERE menu_id BETWEEN 24200 AND 24499;
```

3. 如需删除业务表，必须确认没有业务数据后再执行：

```sql
DROP TABLE IF EXISTS
  pc_product_sync_outbox,
  pc_import_row_issue,
  pc_import_batch,
  pc_product_snapshot_instance,
  pc_product_publish_package,
  pc_publish_approval,
  pc_publish_check_result,
  pc_price_rule_item,
  pc_price_plan_version,
  pc_price_plan,
  pc_config_rule,
  pc_config_option,
  pc_config_question,
  pc_question_group,
  pc_config_template_version,
  pc_config_template,
  pc_sales_variant,
  pc_product_model,
  pc_media_binding,
  pc_media_asset,
  pc_component,
  pc_material,
  pc_category;
```

生产库不建议使用手工 DROP 回退，必须依赖备份、正式 migration rollback 或发布窗口回滚方案。

## 风险说明

- `CREATE TABLE IF NOT EXISTS` 不会自动迁移已有表结构；如果目标库已有同名表，需要先做结构 diff。
- 菜单 ID `24200-24499` 是第一阶段正式菜单范围；`24000-24199` 为历史草案范围，保留用于兼容和对照。
- 当前发布包和 read model schema 仍是 MVP，后续可能追加字段和索引。
- `jsonb` 字段暂未建立 GIN 索引，等真实查询稳定后再加。
