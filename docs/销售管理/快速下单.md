# 快速下单

快速下单服务零售小单和小客户，逐件配置后直接生成正式订单，不经过客户报价。

## 页面

- 草稿列表：唯一可见菜单，使用标准 Grid，支持继续、复制、删除和查看已提交草稿。
- 配置工作台：选择客户、收货信息、产品、房间、分数英寸尺寸和整数成品数量；配置项动态生成。
- 订单复核：展示产品原价、商家折扣、产品应付、邮费和总额。

配置工作台和订单复核是菜单树中 `visible = 0` 的隐藏路由，从草稿列表或上一步跳入，不在侧边栏生成额外入口。

购物车汇总条与内容区等宽，不遮挡导航；展开后显示产品、尺寸、数量、动态配置摘要和金额。

## 提交规则

- 草稿保存时完成当前行计价，输入变化后必须重新计算。
- 提交时重新读取当前正式配方、价格、邮费和商家等级折扣。
- 预期总额变化时不创建订单，要求重新复核。
- 提交使用业务锁和条件更新，成功后草稿变为 `ORDERED`。
- 正式订单 `source_type=QUICK_ORDER`，报价来源字段为空。

## 接口族

- `GET /dealer/quick-orders/list`、`GET /dealer/quick-orders/{id}`
- `POST/PUT/DELETE /dealer/quick-orders/**`
- `POST /{id}/copy`
- `PUT /{id}/calculate-item`、`PUT /{id}/calculate`
- `POST /{id}/submit`
- `/catalog/products`、`/catalog/products/{id}/setup`

商家只获得运行时目录和计价能力，不获得平台产品、配方和价格维护权限。

商家与内部销售使用 `/dealer/quick-orders/**`，按当前租户、`business_origin` 和部门数据权限过滤。平台“下单查询”使用 `/platform-sales/quick-orders/**` 和 `platform:sales:quick-order:*`，默认只读，不借用查询权限编辑或提交租户草稿。
