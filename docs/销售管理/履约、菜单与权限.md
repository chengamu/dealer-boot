# 履约、菜单与权限

## 履约

生产队列只接受 `SUBMITTED + PAID` 订单。开始生产和完成生产读取订单冻结 BOM；生产 PDF 不展示价格。

生产完成后可创建多个草稿包裹。每个包裹按销售订单行分配整数数量，所有已发包裹聚合更新：

- `shipment_count`
- `shipped_quantity`
- `shipment_status`
- 首次和最近发货时间
- 最近承运商与单号

商家逐包裹确认收货；所有订单行已全部发出且所有包裹确认后，订单进入 `COMPLETED / DELIVERED`。平台代确认需要高权限和原因。

业务、平台和工厂可复用履约 Service 与详情组件，但必须从各自菜单、Controller/API 和权限入口进入。商家/内部业务查询继续受 `tenant_id + business_origin + 部门 data_scope` 约束；工厂生产角色通过平台履约与工厂生产/发货权限执行跨租户作业。

## 菜单

业务闭环已从旧“销售业务七入口”拆为职责清晰的工作区：

| 工作区 | 页面 |
| --- | --- |
| 销售业务 | 销售首页、工程测算、报价管理、快速下单 |
| 订单管理 | 销售订单 |
| 财务中心 | 付款记录、信用账户、应付账单 |
| 订单跟踪 | 生产进度、发货包裹、物流跟踪 |
| 销售运营 | 销售首页、报价查询、下单查询、订单查询 |
| 平台财务 | 支付单据、转账审核、信用账户、信用应收、异常对账 |
| 履约中心 | 生产管理、发货管理、包裹查询、物流跟踪 |
| 生产作业 | 生产队列、生产清单 |
| 发货作业 | 发货队列、包裹管理、物流跟踪 |

快速下单的工作台和复核页是隐藏路由，侧边栏只展示草稿列表入口。无权限入口和按钮直接隐藏，后端继续做最终权限、数据范围与状态校验。

## 角色与默认首页

系统固定角色共 8 个：管理员、平台运维、技术科、业务主管、平台销售、工厂生产、店主和营业员。店主 `merchant_admin` 数据范围为本部门及以下，营业员 `merchant_employee` 为仅本人；业务主管合并商家运营与销售运营，工厂生产合并平台财务、平台履约、工厂生产和发货职责。

平台运维可以只读查看系统治理配置，并执行缓存清理、在线用户强制下线、操作日志与登录日志导出、登录账户解锁；用户、角色、菜单等系统治理修改仍只允许管理员，平台运维不能删除或清空日志。技术科默认进入配方模拟，平台运维默认进入服务器资源。

`sys_role`、`sys_menu`、`sys_role_menu` 统一为 `tenant_id=1` 的平台全局数据。商家用户通过保留商户 `tenant_id` 的 `sys_user_role` 关联平台固定商户角色，商家侧不能维护角色或菜单。

普通角色的登录落点由 `sys_role.default_menu_id` 指向本角色已授权的可见有效页面；多角色用户按角色排序解析第一个有效默认页。超级管理员按 `monitor:server:list` 解析服务器资源页；默认页缺失、停用或越权时降级到中性 `/index`。默认首页不授予任何新权限。

所有接口查询必须同时满足菜单/接口权限、租户与部门数据范围、来单归属和业务状态。
