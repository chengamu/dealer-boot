# 履约、菜单与权限

## 履约

生产队列只接受 `SUBMITTED + PAID` 订单。开始生产和完成生产读取订单冻结 BOM；生产 PDF 不展示价格。

生产完成后可创建多个草稿包裹。每个包裹按销售订单行分配整数数量，所有已发包裹聚合更新：

- `shipment_count`
- `shipped_quantity`
- `shipment_status`
- 首次和最近发货时间
- 最近承运商与单号

商家逐包裹确认收货；所有订单行已全部发出且所有包裹确认后，订单进入 `COMPLETED / DELIVERED`。平台代确认需要高权限和原因。

接口族为 `/dealer/fulfillment/**`，覆盖生产列表/详情、开始/完成、包裹 CRUD、发货、物流同步、轨迹和确认收货。

## 菜单

一级菜单“销售业务”当前包含七个入口：

1. 销售仪表盘
2. 工程测算
3. 报价管理
4. 快速下单
5. 销售订单
6. 支付结算
7. 履约管理

无权限入口和按钮直接隐藏，后端继续做最终权限与状态校验。旧 `Order Estimates`、旧订单详情付款/生产/发货动作和旧确认签收菜单均已退场。

## 角色

- 商家负责人：本租户全链路。
- 商家销售：客户、工程测算、报价、快速下单和订单查询。
- 商家财务：支付查询、银行提交、信用使用和应收查询。
- 商家跟单：订单与物流跟踪、确认收货。
- 平台管理员：跨商家查询、支付修复、履约异常处理。
- 平台销售：按本人/团队/分配商家读取报价和订单。
- 平台财务：支付、银行审核、信用和对账。
- 工厂生产：已付款订单、生产和生产单据。
- 工厂发货：完工订单、包裹、发货和轨迹。

接口查询必须同时满足权限、租户/角色数据范围和业务状态。
