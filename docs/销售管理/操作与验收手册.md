# 操作与验收手册

## 工程链路

1. 新建工程测算，录入客户、项目、尺寸、数量和动态配置。
2. 整单测算，确认客户报价。
3. 生成中/英文 Excel、PDF；邮件发送使用同一冻结文件。
4. 预览商家等级折扣，补齐收货信息并转正式订单。
5. 在支付结算选择已配置的 PayPal、银行转账或信用额度。
6. 付款成功后进入生产，完成后创建一个或多个包裹。
7. 发货、同步轨迹，商家逐包裹确认收货。

## 快速链路

1. 新建快速下单，逐件选择产品、分数英寸尺寸和动态配置。
2. 加入订单并在复核页确认产品折扣、邮费和总额。
3. 提交后直接生成正式订单。
4. 后续支付、生产、发货和签收与工程链路一致。

## 验收重点

- 同一产品配置在工程测算和快速下单使用同一稳定选项编码。
- 客户报价不展示商家折扣；正式订单只折扣产品金额。
- 页面、Excel、PDF、邮件和支付金额一致。
- 报价转单与快速下单提交重复调用均不重复创建订单。
- 未付款不能生产，未完工不能发货，包裹不能超量。
- 商家不能读取其他租户数据，平台普通员工不能越过角色数据范围。
- PDF 中文字体必须可渲染；部署环境配置 `PDF_FONT_PATH`。
- PayPal 生产联调必须使用 Sandbox/生产环境变量完成，不在仓库保存密钥。
- 商家和内部销售的同一业务操作必须分别命中 `MERCHANT` 与 `INTERNAL`，并按当前租户、部门 `data_scope` 和负责人过滤。
- 平台销售、财务与履约只能通过各自 `platform:*` 权限和专用接口跨租户；业务权限不能读取其他租户。
- 快速下单工作台和复核页只从草稿链路进入，侧边栏不出现重复菜单。
- 报价、快速下单和订单列表使用标准 Grid；支付异常、生产、发货、物流和销售首页按确认图做真实浏览器视觉对照。
- 浏览器验收结论只在实际执行后填写；未执行时保持“待验证”，不以类型检查或文档检查代替。

## 报价性能验收

分别使用 `1/10/20/50` 行执行不少于 30 轮保存，记录后端总耗时、SQL 数量、锁等待和 `p50/p95/p99`：

- 同一商品多行：验证请求级价格运行时和同币种邮费模板只加载一次。
- 不同商品多行：验证最坏情况下 SQL 随不同商品数增长，而不是因保存后重复整单计算翻倍。
- 保存响应与旧权威计算结果逐字段比对：正式配方版本、选项编码、BOM、单价、产品金额、邮费、总额、币种与失败信息必须一致。
- 50 行保存只允许一次旧明细删除、一次 MyBatis 批量插入和一次主表合计更新；不得逐行插入。

性能数据受机器和数据库状态影响，验收报告必须保留测试环境、样本数量和原始统计，不能只记录单次最快耗时。

本地测试数据可重建；正式环境不得通过初始化脚本重置管理员密码或写入支付凭据。
