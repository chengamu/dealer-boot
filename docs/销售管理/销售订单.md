# 销售订单

销售订单是两条销售入口汇合后的正式交易事实，不提供新增、配置、测算、报价或草稿编辑。

## 详情

订单详情展示来源类型与来源单号、客户和收货信息、冻结产品行、尺寸、数量、动态配置、价格、邮费、付款、生产、包裹和事件流水。产品行可展开查看完整配置；订单和生产 PDF 可同屏预览并收起。

双来源字段：

- 报价：`source_quote_id / source_quote_item_id`
- 快速下单：`source_quick_order_id / source_quick_order_item_id`
- 统一追溯：`source_type / source_no`

## 接口

- `GET /dealer/sales-documents/list`
- `GET /dealer/sales-documents/{id}`
- `GET /dealer/sales-documents/customer/{customerId}`
- `PUT /dealer/sales-documents/{id}/cancel`
- `GET /dealer/sales-documents/{id}/pdf?type=ORDER|PRODUCTION`
- `POST /dealer/sales-documents/{id}/export`
- `POST /dealer/sales-documents/{id}/email`

付款、生产、发货和签收由支付结算与履约管理模块执行，订单详情只显示事实和导航，不保留第二套写入口。

## 数据范围

- 商家账号只能读取本租户订单。
- 平台管理员读取全部商家。
- 平台销售读取本人、团队或被分配商家范围。
- 工厂角色只读取已付款并进入履约范围的订单。

跨租户查询必须先判定角色数据范围；禁止仅凭 `ignoreTenant` 暴露全部数据。
