# 销售管理知识库

销售管理是客户配置、报价、正式交易、支付和工厂履约的完整业务域。当前实现提供两条销售入口，并统一汇入正式销售订单：

```text
工程测算 -> 确认客户报价 -> 转销售订单
快速下单 -> 订单复核 -> 直接生成销售订单
销售订单 -> 支付 -> 生产 -> 多包裹发货 -> 签收完成
```

商家销售与平台内部销售复用上述业务模型与接口族，行级范围固定为 `tenant_id + business_origin + 现有部门 data_scope`；`sales_store_id` 只记录平台内部来单的销售门店来源，不是新的授权体系。平台销售运营、平台财务和平台履约使用独立菜单、Controller/API 和 `platform:*` 权限族执行受控跨租户查询或动作。

阶段 12 的代码与 SQL 实现已同步到本知识库。真实浏览器分角色、越权与视觉最终结果仍以 [阶段 12 设计 QA](../../design/sales-management/design-qa.md) 的实际记录为准。
