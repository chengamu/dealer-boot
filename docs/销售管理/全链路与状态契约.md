# 全链路与状态契约

## 数据关系

- `customer_quote / customer_quote_item`：工程测算与客户报价，确认时冻结配置、BOM、价格和独立邮费快照。
- `dealer_quick_order / dealer_quick_order_item`：零售小单草稿，提交前按当前规则严格复核。
- `dealer_sales_document / dealer_sales_document_item`：正式交易事实，`source_type` 为 `QUOTE` 或 `QUICK_ORDER`。
- `pay_order / pay_order_extension`：一张销售订单对应一个业务支付单，渠道尝试可有多次，成功事实只能有一个。
- `merchant_credit_account / merchant_credit_transaction / merchant_receivable`：商家额度、不可变流水和订单应收。
- `dealer_shipment / dealer_shipment_item / dealer_tracking_event`：多包裹、装箱数量和物流轨迹。
- `dealer_sales_document_event`：付款、生产、发货和签收操作流水，不使用逻辑删除。

来源通过结构化 ID 精确关联，禁止通过 JSON `LIKE` 判断引用。报价和快速下单都只能生成一张正式订单；重复提交返回已有订单。

## 业务归属

报价、快速下单和正式订单保存 `tenant_id`、`business_origin`、`sales_store_id`、`dept_id` 和 `owner_user_id`。

- 商家来单：`business_origin = MERCHANT`，`tenant_id` 为当前商家租户，`sales_store_id = NULL`。
- 内部来单：`business_origin = INTERNAL`，`tenant_id = 1`，`sales_store_id` 来自当前用户部门关联的有效销售门店。
- `dept_id` 和 `owner_user_id` 是现有角色 `data_scope` 的行级过滤依据；`sales_store_id` 不参与权限计算。
- 报价转单和快速下单提交必须原样传递业务归属，支付与履约不重新推断归属。

## 状态机

工程报价：`DRAFT -> CONFIRMED -> VOID`。已转订单的确认报价不可作废；调整时复制为新草稿。

快速下单：`DRAFT -> ORDERED`。已提交草稿只读。

正式订单主状态：

- `SUBMITTED`：正式订单。
- `CANCELLED`：未付款且未生产时取消，不可恢复。
- `COMPLETED`：全部已发包裹确认收货。

独立状态：

- 付款：`UNPAID -> PAID`
- 生产：`PENDING -> IN_PRODUCTION -> COMPLETED`
- 发货：`UNSHIPPED -> PARTIALLY_SHIPPED -> SHIPPED -> DELIVERED`
- 包裹：`DRAFT -> DISPATCHED -> IN_TRANSIT / CARRIER_DELIVERED / EXCEPTION`
- 收货：`PENDING -> CONFIRMED`

未付款不能生产，未完工不能发货，装箱不能超过订单行数量。所有包裹确认收货后订单自动完成。

## 快照与金额

- 报价确认后只读取冻结快照；转订单不重新解释配方、价格、邮费或 BOM。
- 快速下单提交时重新读取当前正式配方、价格、邮费和商家等级折扣；确认后全部冻结到订单。
- 客户报价不应用商家折扣。商家折扣只作用于正式订单产品金额，邮费不打折。
- `listAmount - discountAmount = productAmount`
- `productAmount + shippingAmount + taxAmount = totalAmount`
- 第一版币种为价格设置币种，当前 `USD`；`taxAmount = 0`。
- 物料单价保留 4 位，客户和订单交易金额冻结 2 位；尺寸和数量遵循数字精度规范。
- 生产清单只读取冻结 BOM，不展示价格。

关键写操作按业务 ID 使用统一锁和旧状态条件更新。所有业务时间以 UTC 写入 `timestamptz`，页面按用户时区显示到分钟。
