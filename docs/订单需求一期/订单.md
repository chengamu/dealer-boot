# Window Covering Dealer Portal - 需求规格说明书

**版本**：V1.0
**日期**：2026年5月12日
**目标市场**：美国
**目标用户**：美国窗帘经销商（Dealer / Blind Shop / Interior Designer）

## 1. 项目概述

### 1.1 系统定位

本系统本质属于 Configure Price Quote（CPQ）+ Dealer Portal + Configure-To-Order（CTO）Manufacturing System，不是传统电商系统。

系统核心目标：帮助窗帘工厂实现Dealer在线配置产品、自动价格计算、自动生产约束校验、Dealer级价格体系、Quote到Order自动转换、在线支付、ERP生产状态同步。

### 1.2 核心业务流程

```
Dealer注册（邮箱验证）→ 工厂审核激活 →
Dealer创建客户 → 选择产品配置（下拉 + 分数宽高）
→ Pricing Engine计算价格（Price Grid + 加价 + 约束校验）
→ 生成报价（价格快照）→ Checkout → PayPal支付 →
支付成功 → 生成正式订单（完整快照）→ 工厂查看订单 → ERP同步生产状态 → Dealer追踪订单
```

### 1.3 用户角色

| 角色 | 说明 |
| :--- | :--- |
| 上游管理员（工厂） | 产品管理、定价规则、商户审核、订单监控、生产约束配置 |
| 商户（Dealer） | 美国经销商，为客户创建报价、在线支付、追踪订单 |

### 3.2 Pricing Engine（定价引擎）

#### 3.2.1 定价逻辑概述

窗帘行业最终价格由以下因素共同决定：

| 因素 | 说明 |
| :--- | :--- |
| 宽度 + 高度 | 尺寸越大，价格越高 |
| 面料类型 | 普通 / 遮光 / 高端面料 |
| 控制方式 | 拉珠 / 电动 |
| 安装方式 | Inside / Outside |
| 配件 | 底轨、侧轨、遥控器等 |
| Dealer等级 | Bronze / Silver / Gold / VIP |
| Dealer加价 | 经销商自己的利润率（统一加价系数） |

系统将所有规则自动串起来，实时计算最终报价。

#### 3.2.2 完整计算公式

**最终客户价格公式**：

```
最终客户价格 = 
[Price Grid基础价 + 固定加价 + (Price Grid基础价 × 百分比加价)]
× Dealer等级折扣
× Dealer加价系数
× 数量
```

**加价规则说明**：百分比加价仅基于 Price Grid 基础价计算，固定加价与百分比加价并列叠加，不重复计算。

**逐行分解**：

| 步骤 | 计算项 | 公式 | 示例值 |
| :--- | :--- | :--- | :--- |
| 1 | base_price | 从Price Grid匹配 | $102.00 |
| 2 | factory_base_price | base_price + 固定加价 + (base_price × 百分比加价) | $127.20 |
| 3 | dealer_cost | factory_base_price × Dealer等级折扣 | $114.48 |
| 4 | final_unit_price | dealer_cost × Dealer加价系数 | $143.10 |
| 5 | line_total | final_unit_price × 数量 | $286.20 |

#### 3.2.3 完整计算示例

**输入（Dealer选择的配置）**：

| 配置项 | 选择值 |
| :--- | :--- |
| 产品型号 | RB100 Roller Shade |
| 面料 | YB180002 |
| 宽度 | 30 1/8" |
| 高度 | 60 1/2" |
| 控制方式 | Motorized（电动） |
| 安装方式 | Inside Mount |
| 底轨 | Fabric Wrap |
| 数量 | 2 |
| Dealer等级 | Gold（9折） |
| Dealer加价系数 | 1.25（加价25%） |

**计算过程**：

| 步骤 | 计算 | 结果 |
| :--- | :--- | :--- |
| Step 1 | Price Grid匹配（宽度30-36 / 高度60-72区间） | base_price = $102.00 |
| Step 2 | 电动电机加价10%（仅基于base_price） | +$10.20 |
| Step 3 | 底轨固定加价 | +$15.00 |
| Step 4 | 工厂小计 | $127.20 |
| Step 5 | Gold等级9折 | $114.48 |
| Step 6 | Dealer加价25% | $143.10 |
| Step 7 | × 2个 | **$286.20** |

**最终报价：$286.20**

#### 3.2.4 Price Grid 与产品配置的绑定关系

Price Grid 不是独立存在的价格表，必须与具体产品配置关联。

每一条价格矩阵记录，至少需要绑定以下维度：

| 维度 | 说明 |
| :--- | :--- |
| Product Category | 产品分类，如 Roller Shades |
| Product Code | 产品型号，如 RB100 |
| Fabric Code | 面料编码，如 YB180002 |
| Mechanism | 控制方式，如 Chain / Motorized |
| Mount Type | 安装方式，如 Inside / Outside |
| Width Range | 宽度区间 |
| Height Range | 高度区间 |
| Base Price | 对应基础价格 |
| Effective Date | 生效日期 |

同一个尺寸，在不同产品、不同面料、不同控制方式下，价格可能不同。Price Grid 导入时，系统必须根据 Excel 中的 Product Code、Fabric Code、Mechanism、Mount Type 自动匹配系统内已有的产品配置数据。

#### 3.2.5 Price Grid 导入前置条件

工厂在导入价格矩阵前，必须先维护以下基础数据：

| 前置数据 | 是否必须 | 说明 |
| :--- | :--- | :--- |
| 产品分类 | 是 | 如 Roller Shades |
| 产品型号 | 是 | 如 RB100 |
| 面料编码 | 是 | 如 YB180002 |
| 控制方式 | 是 | Chain / Motorized |
| 安装方式 | 是 | Inside / Outside |
| 配置项选项值 | 是 | 用于校验 Excel 内容是否合法 |

如果 Excel 中出现系统不存在的 Product Code、Fabric Code 或配置选项，系统应拒绝导入，并提示错误原因。

#### 3.2.6 Price Grid Excel 导入流程

```
Step 1：工厂先维护产品基础数据
    ↓
Step 2：工厂下载 Price Grid Excel 模板
    ↓
Step 3：工厂填写产品编码、面料编码、尺寸区间、价格、生效日期
    ↓
Step 4：上传 Excel
    ↓
Step 5：系统校验 Excel 数据
    ↓
Step 6：校验 Product Code / Fabric Code / Mechanism / Mount Type 是否存在
    ↓
Step 7：校验尺寸区间是否冲突
    ↓
Step 8：校验价格、日期、币种是否合法
    ↓
Step 9：校验通过后入库
    ↓
Step 10：Dealer 报价时自动匹配对应价格
```

| 校验项 | 规则 | 失败处理 |
| :--- | :--- | :--- |
| Product Code | 必须存在于产品表 | 拒绝导入 |
| Fabric Code | 必须存在于面料/配置选项中 | 拒绝导入 |
| Mechanism | 必须是系统已维护选项 | 拒绝导入 |
| Mount Type | 必须是系统已维护选项 | 拒绝导入 |
| Width From/To | 必须为正数，From < To | 拒绝导入 |
| Height From/To | 必须为正数，From < To | 拒绝导入 |
| Base Price | 必须大于等于0，保留两位小数 | 拒绝导入 |
| Effective From | 必填 | 拒绝导入 |

#### 3.2.9 导入模式

| 模式 | 说明 | 适用场景 | 优先级 |
| :--- | :--- | :--- | :--- |
| Append | 追加导入，不覆盖已有数据 | 新增价格区间 | P0 |
| Override | 覆盖相同维度和区间的数据 | 更新已有价格 | P0 |
| New Version | 生成新价格版本 | 调价但保留历史价格 | P1 |

#### 3.2.10 Price Grid 命中优先级

当多个价格规则同时命中时（如Product级规则、Fabric级规则、Category级规则），系统按以下优先级匹配：

| 优先级 | 规则级别 | 说明 |
| :--- | :--- | :--- |
| 1（最高） | Product + Fabric | 特定产品+特定面料组合 |
| 2 | Product | 特定产品，适用于所有面料 |
| 3 | Category + Fabric | 产品分类+特定面料组合 |
| 4 | Category | 产品分类，适用于所有面料 |
| 5（最低） | Global | 全局默认价格 |

匹配逻辑：从上到下查找，命中第一条即停止。

#### 3.2.11 Dealer 报价时的价格匹配流程

当 Dealer 配置一个产品时，系统根据以下条件匹配 Price Grid：

```
Product Code
+ Fabric Code
+ Mechanism
+ Mount Type
+ Rounded Width
+ Rounded Height
+ Quote Date
```

匹配逻辑：

```
1. 根据 Product Code 找产品
2. 根据 Fabric Code 找面料
3. 根据 Mechanism 找控制方式
4. 根据 Mount Type 找安装方式
5. 宽高向上取整
6. 按优先级匹配 Price Grid（Product+Fabric > Product > Category+Fabric > Category > Global）
7. 匹配对应 Width Range 和 Height Range
8. 匹配当前日期有效的价格版本
9. 返回 Base Price
```

如果没有匹配到价格：

```
系统提示：No price found for the selected configuration. Please contact factory administrator.
```

#### 3.2.12 示例：从 Excel 到自动报价

**工厂导入 Excel**：

| Product Code | Fabric Code | Mechanism | Mount Type | Width From | Width To | Height From | Height To | Base Price |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| RB100 | YB180002 | Chain | Inside | 30 | 36 | 60 | 72 | 102.00 |

**Dealer 选择**：

| 配置 | 值 |
| :--- | :--- |
| Product | RB100 |
| Fabric | YB180002 |
| Mechanism | Chain |
| Mount | Inside |
| Width | 30 1/8" |
| Height | 60 1/2" |

**系统处理**：

```
Width 30 1/8" → 向上匹配 30-36 区间
Height 60 1/2" → 向上匹配 60-72 区间
命中：Base Price = $102.00
```

然后继续计算：

```
Base Price
+ Fixed Surcharge
+ Percentage Surcharge
× Dealer Tier Discount
× Dealer Margin
× Quantity
```

最终生成报价。

#### 3.2.13 加价规则（Surcharge）

| 类型 | 说明 | 示例 | 计算方式 |
| :--- | :--- | :--- | :--- |
| 固定金额加价 | 选了这个配件，加固定美元 | 底轨 +$15 | 直接加到 base_price |
| 百分比加价 | 选了这个配置，加百分比 | 电动电机 +10% | 仅基于 base_price 计算，不与固定加价叠加计算 |

固定加价和百分比加价同时生效时：
```
factory_base_price = base_price + 固定加价 + (base_price × 百分比加价)
```

#### 3.2.15 边界情况处理

| 场景 | 系统行为 |
| :--- | :--- |
| 尺寸未命中任何Price Grid区间 | 返回最高可用价格 + 警告，不允许直接下单 |
| Price Grid区间重叠 | Excel导入时拒绝，提示冲突 |
| 加价规则冲突（如两个配件都加价） | 全部叠加 |

### 3.3 Production Constraints（生产约束）

#### 3.3.1 约束规则定义

| 约束类型 | 说明 | 示例 |

### 4.5 结算与支付（Checkout & Payment）

#### 4.5.1 结算金额构成

最终结算金额（Grand Total）包含以下部分：

| 项目 | 说明 | 计算方式 |
| :--- | :--- | :--- |
| Subtotal | 产品总价 | 所有line_total之和 |
| Shipping Fee | 运费 | 见运费策略 |
| Tax Amount | 税费 | V1默认为0 |
| Grand Total | 最终应付金额 | Subtotal + Shipping Fee + Tax Amount |

#### 4.5.2 税费策略（Tax Strategy）

V1默认不自动计算美国Sales Tax。原因如下：

- 美国州税规则复杂，不同州、城市税率不同
- Dealer可能具备免税资格（Tax Exemption / Resale Certificate）
- 不同运输州存在不同税务规则

因此V1默认：

| 配置 | 值 |
| :--- | :--- |
| Tax Amount | 0 |
| Tax Calculation | 不自动计算 |
| Tax Field | 系统保留字段，供未来扩展 |

未来可扩展第三方税务服务：Avalara、TaxJar、或基于州的税率规则。

#### 4.5.3 运费策略（Shipping Strategy）

| 模式 | 说明 | 优先级 |
| :--- | :--- | :--- |
| Flat Rate | 固定运费（如$20/订单） | P1 |
| By Order Value | 按订单金额阶梯（满额免运费） | P2 |
| Manual Quote | 工厂人工填写运费 | P2 |

V1推荐：Flat Rate，工厂后台可配置固定运费金额。默认$0。

#### 4.5.4 货币策略（Currency Strategy）

V1仅支持美元（USD），但系统预留多币种扩展能力。

| 规则 | 说明 |
| :--- | :--- |
| 默认货币 | USD（美元） |
| 金额存储 | 订单表、报价单表均存储currency_code字段 |
| 汇率 | V1不做汇率转换，固定为USD |
| 扩展预留 | 未来支持CAD、EUR等货币时，增加exchange_rate字段 |

#### 4.5.5 PayPal支付流程

V1仅支持PayPal在线支付。

**支付流程**：

```
Quote状态为ACTIVE
    ↓
Dealer点击Checkout
    ↓
进入Checkout页面
    ↓
确认Subtotal + Shipping + Tax
    ↓
点击"Pay with PayPal"
    ↓
系统创建PayPal Order（金额：Grand Total，币种：USD）
    ↓
跳转PayPal Checkout页面
    ↓
Dealer完成支付
    ↓
PayPal回调支付结果
    ↓
支付成功 → 系统生成正式Order
    ↓
Quote状态改为CONVERTED
    ↓
写入Order Item Snapshot（包含完整配置快照）
    ↓
写入Shipping Address Snapshot
    ↓
通知工厂（邮件/站内信）
    ↓
跳转到订单详情页
```

**支付状态**：

| 状态码 | 状态名 | 说明 |
| :--- | :--- | :--- |
| UNPAID | 未支付 | Checkout创建后，等待支付 |
| PAID | 已支付 | PayPal支付成功，订单已生成 |
| FAILED | 支付失败 | 支付过程中出错 |
| REFUNDED | 已退款 | 订单已退款 |

**关键规则**：

- 未支付成功前，系统不生成正式订单
- PayPal支付成功后，才创建Order并锁定订单快照
- 支付失败时，Quote保持ACTIVE状态，Dealer可重新发起支付

#### 4.5.6 Quote to Order 转换规则

当 Dealer 完成支付后，系统执行以下操作：

| 步骤 | 操作 |
| :--- | :--- |
| 1 | 校验 Quote 是否过期（状态为ACTIVE） |
| 2 | 重新执行生产约束校验 |
| 3 | 重新验证 Price Grid 是否存在（防止价格表变更后失效） |
| 4 | 重新计算 Checkout Summary |
| 5 | 锁定最终价格快照 |
| 6 | 生成正式 Order Number（格式：ORD-YYYYMMDD-XXXXXX） |
| 7 | Quote 状态改为 CONVERTED |
| 8 | 创建 Order 记录 |
| 9 | 写入 Order Item Snapshot（完整配置文本快照） |
| 10 | 写入 Shipping Address Snapshot |
| 11 | 通知工厂（邮件/站内信） |

#### 4.5.8 PayPal Webhook 幂等性

PayPal 回调可能重复推送、延迟或乱序，系统必须保证幂等处理。

| 规则 | 说明 |
| :--- | :--- |
| 唯一标识 | 以 PayPal `capture_id` 作为支付唯一凭证 |
| 幂等校验 | 同一 `capture_id` 只能处理一次，重复回调直接忽略 |
| 状态校验 | 仅 `COMPLETED` 状态触发订单生成，其他状态记录日志 |
| 重试安全 | 订单生成后再次收到相同回调不重复创建 |

## 5. 数据规范与关键业务规则

### 5.1 金额精度规范

| 规则 | 说明 |
| :--- | :--- |
| 数据库存储 | 所有金额字段统一使用 `DECIMAL(18,4)`，保留4位小数 |
| 内部计算 | 定价引擎计算过程保留4位小数 |
| 前端展示 | 格式化显示为2位小数（如 $102.00） |
| 禁止类型 | 严禁使用 `float` 或 `double` 类型存储金额 |

### 5.2 货币字段预留

所有订单表、报价单表均包含货币字段：

| 字段 | 类型 | 说明 | V1值 |
| :--- | :--- | :--- | :--- |
| currency_code | VARCHAR(3) | 货币代码 | USD |
| exchange_rate | DECIMAL(18,6) | 汇率（对USD） | 1.000000 |

V1中汇率固定为1，但字段预留便于未来多币种扩展。

### 5.3 Order Snapshot 完整冻结

支付成功后生成的订单快照，不仅冻结价格，还必须冻结完整的配置文本。

| 冻结内容 | 说明 | 示例 |
| :--- | :--- | :--- |
| 产品信息 | 产品名称、型号、版本 | Roller Shades RB100 V1 |
| 面料信息 | 面料编码、名称 | YB180002 Blackout Fabric |
| 配置选项 | 所有下拉选项的显示文本 | Chain Operated, Left, Inside Mount |
| 尺寸信息 | 客户尺寸 + 生产尺寸 | 30 1/8" → 29 5/8" |
| 配件信息 | 配件名称和加价 | Bottom Rail (+$15.00) |
| 价格快照 | 完整的定价计算过程 | base_price, surcharge, discount... |
| Dealer信息 | Dealer名称、等级、Margin系数 | Gold Dealer, Margin 1.25 |
| 客户信息 | 客户名称（快照，不与客户表关联） | ABC Construction |

**关键规则**：订单生成后，即使工厂删除或修改了产品、面料、配置选项，历史订单详情页仍能完整显示下单时的配置。订单快照中存储的是文本副本，不是外键引用。

### 5.4 收货地址快照

订单生成时，必须冻结收货地址快照，后续客户修改地址不影响历史订单。

| 字段 | 说明 |
| :--- | :--- |
| shipping_company | 公司名称（快照） |
| shipping_name | 收货人姓名 |
| shipping_address_line1 | 地址第一行 |
| shipping_address_line2 | 地址第二行（可选） |
| shipping_city | 城市 |
| shipping_state | 州 |
| shipping_zip | 邮编 |
| shipping_phone | 联系电话 |
| shipping_email | 邮箱 |

**关键规则**：订单生成后，即使Dealer修改了客户的主地址，该订单的发货地址保持不变。

### 5.5 Dealer组织架构预留

V1版本中，一个Dealer对应一个账号。但系统设计需预留多账号扩展能力。

| 预留字段 | 说明 | V1行为 |
| :--- | :--- | :--- |
| dealer_company_id | 经销商公司ID | 每个Dealer账号独立，company_id等于自己 |
| parent_dealer_id | 上级经销商 | 固定为NULL |
| role | 用户角色 | 固定为 DEALER_ADMIN |

**未来扩展方向**（V2/V3）：

```
Dealer Company (公司)
    ├── Owner (拥有者)
    ├── Sales (销售员，只能报价)
    ├── Designer (设计师，只能配置)
    └── Installer (安装工，只读订单)
```

V1不做多用户，但数据库表结构预留字段。

### 5.6 Dealer税费豁免预留

| 字段 | 类型 | 说明 | V1默认值 |
| :--- | :--- | :--- | :--- |
| tax_exempt | BOOLEAN | 是否免税 | false |
| tax_exempt_number | VARCHAR(50) | 免税证号 | NULL |
| resale_certificate | VARCHAR(255) | 免税证书文件URL | NULL |

V1中这些字段仅存储，不参与税费计算。V2扩展时可用于自动税费计算。

### 5.7 附件存储规范

| 规则 | 说明 |
| :--- | :--- |
| 存储方式 | 对象存储（S3 / Cloudflare R2 / MinIO），不存储在本地磁盘 |
| 文件命名 | UUID格式，避免文件名冲突 |
| 访问控制 | 生成带时效的临时访问URL（如15分钟有效期） |
| 存储隔离 | 按Dealer ID分目录存储 |
| 容量告警 | 预留存储容量告警机制 |

### 5.8 业务规则汇总

| 编号 | 规则 | 说明 |
| :--- | :--- | :--- |
| BR-01 | 报价有效期 | 默认30天，工厂可全局配置 |
| BR-02 | 过期不可支付 | EXPIRED状态不可进入Checkout |
| BR-03 | 价格快照 | 支付成功后锁定当前价格到item级别 |
| BR-04 | 配置快照 | 订单快照包含完整配置文本，不依赖外键 |
| BR-05 | 历史订单不变 | 订单生成后价格和配置不受后续变更影响 |
| BR-06 | 草稿可编辑 | DRAFT状态完全可编辑 |
| BR-07 | 支付后不可编辑 | 支付成功后Order已生成，不可修改 |
| BR-08 | 取整方向 | 始终向上取整（生产安全原则） |
| BR-09 | 扣减模板 | 宽高独立扣减，支持规则版本 |
| BR-10 | 工厂价格更新不影响已支付订单 | 通过价格快照保证 |
| BR-11 | 软删除 | 产品、价格、面料等核心数据不支持物理删除 |
| BR-12 | 订单号生成 | ORD-YYYYMMDD-XXXXXX格式，全局唯一 |
| BR-13 | 报价单号生成 | QTE-YYYYMMDD-XXXXXX格式，全局唯一 |
| BR-14 | 支付与订单关系 | 支付成功后才生成正式订单 |
| BR-15 | 百分比加价基准 | 百分比加价仅基于Price Grid基础价，不与固定加价叠加计算 |
| BR-16 | PayPal幂等 | 同一capture_id只能处理一次 |
| BR-17 | 报价重新计算 | EXPIRED重新计算时生成新报价单，不覆盖原报价 |
| BR-18 | Price Grid优先级 | Product+Fabric > Product > Category+Fabric > Category > Global |
