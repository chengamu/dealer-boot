## 1. AI战略定位

本系统不仅仅是窗帘行业的配置型报价系统（CPQ），更是一个面向未来的 **AI Native SaaS 平台**。

AI不是附加功能，而是系统的核心能力层。通过AI技术，系统能够自动完成报价、配置推荐、约束解释、客户支持等任务，大幅降低Dealer的学习成本和操作门槛，同时减少工厂的人工介入。

**AI战略三原则**：

| 原则 | 说明 |
| :--- | :--- |
| 渐进增强 | AI能力分阶段上线，每个阶段独立可用 |
| 人机协同 | AI辅助决策，最终审批权在人工 |
| 数据闭环 | AI输出可反馈、可学习、可优化 |

## 3. AI能力详情

### 3.1 AI智能报价助手（Phase 2）

**功能描述**：

当Dealer输入客户需求（如“我需要一个卧室遮光卷帘，大约35寸宽，60寸长”），系统自动推荐最接近的产品配置和价格。

**使用场景**：

- Dealer不熟悉产品线
- Dealer快速创建报价
- 新Dealer培训期

**输入示例**：

```
用户输入：I need a blackout roller shade for bedroom, around 35" wide, 60" long.
```

**系统输出**：

```
AI推荐配置：
- Product: Roller Shades
- Fabric: YB180002 Blackout
- Width: 35" (建议尺寸)
- Length: 60"
- Control: Chain Operated
- Mount: Inside Mount
- Estimated Price: $142.50

[Apply Configuration] [Modify]
```

**技术实现**：

| 组件 | 说明 |
| :--- | :--- |
| 模型能力 | 自然语言理解 + 产品知识库检索 |
| 数据来源 | 产品目录、历史报价、Price Grid |
| 输出格式 | 结构化配置 + 价格预估 |

**优先级**：P1（V1.5）

---

### 3.2 AI产品配置助手（Phase 2）

**功能描述**：

当Dealer在产品配置页面选择选项时，AI主动推荐下一步最佳选择，并解释推荐原因。

**使用场景**：

- Dealer不确定某个选项的含义
- Dealer想快速完成配置
- 减少配置错误

**交互示例**：

```
Dealer选择了：Fabric = YB180002 Blackout

AI提示：
💡 Based on your fabric selection, we recommend:
   - Bottom Rail: Fabric Wrap (matches fabric)
   - Control: Chain Operated (most popular with this fabric)
   
[Apply All] [Ignore]
```

**技术实现**：

| 组件 | 说明 |
| :--- | :--- |
| 模型能力 | 协同过滤 + 关联规则学习 |
| 数据来源 | 历史报价单、配置完成率、错误率 |
| 输出格式 | 推荐选项 + 推荐理由 |

**优先级**：P1（V1.5）

---

### 3.3 AI约束解释（Phase 3）

**功能描述**：

当Dealer的配置违反生产约束时，系统不只显示错误信息，还用自然语言解释原因，并提供修改建议。

**使用场景**：

- Dealer配置被拦截
- Dealer不理解为什么不能选某个选项
- 减少Dealer反复尝试

**错误提示对比**：

| 传统提示 | AI解释 |
| :--- | :--- |
| Width exceeds maximum (96") | The fabric "YB180002" has a physical width limit of 96 inches. Your current width of 120" would cause the fabric to sag. Please reduce width to 96" or less, or select a different fabric. |

**技术实现**：

| 组件 | 说明 |
| :--- | :--- |
| 模型能力 | 约束规则 + 自然语言生成 |
| 数据来源 | 约束规则定义、产品物理参数 |
| 输出格式 | 解释文本 + 修改建议 + 替代方案 |

**优先级**：P2（V2.0）

### 3.4 AI Proposal PDF生成（Phase 3）

**功能描述**：

系统自动生成面向终端业主的专业英文Proposal PDF，包含产品说明、安装建议、维护指南，而非简单的价格列表。

**使用场景**：

- Dealer需要给终端客户发报价
- Dealer希望提升专业形象
- 减少Dealer手工制作Proposal的时间

**Proposal内容结构**：

```
[Company Logo]
[Dealer Name] Proposal

Customer: ABC Construction
Date: 2026-05-13

PRODUCT SUMMARY
- Roller Shades × 2
- Fabric: YB180002 Blackout
- Size: 30 1/8" × 60 1/2"

PRODUCT DESCRIPTION
[AI生成的产品描述：面料特性、遮光等级、适用场景]

INSTALLATION RECOMMENDATION
[AI根据安装方式生成建议]

CARE & MAINTENANCE
[AI生成维护指南]

PRICE
Total: $306.20

[Signature Area]
```

**技术实现**：

| 组件 | 说明 |
| :--- | :--- |
| 模型能力 | 产品知识库 + 文本生成 |
| 数据来源 | 产品手册、面料说明书、安装指南 |
| 输出格式 | HTML/PDF，支持品牌定制 |

**优先级**：P2（V2.0）

### 3.5 AI客户支持（Phase 4）

**功能描述**：

系统内置AI客服，Dealer可以直接提问并获得即时回复。当AI无法回答时，自动转人工并携带对话上下文。

**使用场景**：

- Dealer遇到操作问题
- Dealer询问产品信息
- Dealer询问订单状态

**支持范围**：

| 类型 | 示例问题 |
| :--- | :--- |
| 产品知识 | What is the difference between YB180002 and YB180003? |
| 操作指导 | How do I create a quote for a new customer? |
| 订单查询 | What is the status of order ORD-20260513-000123? |
| 政策询问 | What is your return policy? |
| 故障排查 | Why can't I select Motorized for this fabric? |

**技术实现**：

| 组件 | 说明 |
| :--- | :--- |
| 模型能力 | RAG（检索增强生成） |
| 数据来源 | 产品手册、帮助文档、FAQ、历史工单 |
| 转人工 | 携带对话上下文，无缝切换 |

**优先级**：P2（V2.5）

---

### 3.6 AI安装助手 + 视觉识别（Phase 5）

**功能描述**：

Dealer上传窗户照片，系统自动识别窗户类型、尺寸估算、安装方式建议。

**使用场景**：

- Dealer在客户家中现场测量
- 远程报价（无需上门测量）
- 减少测量错误

**交互流程**：

```
Dealer上传窗户照片
    ↓
AI识别：窗户类型（Casement）、估算尺寸（约35" × 55"）
    ↓
AI推荐：Mount Position = Inside Mount
    ↓
Dealer确认/修改尺寸
    ↓
生成报价
```

**技术实现**：

| 组件 | 说明 |
| :--- | :--- |
| 模型能力 | 计算机视觉（物体检测/分割/尺寸估算） |
| 数据来源 | 标注的窗户图片数据集 |
| 输出格式 | 窗户类型、尺寸估算、安装建议 |

**优先级**：P3（V3.0）

---

### 3.7 AI ERP摘要（Phase 4）

**功能描述**：

工厂管理员收到ERP同步的海量生产数据，AI自动生成人类可读的生产摘要和异常预警。

**使用场景**：

- 工厂管理员查看生产状态
- 识别生产异常
- 快速决策

**示例输出**：

```
生产摘要（2026-05-13）

📊 今日生产：
- 完成订单：47单
- 生产中：23单
- 延迟：3单（Delayed: ORD-001, ORD-002, ORD-003）

⚠️ 异常预警：
- 面料YB180002库存低于安全线（剩余8卷）
- 电机供应商发货延迟，预计影响12个订单

💡 建议：
- 建议今日补订YB180002面料
- 建议通知受影响的Dealer
```

**技术实现**：

| 组件 | 说明 |
| :--- | :--- |
| 模型能力 | 数据聚合 + 文本生成 + 异常检测 |
| 数据来源 | ERP生产数据、库存数据、供应链数据 |
| 输出格式 | 结构化摘要 + 预警 + 建议 |

**优先级**：P2（V2.5）

### 3.8 AI动态定价（Phase 5）

**功能描述**：

基于市场需求、库存水平、Dealer历史行为，AI辅助工厂制定动态价格策略。

**使用场景**：

- 工厂需要调整价格
- 清库存促销
- 针对特定Dealer的个性化定价

**能力范围**：

| 能力 | 说明 |
| :--- | :--- |
| 需求预测 | 预测未来30天各产品的需求量 |
| 库存敏感定价 | 库存低于安全线时建议涨价 |
| Dealer敏感定价 | 高价值Dealer可申请特殊折扣 |
| 促销建议 | 识别滞销产品，建议促销 |

**重要原则**：

> 动态定价仅提供建议，最终审批权在工厂。AI不自动修改Price Grid。

**技术实现**：

| 组件 | 说明 |
| :--- | :--- |
| 模型能力 | 时间序列预测 + 强化学习 |
| 数据来源 | 历史订单、库存数据、销售数据 |
| 输出格式 | 价格建议 + 预期影响分析 |

**优先级**：P3（V3.0）

---

### 3.9 AI规则学习（Phase 5）

**功能描述**：

系统从历史报价和订单中学习，自动发现新的定价规则、约束规则、配置组合，辅助工厂优化产品管理。

**使用场景**：

- 工厂不知道哪些面料最受欢迎
- 工厂不知道哪些配置组合经常出问题
- 优化Price Grid

**学习输出示例**：

```
📊 AI发现：

1. 价格规则建议
   宽度48-54"、高度60-72"区间，过去30天有47个订单落在相邻区间。
   建议新增此区间，基础价$135.00（预估）

2. 约束规则建议
   面料YB180003 + Outside Mount 组合，退单率高达23%。
   建议增加约束：不支持此组合

3. 配置推荐
   85%的客户在购买Blackout面料时，同时选择了Fabric Wrap底轨。
   建议在AI推荐中强化此组合
```

**技术实现**：

| 组件 | 说明 |
| :--- | :--- |
| 模型能力 | 聚类分析 + 关联规则挖掘 |
| 数据来源 | 历史报价单、订单转化率、退单率 |
| 输出格式 | 规则建议 + 置信度 + 影响分析 |

**优先级**：P3（V3.0）

## 4. AI架构设计

### 4.1 AI Agent架构（零SQL侵入）

```
┌─────────────────────────────────────────────────────────────┐
│                      Dealer Portal UI                        │
├─────────────────────────────────────────────────────────────┤
│                    AI Agent Layer                            │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐           │
│  │报价Agent │ │配置Agent│ │约束Agent│ │客服Agent│           │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘           │
├─────────────────────────────────────────────────────────────┤
│                    Tool Calling Layer                        │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  Function Tools (业务API)                            │    │
│  │  - search_products()                                 │    │
│  │  - get_price_grid()                                  │    │
│  │  - check_constraint()                                │    │
│  │  - get_order_status()                                │    │
│  │  - search_faq()                                      │    │
│  └─────────────────────────────────────────────────────┘    │
├─────────────────────────────────────────────────────────────┤
│                    Core System                              │
│         Product / Price / Order / User / Payment            │
└─────────────────────────────────────────────────────────────┘
```

**关键设计原则**：

| 原则 | 说明 |
| :--- | :--- |
| 零SQL侵入 | AI不直接访问数据库，通过Tool Calling调用业务API |
| 可控性 | AI输出可追溯、可审核、可回滚 |
| 可扩展 | 新增AI能力只需增加Agent，不影响核心系统 |

### 4.2 模型与部署策略

| 能力 | 推荐模型 | 部署方式 |
| :--- | :--- | :--- |
| 智能报价/配置推荐 | GPT-4o / Claude 3.5 | API调用 |
| 约束解释 | GPT-4o-mini | API调用 |
| Proposal生成 | GPT-4o | API调用 |
| 客户支持（RAG） | GPT-4o + 向量数据库 | API调用 |

## 5. AI Native SaaS演进方向

本系统的长期目标是成为窗帘行业**第一个AI Native CPQ平台**。

**演进路径**：

```
V1.0 (当前)
基础CPQ + Dealer Portal
    ↓
V1.5 - V2.0
AI增强：报价、配置、约束、Proposal
    ↓
V2.5 - V3.0
AI自动化：客服、定价、规则学习
    ↓
V3.0+
AI自治：系统主动优化自己
```

**行业壁垒**：

当AI能力完整上线后，本系统不再是“一个订单系统”，而是：

- 沉淀了窗帘行业产品知识的**知识引擎**
- 自动优化定价和配置的**决策引擎**
- 帮助Dealer降低门槛的**智能助手**

这些能力是通用SaaS平台无法复制的行业深度。
