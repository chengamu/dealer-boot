# 商家中心操作手册

## 功能定位

商家中心用于商家租户登录后维护自己的商家资料和员工账号。它解决的是“商家自己管理自己”的问题，不是平台跨商家审核和管理。

## 页面入口

| 功能 | 菜单 | 路由 |
| --- | --- | --- |
| 商家资料 | 商家中心 / 商家资料 | `/merchant/profile` |
| 商家用户 | 商家中心 / 商家用户 | `/merchant/users` |

## 商家资料

商家资料页面分为两类字段：

| 字段类型 | 当前行为 |
| --- | --- |
| 锁定字段 | 商家名称、公司名称、主邮箱、国家或地区不可在商家中心修改。 |
| 可编辑字段 | 联系人名、联系人姓、办公电话、手机、州/省、城市、地址、邮编、备注。 |

保存后系统根据联系人名和联系人姓生成联系人姓名。

## 商家用户

商家用户用于维护本商家租户内的员工账号。

常用操作：

1. 查询员工：按登录账号、手机号、状态、创建时间查询。
2. 新增员工：填写昵称、登录账号、初始密码、手机号、邮箱、性别、状态和备注。
3. 编辑员工：维护基础资料，登录账号唯一。
4. 启停员工：控制员工是否可继续登录。
5. 删除员工：不能删除当前登录用户，不能删除店主账号。

新增员工会自动进入当前商家租户，由后端分配系统默认部门和“营业员”角色；页面不展示部门或角色选择。

## 使用前准备

- 商家必须先通过商家管理里的申请审核。
- 审核通过后系统会创建商家租户、商家资料、系统默认部门，以及“店主”“营业员”两个默认角色。
- 注册申请人自动成为店主，店主登录后才能维护商家用户。

## 常见问题

| 问题 | 原因 | 处理方式 |
| --- | --- | --- |
| 商家名称不能改 | 商家主体名称来自申请审核结果 | 由平台在商家管理中确认是否调整可编辑资料。 |
| 新增员工没有商家选择 | 商家中心只维护当前租户员工 | 平台跨商家维护请到商家管理 / 商家员工。 |
| 不能删除店主 | 店主是商家租户核心账号 | 如需停用，先确认是否有其他可用店主。 |

## 如何确认维护成功

- 商家资料保存后刷新页面仍显示最新联系方式和地址。
- 新增员工后能在商家用户列表查到。
- 停用员工后，该员工不应继续正常登录。

## AI 代办能力

### 支持的意图

```text
意图名称：更新当前商家联系资料
用户说法：把我的办公电话改成 020-12345678，联系人改成 Sarah Johnson。
```

```text
意图名称：新增商家员工
用户说法：给门店新增一个员工账号，邮箱是 staff@example.com，昵称 Sarah。
```

### 必填槽位

更新商家资料：

- 至少一个可编辑字段。

新增商家员工：

- 登录账号。
- 昵称。
- 初始密码。

### 需要追问

- 用户要求修改锁定字段，例如商家名称、主邮箱、国家或地区。
- 用户要求删除、停用或重置密码时，必须保存前确认。
- 新增员工缺少登录账号、昵称或初始密码。

### 保存前确认摘要

```text
将更新当前商家资料：
- 联系人：Sarah Johnson
- 办公电话：020-12345678
是否确认保存？
```

```text
将新增商家员工：
- 登录账号：staff@example.com
- 昵称：Sarah
- 状态：正常
是否确认保存？
```
