# 商家体系 AI 代办能力规范

本文定义商家中心和商家管理接入 AI 助手时的共同规则。具体页面、接口、权限、槽位和失败提示以各模块 `接口与数据契约.md` 为准。

AI 阅读顺序：

1. [商家体系文档入口](README.md)
2. 本文
3. 对应模块的 `操作手册.md`
4. 对应模块的 `接口与数据契约.md`
5. 对应模块的 `体验问题清单.md`

AI 代办不是绕过页面直接写库，而是把自然语言转换为可确认、可校验、可审计的业务操作。

## 基本原则

- AI 只能调用已有接口，不直接写数据库。
- AI 不能暴露验证码、临时密码、密码哈希、Token、内部租户 ID、内部用户 ID。
- AI 不能让用户填写内部 ID、JSON、技术字段。
- AI 必须先补齐必填信息，再提交。
- AI 对审核、删除、停用、重置密码等高风险动作必须二次确认。
- AI 返回给用户的是业务摘要，不是接口 payload。
- AI 失败时必须说明业务原因和下一步动作。

## 账号和租户边界

商家体系有两类上下文：

| 上下文 | 说明 | 典型功能 |
| --- | --- | --- |
| 商家租户 | 当前商家维护自己的资料和员工 | 商家中心 / 商家资料、商家中心 / 商家用户 |
| 平台租户 | 平台/厂家跨商家管理申请、资料和员工 | 商家管理 / 商家申请、商家管理 / 商家资料、商家管理 / 商家员工 |

AI 必须按当前账号权限判断能做什么：

- 商家租户不能审核商家申请。
- 商家租户不能跨商家查询或维护员工。
- 平台租户维护商家员工时，必须先定位唯一目标商家。
- 平台租户审核申请时，必须确认申请仍是 `PENDING`。

## 操作风险分级

| 风险级别 | 动作 | AI 行为 |
| --- | --- | --- |
| 低风险 | 查询列表、查看详情 | 可直接执行并总结结果。 |
| 中风险 | 更新联系资料、地址、备注、新增员工 | 必须展示保存前摘要并确认。 |
| 高风险 | 批准申请、拒绝申请、停用账号、删除账号、重置密码 | 必须二次确认；拒绝申请必须有拒绝原因。 |
| 禁止自动执行 | 绕过权限、直接写库、暴露密码或验证码、自动批准/拒绝申请 | 直接拒绝并说明原因。 |

## 意图定义

每个支持 AI 代办的功能，需要在对应文档里定义意图。

```text
意图名称：
用户说法：
适用页面：
调用接口：
权限要求：
风险级别：
```

示例：

```text
意图名称：查询待审核商家申请
用户说法：帮我看一下还有哪些商家申请没审核。
适用页面：商家管理 / 商家申请
调用接口：GET /system/tenant/applications?status=PENDING
权限要求：system:tenant:application:list
风险级别：低风险
```

```text
意图名称：批准商家申请
用户说法：批准 ABC Dealer 的申请。
适用页面：商家管理 / 商家申请
调用接口：POST /system/tenant/applications/{id}/approve
权限要求：system:tenant:application:approve
风险级别：高风险
```

## 槽位定义

槽位分为四类：

- 必填槽位：缺失时必须追问。
- 可选槽位：用户提供就保存，未提供用默认值或留空。
- 可推断槽位：可根据当前账号、页面上下文或查询结果推断。
- 禁止槽位：用户不应填写，也不应暴露给用户。

商家体系常见槽位：

```text
商家申请审核：
- 必填：申请记录、审核动作
- 拒绝必填：拒绝原因
- 可推断：当前审核人、审核时间
- 禁止：验证码、临时密码、内部租户 ID

商家资料维护：
- 必填：至少一个可编辑字段
- 可选：联系人、电话、地址、备注
- 禁止：商家名称、主邮箱、审核状态、租户 ID

商家员工维护：
- 新增必填：登录账号、昵称、初始密码
- 平台侧必填：目标商家
- 可推断：商家租户、默认部门、默认员工角色
- 禁止：密码哈希、内部用户 ID
```

## 保存前确认

新增、编辑、审核、拒绝、删除、停用、重置密码必须保存前确认。

确认内容必须是业务摘要：

```text
将批准商家申请：
- 商家名称：ABC Dealer
- 公司名称：ABC Window LLC
- 登录邮箱：owner@example.com
- 国家或地区：United States
批准后会创建商家租户、商家资料和管理员账号。
是否确认？
```

```text
将新增商家员工：
- 目标商家：ABC Dealer
- 登录账号：staff@example.com
- 昵称：Sarah
- 状态：正常
是否确认保存？
```

## 错误处理

错误提示要从技术错误翻译成业务动作。

| 错误场景 | 用户提示 |
| --- | --- |
| 申请不是待审核 | 只有待审核申请可以批准或拒绝。 |
| 邮箱已被使用 | 该邮箱已有账号或有效申请，不能重复批准。 |
| 非平台租户审核 | 当前账号不能审核商家申请。 |
| 平台新增员工未选择商家 | 请先选择目标商家。 |
| 修改锁定字段 | 商家名称、主邮箱、国家或审核状态不能在当前入口修改。 |
| 删除店主 | 店主账号不能删除。 |

## 禁止事项

- 不允许 AI 自动批准或拒绝商家申请。
- 不允许 AI 自动删除、停用、重置密码。
- 不允许 AI 展示验证码、临时密码、密码哈希、Token。
- 不允许 AI 要求用户提供内部租户 ID、内部用户 ID。
- 不允许 AI 绕过平台租户和商家租户权限边界。
- 不允许 AI 以“我已经保存”为结论，除非接口真实返回成功。
