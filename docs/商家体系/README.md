## 模块边界

### 商家中心

商家中心是商家租户登录后维护自身资料和内部员工的入口。

当前包含：

- 商家资料：当前商家查看和维护可编辑联系资料。
- 商家用户：当前商家维护本租户员工账号。

不包含：

- 平台审核商家申请。
- 平台跨商家维护资料。
- 经销商下单门户 demo 页面。

### 商家管理

商家管理是平台/厂家租户管理商家主体、申请和商家员工的入口。

当前包含：

- 商家申请审核。
- 商家资料管理。
- 商家员工管理。

不包含：

- 商家自己的资料维护入口。
- 产品配置、订单、报价、支付流水。

## 文档更新规则

- 正确用法、审核顺序、维护边界变化写入 `操作手册.md`。
- 路由、接口、权限、字段、状态、AI 槽位变化写入 `接口与数据契约.md`。
- 页面缺陷、体验问题、待确认设计问题写入 `体验问题清单.md`。
- 修复并确认的问题从体验问题清单移除，不保留过程记录。

## AI 代办边界

商家体系涉及账号、租户、审核和敏感操作。共同规则见 [AI 代办能力规范](AI代办能力规范.md)。后续接入 AI 代办时必须遵守：

- 不自动批准或拒绝商家申请，除非用户明确确认。
- 不自动重置密码、删除账号、停用账号。
- 不暴露临时密码、验证码、内部租户 ID、内部用户 ID。
- 保存前必须展示业务摘要并等待用户确认。
- 平台侧操作和商家侧操作必须按当前账号权限分开。
