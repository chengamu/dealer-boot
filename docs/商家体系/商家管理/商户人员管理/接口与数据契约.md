# 商家人员管理接口与数据契约

## 页面与权限

| 项目 | 内容 |
| --- | --- |
| 菜单 | 商家管理 / 商家人员管理 |
| 前端路由 | `/merchantManagement/merchantStaff` |
| 前端组件 | `admin-ui/src/pages/merchant/PlatformMerchantUserPage.vue` |
| 权限码 | `merchant:user:*` |

## 后端接口

商家人员管理复用 `/merchant/user` 接口。平台侧调用必须传目标 `tenantId`。

| 动作 | 方法 | 路径 | 权限 |
| --- | --- | --- | --- |
| 列表 | GET | `/merchant/user/list` | `merchant:user:list` |
| 详情 / 初始化表单 | GET | `/merchant/user/`、`/merchant/user/{userId}` | `merchant:user:query` |
| 新增 | POST | `/merchant/user` | `merchant:user:add` |
| 修改 | PUT | `/merchant/user` | `merchant:user:edit` |
| 删除 | DELETE | `/merchant/user/{userIds}` | `merchant:user:remove` |
| 重置密码 | PUT | `/merchant/user/resetPwd` | `merchant:user:resetPwd` |
| 启停 | PUT | `/merchant/user/changeStatus` | `merchant:user:edit` |

## 选择器来源

| 页面字段 | 来源 |
| --- | --- |
| 目标商家 | `/system/merchant/profile/list` |

## 关键规则

- 平台租户调用时必须传目标商家 `tenantId`。
- 后端校验目标租户必须是 `MERCHANT`。
- 新增员工由后端强制写入目标商家租户、系统默认部门和营业员 `merchant_employee` 角色，忽略客户端提交的部门和角色值。
- 详情 / 初始化表单不返回部门、角色 options 或角色关联字段。
- 不能删除当前登录用户。
- 不能删除平台管理员或店主角色账号。

## 可检索性与引用关系

业务编码或单号：

- 当前代码使用 `userId` 和 `userName` 作为账号入口，未提供独立商家人员编号。

搜索展示字段：

- `userName`
- `nickName`
- `email`
- `phonenumber`
- `status`
- 目标商家名称；内部接口调用携带目标商家 `tenantId`

主要引用关系：

- 商家人员通过 `tenantId` 归属目标商家租户。
- 账号通过 `sys_user_role` 关联营业员角色。
- 平台模式列表依赖 `/system/merchant/profile/list` 选择目标商家。

权限过滤依据：

- `tenantId`
- `merchant:user:list` / `merchant:user:query`
- 平台模式目标商家 `tenantId`
- `status`

详情入口：

| 入口 | 内容 |
| --- | --- |
| 前端路由 | `/merchantManagement/merchantStaff` |
| 后端详情接口 | `GET /merchant/user/{userId}` |

说明：搜索结果只作为入口和摘要展示；账号状态、角色关系、目标租户和可操作范围以业务数据库和详情接口为准。

## AI 工具调用契约

### 意图

```text
意图名称：平台新增商家员工
调用接口：POST /merchant/user
权限要求：merchant:user:add
风险级别：中风险
```

### Payload 边界

- AI 必须先定位唯一目标商家。
- AI 不向用户暴露内部租户 ID 和内部用户 ID。
- 删除、停用、重置密码必须二次确认。

### 失败提示

| 场景 | 用户提示 |
| --- | --- |
| 未选择目标商家 | 请先选择目标商家。 |
| 目标租户不是商家 | 请选择有效商家租户。 |
| 删除店主 | 店主账号不能删除。 |
