# 等级折扣接口与数据契约

## 功能定位

等级折扣用于配置“商户等级 + 产品分类 + 产品类型”的产品折扣。折扣优先级为：等级产品折扣 > 等级默认折扣 > `1.00`。

## 接口

| 动作 | 方法 | 路径 | 权限 |
| --- | --- | --- | --- |
| 分页 | GET | `/merchant/level-discounts/list` | `merchant:levelDiscount:list` |
| 导出 | POST | `/merchant/level-discounts/export` | `merchant:levelDiscount:export` |
| 详情 | GET | `/merchant/level-discounts/{id}` | `merchant:levelDiscount:query` |
| 新增 | POST | `/merchant/level-discounts` | `merchant:levelDiscount:add` |
| 修改 | PUT | `/merchant/level-discounts` | `merchant:levelDiscount:edit` |
| 删除 | DELETE | `/merchant/level-discounts/{ids}` | `merchant:levelDiscount:remove` |
| 启停 | PUT | `/merchant/level-discounts/change-status/{id}/{status}` | `merchant:levelDiscount:edit` |

## 数据表

| 表 | 说明 |
| --- | --- |
| `merchant_level_discount` | 等级产品折扣规则。 |
| `merchant_level` | 折扣所属等级。 |
| `pc_product_category` | 产品分类来源。 |
| `pc_product_dict_item` | 产品类型来源，字典类型为 `product_type`。 |

## 规则

- 只有平台/厂家租户可维护。
- 同一等级、产品分类、产品类型只能有一条规则，由 Service 控制唯一。
- 折扣必须在 `0` 到 `1` 之间。
- 产品分类必须存在且启用。
- 产品类型必须来自启用产品字典，不写死枚举。
- 启用状态不可编辑、不可删除，可停用。
