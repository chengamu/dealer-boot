# 商户等级接口与数据契约

## 功能定位

商户等级由平台/厂家维护，用于统一管理普通、VIP、大客户等商家层级，并为商家资料带出默认折扣和默认信用额度。

## 接口

| 动作 | 方法 | 路径 | 权限 |
| --- | --- | --- | --- |
| 分页 | GET | `/merchant/levels/list` | `merchant:level:list` |
| 选项 | GET | `/merchant/levels/options` | `merchant:level:list` |
| 导出 | POST | `/merchant/levels/export` | `merchant:level:export` |
| 详情 | GET | `/merchant/levels/{id}` | `merchant:level:query` |
| 新增 | POST | `/merchant/levels` | `merchant:level:add` |
| 修改 | PUT | `/merchant/levels` | `merchant:level:edit` |
| 删除 | DELETE | `/merchant/levels/{ids}` | `merchant:level:remove` |
| 启停 | PUT | `/merchant/levels/change-status/{id}/{status}` | `merchant:level:edit` |

## 数据表

| 表 | 说明 |
| --- | --- |
| `merchant_level` | 商户等级主表，平台租户数据。 |
| `merchant_profile` | 引用等级并冗余等级编码、名称、折扣、信用额度。 |

## 关键字段

| 字段 | 说明 |
| --- | --- |
| `level_code` | 等级编码，Service 控制唯一。 |
| `level_name` | 等级名称。 |
| `default_discount_rate` | 等级默认折扣，等级产品折扣不存在时使用。 |
| `default_credit_limit` | 默认信用额度，本轮只做配置，不接 pay。 |
| `default_flag` | 默认等级，审核通过创建商家资料时带入。 |
| `status` | `ENABLED` / `DISABLED`。 |

## 规则

- 只有平台/厂家租户可维护。
- 编码唯一由 Service 控制，不依赖数据库唯一约束。
- 默认折扣必须在 `0` 到 `1` 之间。
- 默认信用额度不能小于 `0`。
- 启用状态不可编辑、不可删除，可停用。
- 已被商家资料或等级折扣引用时不可删除。
- 同一时间只保留一个默认等级。
