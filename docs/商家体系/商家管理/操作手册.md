# 商家管理操作手册

## 功能定位

商家管理是平台/厂家租户管理商家生命周期的入口。它解决商家申请审核、商家资料查看维护、商家员工跨租户维护的问题。

## 商家申请审核

商家申请来源于公开申请页。平台侧可以查询、查看详情、批准和拒绝申请。

状态：

| 状态 | 含义 | 可执行动作 |
| --- | --- | --- |
| `PENDING` | 待审核 | 批准、拒绝 |
| `APPROVED` | 已通过 | 只读 |
| `REJECTED` | 已拒绝 | 只读 |

批准申请后，系统会创建：

- `sys_tenant` 商家租户。
- `merchant_profile` 商家资料。
- 店主用户。
- 系统默认部门，以及店主、营业员两个默认角色和菜单权限。

拒绝申请时需要填写拒绝原因。

## 商家资料管理

平台侧可查询所有商家资料，查看详情，并维护可编辑联系资料和地址资料。

平台侧当前可编辑字段包含：

- 联系人名、联系人姓。
- 办公电话、手机。
- 州/省、城市、地址、邮编。
- 商家等级、当前折扣、当前信用额度。
- 备注。

商家名称、公司名称、主邮箱、国家、审核状态不是当前普通编辑字段。

## 商家等级和等级折扣

商家等级用于统一维护普通、VIP、大客户等层级。等级折扣用于配置“等级 + 产品分类 + 产品类型”的产品折扣。

等级和等级折扣新增后默认停用，状态不在编辑表单中直接改，由行内启用/停用按钮单独控制。折扣统一按两位小数展示。

折扣优先级：

1. 等级产品折扣。
2. 等级默认折扣。
3. `1.00`。

## 商家员工管理

平台侧商家员工管理复用 `/merchant/user` 后端接口，前端使用 `admin-ui/src/pages/merchant/PlatformMerchantUserPage.vue`，并开启平台模式。

平台模式下：

1. 先选择商家。
2. 再查询、新增、编辑、删除或启停该商家员工。
3. 新增员工会写入所选商家的租户。

如果没有选择商家，新增按钮不可用。
部门和角色由后端自动分配，页面不向平台操作员或商家展示选择入口。

## 常见问题

| 问题 | 原因 | 处理方式 |
| --- | --- | --- |
| 为什么申请只能审核一次 | 审批接口只允许处理 `PENDING` 申请 | 已通过或已拒绝申请只能查看。 |
| 批准后为什么生成账号 | 商家需要店主账号进入商家中心 | 系统会生成初始密码，并在邮件启用时发送。 |
| 平台新增员工为什么要选商家 | 平台是跨租户维护，必须明确目标租户 | 先在商家下拉中选择目标商家。 |
| 为什么不能删除店主 | 店主是商家租户的核心账号 | 先确认替代店主和权限迁移方案。 |

## 如何确认维护成功

- 批准申请后，申请状态变为 `APPROVED`，并能在商家资料列表查到商家。
- 拒绝申请后，申请状态变为 `REJECTED`，详情中能看到拒绝原因。
- 平台编辑商家资料后，商家中心刷新可看到对应联系资料变化。
- 平台新增商家员工后，在所选商家的员工列表中能查到。

## 本地演示数据

初始化 SQL 会写入一个本地演示商家租户：

- 商家：Demo Merchant。
- 租户：`300001`，类型为 `MERCHANT`。
- 用户：`demo_merchant`，用于验证商家中心资料和商家用户入口。

该数据只用于本地开发验证，正式数据仍以商家申请审核或后续导入为准。

## 导出策略

商家等级和等级折扣当前保留导出按钮，因为后端导出接口和权限已齐备。客户资料、全部客户等其他 grid 是否统一补导出，需要按全局 grid 导出策略单独评估后再实施。

### 需要追问

- 批准或拒绝申请前，必须确认目标申请。
- 拒绝申请时缺少拒绝原因必须追问。
- 平台新增商家员工时，必须确认目标商家。
- 删除、停用、重置密码必须二次确认。

```text

- 登录邮箱：owner@example.com
批准后会创建商家租户、商家资料和管理员账号。
是否确认？
```
