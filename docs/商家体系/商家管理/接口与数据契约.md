# 商家管理接口与数据契约

## 页面与权限

| 功能 | 前端路由 | 前端组件 | 权限码 |
| --- | --- | --- | --- |
| 商家申请审核 | `/merchantManagement/tenantApplication` | `admin-ui/src/pages/system/TenantApplicationsPlaceholder.vue` | `system:tenant:application:*` |
| 商家资料管理 | `/merchantManagement/merchantProfile` | `admin-ui/src/pages/system/MerchantProfilePage.vue` | `system:merchant:profile:*` |
| 商家员工管理 | `/merchantManagement/merchantStaff` | `admin-ui/src/pages/merchant/PlatformMerchantUserPage.vue` | `merchant:user:*` |

兼容路由：

- `/system/tenant/applications`
- `/system/merchant/profile`
- `/merchant/users`

## 后端接口

### 公开申请入口

| 动作 | 方法 | 路径 | 权限 |
| --- | --- | --- | --- |
| 发送邮箱验证码 | POST | `/merchant/applications/email-code` | 不需要登录 |
| 提交商家申请 | POST | `/merchant/applications` | 不需要登录 |

### 商家申请审核

| 动作 | 方法 | 路径 | 权限 |
| --- | --- | --- | --- |
| 列表 | GET | `/system/tenant/applications` | `system:tenant:application:list` |
| 详情 | GET | `/system/tenant/applications/{id}` | `system:tenant:application:query` |
| 批准 | POST | `/system/tenant/applications/{id}/approve` | `system:tenant:application:approve` |
| 拒绝 | POST | `/system/tenant/applications/{id}/reject` | `system:tenant:application:reject` |

### 商家资料管理

| 动作 | 方法 | 路径 | 权限 |
| --- | --- | --- | --- |
| 列表 | GET | `/system/merchant/profile/list` | `system:merchant:profile:list` |
| 详情 | GET | `/system/merchant/profile/{merchantId}` | `system:merchant:profile:query` |
| 修改 | PUT | `/system/merchant/profile` | `system:merchant:profile:edit` |

### 商家员工管理

商家员工管理复用 `/merchant/user` 接口。平台侧调用时必须传目标 `tenantId`。

| 动作 | 方法 | 路径 | 权限 |
| --- | --- | --- | --- |
| 列表 | GET | `/merchant/user/list` | `merchant:user:list` |
| 详情 / 初始化表单 | GET | `/merchant/user/`、`/merchant/user/{userId}` | `merchant:user:query` |
| 新增 | POST | `/merchant/user` | `merchant:user:add` |
| 修改 | PUT | `/merchant/user` | `merchant:user:edit` |
| 删除 | DELETE | `/merchant/user/{userIds}` | `merchant:user:remove` |
| 重置密码 | PUT | `/merchant/user/resetPwd` | `merchant:user:resetPwd` |
| 启停 | PUT | `/merchant/user/changeStatus` | `merchant:user:edit` |

## 数据表

| 表名 | 用途 |
| --- | --- |
| `sys_tenant_apply` | 商家申请记录。 |
| `sys_tenant` | 商家租户，审核通过后创建。 |
| `merchant_profile` | 商家主体资料，审核通过后创建。 |
| `sys_user` | 商家管理员和员工账号。 |
| `sys_dept` | 商家默认部门。 |
| `sys_role`、`sys_role_menu`、`sys_user_role` | 商家默认角色和菜单权限。 |

## 状态值

| 对象 | 字段 | 取值 |
| --- | --- | --- |
| 商家申请 | `status` | `PENDING` 待审核，`APPROVED` 已通过，`REJECTED` 已拒绝 |
| 商家资料 | `status` | `1` 正常，`0` 停用 |
| 商家资料 | `audit_status` | `PENDING`、`APPROVED`、`REJECTED` |
| 商家员工 | `status` | `1` 正常，`0` 停用 |

## 关键规则

- 商家申请公开提交前必须同意条款。
- 公开提交需要邮箱验证码；验证码有冷却、过期、每日次数和错误次数限制。
- 同一邮箱已有用户，或已有 `PENDING` / `APPROVED` 申请时，不能再次提交。
- 申请批准和拒绝只允许平台租户操作。
- 申请批准只允许处理 `PENDING` 状态。
- 批准申请会在事务内创建商家租户、默认部门、默认角色、商家管理员和商家资料。
- 商家资料列表和详情是平台跨租户查询。
- 商家员工平台模式必须传目标商家 `tenantId`，后端会校验目标租户必须是 `MERCHANT`。
- 商家员工删除不能删除当前登录用户，不能删除平台管理员或商家管理员角色账号。

## AI 工具调用契约

### 意图

```text
意图名称：查询待审核商家申请
调用接口：GET /system/tenant/applications?status=PENDING
权限要求：system:tenant:application:list
```

```text
意图名称：批准商家申请
调用接口：POST /system/tenant/applications/{id}/approve
权限要求：system:tenant:application:approve
```

```text
意图名称：拒绝商家申请
调用接口：POST /system/tenant/applications/{id}/reject
权限要求：system:tenant:application:reject
```

```text
意图名称：平台新增商家员工
调用接口：POST /merchant/user
权限要求：merchant:user:add
```

### Payload 边界

- AI 不提交或暴露验证码、临时密码、内部租户 ID、内部用户 ID，除非接口调用必须且来自系统查询结果。
- AI 不自动批准、拒绝、删除、停用、重置密码。
- 审核动作必须展示申请摘要并等待用户确认。
- 拒绝申请必须有拒绝原因。
- 平台新增商家员工必须先定位唯一商家。

### 保存前确认格式

```text
将批准商家申请：
- 商家名称：ABC Dealer
- 公司名称：ABC Window LLC
- 登录邮箱：owner@example.com
- 国家或地区：United States
批准后会创建商家租户、商家资料和管理员账号。
是否确认？
```

### 失败提示

| 场景 | 用户提示 |
| --- | --- |
| 非平台租户操作审核 | 当前账号不能审核商家申请。 |
| 申请不存在 | 未找到该商家申请。 |
| 申请不是待审核 | 只有待审核申请可以批准或拒绝。 |
| 邮箱已被使用 | 该邮箱已有账号或有效申请，不能重复批准。 |
| 平台新增员工未选择商家 | 请先选择目标商家。 |
| 目标租户不是商家 | 请选择有效商家租户。 |
| 删除商家管理员 | 商家管理员账号不能删除。 |
