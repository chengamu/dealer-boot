# 商家审核接口与数据契约

## 页面与权限

| 项目 | 内容 |
| --- | --- |
| 菜单 | 商家管理 / 商家审核 |
| 前端路由 | `/merchantManagement/tenantApplication` |
| 前端组件 | `admin-ui/src/pages/system/TenantApplicationsPlaceholder.vue` |
| 权限码 | `system:tenant:application:*` |

## 后端接口

| 动作 | 方法 | 路径 | 权限 |
| --- | --- | --- | --- |
| 发送邮箱验证码 | POST | `/merchant/applications/email-code` | 不需要登录 |
| 提交商家申请 | POST | `/merchant/applications` | 不需要登录 |
| 审核列表 | GET | `/system/tenant/applications` | `system:tenant:application:list` |
| 详情 | GET | `/system/tenant/applications/{id}` | `system:tenant:application:query` |
| 批准 | POST | `/system/tenant/applications/{id}/approve` | `system:tenant:application:approve` |
| 拒绝 | POST | `/system/tenant/applications/{id}/reject` | `system:tenant:application:reject` |

## 数据表

| 表名 | 用途 |
| --- | --- |
| `sys_tenant_apply` | 商家申请记录。 |
| `sys_tenant` | 审核通过后创建商家租户。 |
| `merchant_profile` | 审核通过后创建商家资料。 |
| `sys_user` | 审核通过后创建店主账号。 |
| `sys_dept` | 审核通过后创建系统默认部门。 |
| `sys_role` / `sys_user_role` | 创建店主、营业员默认角色，并把申请人关联为店主。 |

## 关键规则

- 公开申请必须同意条款并通过邮箱验证码。
- 同一邮箱已有账号，或已有 `PENDING` / `APPROVED` 申请时，不能再次提交。
- 审核动作只允许平台租户执行。
- 只有 `PENDING` 申请可以批准或拒绝。
- 审核通过后只创建店主 `merchant_admin` 与营业员 `merchant_employee` 两个默认角色，数据范围分别为 `4` 与 `5`。

## 可检索性与引用关系

业务编码或单号：

- 当前代码使用 `applyId` 作为详情入口，未提供独立申请编号。

搜索展示字段：

- `merchantName`
- `companyName`
- `email`
- `status`
- `auditTime`
- `createTime`

主要引用关系：

- 审核通过后创建 `sys_tenant`、`merchant_profile`、系统默认部门和店主 `sys_user`。
- 申请状态驱动商家租户和商家资料初始化，不直接作为后续业务事实来源。

权限过滤依据：

- `system:tenant:application:list` / `system:tenant:application:query`
- `status`

详情入口：

| 入口 | 内容 |
| --- | --- |
| 前端路由 | `/merchantManagement/tenantApplication` |
| 后端详情接口 | `GET /system/tenant/applications/{id}` |

说明：搜索结果只作为入口和摘要展示；审核状态、拒绝原因和审核动作权限以业务数据库和详情接口为准。

## AI 工具调用契约

### 意图

```text
意图名称：查询待审核商家申请
调用接口：GET /system/tenant/applications?status=PENDING
权限要求：system:tenant:application:list
风险级别：低风险
```

```text
意图名称：批准商家申请
调用接口：POST /system/tenant/applications/{id}/approve
权限要求：system:tenant:application:approve
风险级别：高风险
```

### Payload 边界

- AI 不暴露验证码、临时密码、内部租户 ID。
- AI 不自动批准或拒绝申请。
- 拒绝申请必须有拒绝原因。

### 失败提示

| 场景 | 用户提示 |
| --- | --- |
| 非平台租户操作审核 | 当前账号不能审核商家申请。 |
| 申请不是待审核 | 只有待审核申请可以批准或拒绝。 |
| 邮箱已被使用 | 该邮箱已有账号或有效申请，不能重复批准。 |
