# 客户资料接口与数据契约

## 功能定位

客户资料是租户客户池。平台/厂家租户维护自己的客户；商家租户维护自己的客户。

## 接口

| 动作 | 方法 | 路径 | 权限 |
| --- | --- | --- | --- |
| 分页 | GET | `/customer/customers/list` | `customer:profile:list` |
| 选项 | GET | `/customer/customers/options` | `customer:profile:list` |
| 负责人选项 | GET | `/customer/customers/owner-options` | `customer:profile:list` |
| 详情 | GET | `/customer/customers/{id}` | `customer:profile:query` |
| 新增 | POST | `/customer/customers` | `customer:profile:add` |
| 修改 | PUT | `/customer/customers` | `customer:profile:edit` |
| 删除 | DELETE | `/customer/customers/{ids}` | `customer:profile:remove` |
| 启停 | PUT | `/customer/customers/change-status/{id}/{status}` | `customer:profile:edit` |

## 数据表

| 表 | 说明 |
| --- | --- |
| `customer_profile` | 租户客户资料。 |
| `sys_user` | 客户负责人来源。 |
| `merchant_profile` | 商家租户写入客户时冗余商家名称。 |

## 关键字段

| 字段 | 说明 |
| --- | --- |
| `tenant_id` | 客户所属租户。 |
| `merchant_id` / `merchant_name` | 商家租户客户冗余商家资料，平台自有客户为空。 |
| `customer_name` | 客户名称。 |
| `email` | 主要识别字段，必填，同租户唯一。 |
| `owner_user_id` / `owner_name` | 客户负责人，必须属于目标租户。 |
| `status` | `ENABLED` / `DISABLED`。 |

## 规则

- 普通入口只能查询、维护当前 `tenant_id` 的客户。
- 同一租户 email 唯一，跨租户可重复。
- 负责人选项由客户模块提供，只返回当前租户启用且未删除用户。
- 启用客户不可删除，可停用。
- 删除只允许停用客户，且删除检查和删除执行都必须限定当前 `tenant_id`。
