# 后台视觉与密度标准

本文是当前后台管理端视觉标准的唯一依据。旧的大卡片、大胶囊、大留白、默认组件拼装风格不再作为后续开发参考。

## 总方向

- 后台定位为紧凑型 ERP / 管理系统，不做营销页、SaaS 大留白或大卡片风格。
- 优先保证信息密度、扫描效率和稳定布局。
- 页面主背景保持浅灰，内容容器保持白色或接近白色。
- 不使用厚重阴影，不使用大圆角，不用强烈装饰性渐变。
- 全局视觉变量优先使用：
  - 主色：`#1677ff`
  - 页面背景：`#f3f6fa`
  - 内容背景：`#ffffff`
  - 主文字：`#1d2129`
  - 次级文字：`#667085`
  - 弱文字：`#98a2b3`
  - 分割线：`#eef0f5`
  - 卡片边框：`#e9edf5` 或 `#eef0f5`
  - hover 背景：`#f5f8ff` / `#f8fbff`
  - 激活背景：`#eef5ff`

## 页面密度

- 内容区 padding 控制在 `8px - 12px`。
- 搜索区、工具栏、表格和分页之间的垂直间距控制在 `6px - 10px`。
- 不出现 `16px` 以上的大段垂直留白，除非是页面外边距。
- 卡片圆角统一 `6px - 8px`。
- 所有边框使用浅边框，不加厚重阴影。

## 左侧导航

- 侧边栏宽度保持当前产品宽度，不为了视觉效果变宽。
- 背景使用 `#ffffff`，右侧边界使用 `#eef0f5`。
- Logo 区高度控制在 `48px - 52px`。
- 一级菜单：
  - 高度 `38px - 40px`
  - 字号 `14px`
  - 字重 `500`
  - 左 padding `12px`
  - 图标 `16px`
  - 图标和文字间距 `8px`
  - 圆角 `6px`
- 二级菜单：
  - 高度 `32px - 34px`
  - 字号 `13px`
  - 左 padding `28px - 30px`，不能超过 `32px`
  - 图标 `14px`
  - 图标和文字间距 `6px - 8px`
  - 圆角 `6px`
- 展开一级菜单只表示分组打开，使用更轻的浅蓝背景和 `#2563eb` / `#3b82f6`，字重保持 `500`。
- 当前二级菜单使用浅蓝背景、`#1677ff` 文字和图标、字重 `600`。
- 当前二级菜单不再增加额外小蓝点，避免“小蓝点 + 图标 + 蓝色文字”的视觉噪音。

## Header 与面包屑

- Header 高度控制在 `48px - 52px`。
- 图标尺寸保持 `16px - 18px`，所有元素垂直居中。
- Header 底部分割线使用 `#eef0f5`。
- 面包屑字号 `13px`。
- 面包屑非当前项使用 `#98a2b3`，当前项使用 `#667085` 或 `#344054`。
- 面包屑不能比 Tab 或页面主内容更抢眼。

## 顶部多页签 Tab

- 使用紧凑型后台 Tab 导航，不做大卡片、不做大胶囊。
- Tab 容器高度控制在 `34px - 36px`，只保留底部浅分割线。
- Tab item 高度 `30px - 32px`，左右 padding `10px - 12px`，间距 `4px - 6px`。
- 非激活 Tab 使用灰色文字和透明背景。
- hover 使用极浅灰蓝背景，文字变为主色。
- 当前 Tab 使用 `#1677ff` 文字、字重 `600`、底部 `2px` 蓝色下划线。
- 关闭按钮默认弱化，hover 时再明显；激活 Tab 可常显但颜色必须淡。
- Tab 多时横向滚动，不换行、不挤压、不撑坏内容区。

## 搜索区

- 搜索区保持紧凑，字段多时优先按业务重要度收缩字段宽度，而不是强行隐藏滚动条。
- 搜索区上下 padding `6px - 8px`，左右 padding `10px - 12px`。
- 输入框、下拉框、按钮高度统一约 `32px`。
- label 与输入框间距 `6px`。
- 筛选项之间水平间距约 `12px`。
- 查询按钮使用蓝色主按钮。
- 重置按钮使用普通次要按钮：
  - 边框 `#d9e0ea`
  - 文字 `#475467`
  - hover 时轻微加深。
- 业务检索字段由具体页面决定，不做全局随意删减。

## 工具栏按钮

- 工具栏高度 `38px - 40px`，按钮高度 `30px - 32px`，按钮间距 `8px`。
- 新增：蓝色主按钮。
- 修改：普通描边或蓝色弱化描边，不使用绿色实色。
- 删除：浅红描边或浅红文字，保留风险提示但不要过度抢眼。
- 超级修改、取消审核：橙色弱化描边。
- 引用检查：绿色或普通描边，但不能比主按钮更重。
- 变更记录：普通描边或蓝色弱化描边。
- 禁用按钮统一浅灰。

## 表格

- 普通资料维护页优先使用 `el-table`，复杂录入型表格才使用 `vxe-table`。
- 表头高度 `38px - 40px`，背景 `#f7f9fc`。
- 表格行高 `40px - 42px`。
- 单元格 padding 使用 `8px 12px` 或等价紧凑设置。
- 表格边框统一 `#eef0f5`。
- hover 行背景使用 `#f8fbff`。
- 排序图标颜色弱化，不能比表头文字更抢眼。
- 文本类字段左对齐。
- 数字、状态、开关、操作列居中或按业务需要右对齐。
- 操作列固定右侧，统一使用 `AdminTableActions` 或同一套行操作样式；禁止同一系统里同时出现胶囊按钮和裸 link 两套风格。
- 高风险行操作要弱化或收进“更多”，并保留确认提示。
- 表格整行强调使用标准 grid 的 `rowTone` 语义色，不在业务页面单独写行背景色。
- 行语义色保持轻量，不使用强底色：`danger` 淡红用于驳回、失败、严重异常；`warning` 淡橙用于待处理、未校验、风险提示；`success` 淡绿用于已通过、已启用、生效；`muted` 淡灰用于停用、作废、归档。

## 分页和空状态

- 分页使用项目现有 `pagination` 组件。
- 总条数、每页条数、页码、跳转区域保持统一布局。
- 分页区域不加厚重阴影。
- 空数据区域不要无限撑大，表格空状态最小高度控制在 `180px - 240px`。
- 空状态文案保持简洁，不添加大插画或大面积留白。

## 基础信息表格页定版基准

第一批基础信息表格页以当前以下页面作为视觉基准：

- 基础字典
- 单位管理
- 物料管理

后续同类资料维护页应对齐这三页的搜索区、工具栏、表格、分页和页面密度，不再回退到默认组件拼装风格。

## 禁止回退的旧方向

- 不做大胶囊 Tab。
- 不做大卡片式页面外壳。
- 不用强红删除按钮作为默认常态。
- 不把所有搜索字段拉成过宽输入框。
- 不通过隐藏滚动条掩盖宽度问题。
- 不保留两套表格操作按钮风格。
- 不在后台资料页使用大留白、大插画、大装饰背景。
