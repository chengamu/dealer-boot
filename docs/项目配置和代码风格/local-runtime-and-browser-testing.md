# 本地运行与浏览器验证

本文件只记录 Codex 做本地运行和浏览器验证时容易用错的环境信息，不作为完整启动教程。

## 后端运行产物

- 当前本地后端运行产物在 `bocoo-admin/target/dist/`。
- 该目录内包含 `bocoo-admin.jar` 和 `config/application-dev.yml`。
- `target/dist` 是构建产物和运行目录，不是源码目录；不要在这里修改业务代码或配置源文件。
- 需要验证后端状态时，先确认已有服务、端口和进程，再决定是否启动；不要默认重启服务、杀进程或覆盖用户正在运行的环境。

## 前端本地服务

- 管理端源码在 `admin-ui/`。
- 前端本地启动、端口和代理规则以 `admin-ui/README.md` 为准；当前 dev server 默认访问 `http://127.0.0.1:8083`。
- 浏览器访问后端 API 时，优先走前端代理或同源代理规则，不把容器内部服务名写到浏览器可见地址里。

## 浏览器验证

- 页面交互验证优先使用 Codex 内部浏览器 / in-app browser。
- 不默认调用外部 Chrome 或系统浏览器。
- 只有用户明确要求，或验证必须依赖已有登录态、插件、真实浏览器状态时，才使用外部浏览器。
- 回复验证结果时说明使用的浏览器环境、访问 URL、是否复用已有前后端服务；没有执行浏览器验证时直接说明原因。

## 不进入文档的内容

- 不把临时端口、一次性命令、完整日志、过程截图目录写进 `docs`。
- 不新增项目内执行态目录、临时任务队列或缓存式 workflow 目录。
- 本文件不替代业务操作手册、接口与数据契约、体验问题清单。
