# 数值精度测试矩阵

本矩阵是所有 P0/P1 模块新增或重构数值字段时的强制验收基线。测试必须同时覆盖前端编辑、API、后端校验、权威计算、快照和输出。

运行监控、图表坐标、分页、排序、ID 和其他不参与业务计算或快照的技术数值不纳入本矩阵，可按界面需要保留两位等展示精度。

## 公共组件

| 场景 | 输入 | 预期 |
| --- | --- | --- |
| 空值 | `null` | 输入为空，提交仍为 `null`，不得变成 `0` |
| 普通数量 | `0.325` | 保存 `"0.325"`，展示 `0.325` |
| 尾零 | `4.56000` | 草稿保留；规范值 `"4.56"`；展示 `4.56` |
| 有效高精度 | `4.41002` | 六位契约接受并原样展示 |
| 超精度 | 五位值进入四位单价字段 | 保留草稿、显示错误、禁止提交 |
| 非法字符 | `1a.2` | 保留草稿、显示错误、禁止提交 |
| 计件数量 | `1.5` | 拒绝，不得舍入为 `2` |
| 范围 | 小于 `min` 或大于 `max` | 拒绝，不得自动夹取边界 |

## 英寸

| 场景 | 输入 | 预期 |
| --- | --- | --- |
| 空值 | `null` | 整数和分数控件均为空 |
| 合法分数 | `72.125` | 显示 `72 1/8 in`，往返仍为 `"72.125"` |
| 整数 | `72` | 显示 `72 in` |
| 非法步长 | `2.123` | 显示原值和异常标记，禁止保存 |
| 批量粘贴 | `60.5` | 解析为 `60 1/2 in` |
| 不支持分母 | `1/3` | 拒绝配置，不得近似保存 |

## 业务类型

| 类型 | 必测边界 |
| --- | --- |
| `COUNT` | 零值策略、正整数、负数、小数 |
| `QUANTITY` | 单位精度 0/3/6、单位步长、负数和尾零 |
| `UNIT_PRICE` | `0.0156`、四位上限、进入公式前不舍入 |
| `MONEY` | USD 两位、零位币种、三位币种、负数冲销场景 |
| `RATIO` | `0.15` 表示 15%，范围通常为 `0..1` |
| `PERCENT` | `15` 表示 15%，范围按字段契约 |

## 纵向业务链路

1. 配方：尺寸、原料用量、损耗率、变量、模拟、审核版本快照一致。
2. 工程测算：Grid 输入、批量粘贴、逐行测算、整单合计、确认快照一致。
3. 快速下单：逐件配置、购物车、复核、转正式订单一致。
4. 销售订单：来源快照、列表、详情、PDF、Excel、邮件一致。
5. 支付：十进制金额精确转换为最小货币单位；页面、支付单和回调一致。
6. 履约：包裹重量和尺寸按单位精度；生产用量按物料单位精度展示。

## 计算与快照

- 人工输入超精度必须拒绝；公式计算允许保留中间精度并在明确边界舍入。
- `0.0156 × 1000` 的正式 USD 金额必须为 `15.60`。
- 已确认报价、销售订单、支付和生产快照不得因展示规则变化被重算或改写。
- 页面合计、后端合计、PDF、Excel、邮件和支付最小单位必须逐字段一致。

## 静态审计范围

静态检查只扫描业务数值目录，不全局禁止技术计算。重点检查：

- 业务页面直接使用 `el-input-number` 的 `precision`。
- 业务小数进入 API 前使用 `Number/parseFloat/toFixed`。
- 金额通过 `Math.round(amount * 100)` 转换支付最小单位。
- 尺寸 formatter 自动寻找最近分数。
- Java 权威计算使用 `double/float/doubleValue()`。
- `BigDecimal.setScale()` 未声明 `RoundingMode`。

分页、排序、ID、UI 坐标和公式 DSL 自身的受控 `round()` 不属于违规项，必须通过目录和字段语义白名单判断。
