# 代码生成器规范

## 1. 适用场景

适合先生成再局部增强：

- 基础资料。
- 字典式主数据。
- 普通列表。
- 导入批次。
- 行级问题。
- 同步日志。
- 发布包列表。
- 缺口待办。

不适合只靠生成器：

- 工作台。
- 配置编辑器。
- 价格矩阵。
- 价格试算。
- 发布检查。
- 发布包生成。
- 快照构建。
- 销售只读聚合。

## 2. 生成后必须检查

- 前端列表是否是标准 `app-container + 查询区 + toolbar + table + pagination`。
- 新增、编辑、详情是否使用右侧抽屉承载长表单。
- 短确认是否使用 `ElMessageBox.confirm`，没有把轻量确认做成长表单弹窗。
- 后端是否生成 `/export`，且前端按钮、后端注解、数据库按钮权限一致。
- `changeStatus` 是否使用真实主键字段。
- 菜单 SQL 是否包含 list/query/add/edit/remove/export 对应按钮。
- 新增文案是否补 i18n。
- 时间字段是否走 UTC 工具。

## 3. 生成器边界

- 生成器产物是起点，不是最终规范豁免。
- 生成后的页面必须按前端规范调整 drawer、权限、i18n、UTC 和浏览器验证。
- 生成后的后端必须按后端规范检查模块单一化、分层、权限注解、导出和状态变更。
- 如果生成结果和本目录规范冲突，以本目录规范为准。
