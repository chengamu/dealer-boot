# 销售订单接口与数据契约

## 模块定位

销售订单只承接已确认客户报价，不再承担草稿录入、产品配置、测算或报价职责。一份确认报价只能生成一个销售订单。

- `dealer_sales_document.source_quote_id` 精确关联来源报价。
- `dealer_sales_document_item.source_quote_item_id` 精确关联来源明细。
- 订单明细冻结配置、BOM、价格、折扣和邮费快照。
- `dealer_sales_document_event` 保存转单与履约流水。

## 状态

主状态只保留：

- `SUBMITTED`：正式订单，等待付款或履约。
- `CANCELLED`：已取消，不可恢复。
- `COMPLETED`：已签收完成。

独立履约状态：

- `payment_status`：`UNPAID / PAID`
- `production_status`：`PENDING / IN_PRODUCTION / COMPLETED`
- `shipment_status`：`UNSHIPPED / SHIPPED / DELIVERED`

付款、生产、发货和签收按 `salesDocumentId` 使用同一把业务锁，并携带旧状态条件更新；并发重复请求不会覆盖付款或物流信息。

## 金额与快照

```text
listAmount - discountAmount = productAmount
productAmount + shippingAmount + taxAmount = totalAmount
```

- `listAmount` 来自确认客户报价的冻结产品金额。
- 商家等级折扣只作用于 `productAmount`。
- `shippingAmount` 使用报价冻结邮费，不参与折扣。
- 第一版 `taxAmount = 0`，币种由价格设置确定，当前为 `USD`。
- 转单后不重新查询配方、价格或邮费模板。
- 生产清单只读取订单 `bom_snapshot_json`，不展示价格。

## 接口

接口前缀 `/dealer/sales-documents`：

- `GET /list`、`GET /{id}`：列表和详情。
- `GET /customer/{customerId}`：客户订单历史。
- `PUT /{id}/cancel`：取消未进入受限履约阶段的订单。
- `PUT /{id}/payment`：确认付款。
- `PUT /{id}/production/start`、`PUT /{id}/production/complete`：生产流转。
- `PUT /{id}/ship`、`PUT /{id}/deliver`：发货和签收。
- `GET /{id}/pdf?type=ORDER|PRODUCTION`：订单或生产清单 PDF。
- `POST /{id}/export`、`POST /{id}/email`：Excel 和订单邮件。

销售订单没有新增、编辑、复制、删除、目录、测算、报价、撤回和提交接口。客户报价 PDF 由订单测算模块输出。

## 权限与引用

保留 `dealer:sales:list/query/cancel/payment/production/shipment/deliver/document/email`。无权限或状态不允许的按钮直接隐藏，后端继续做最终校验。

客户、可售产品、配方版本、邮费模板和来源报价的引用均通过结构化 ID 精确查询，不使用 JSON `LIKE`。订单取消不解除来源报价的一次转换关系。

所有业务时间按 UTC 写入，PostgreSQL 使用 `timestamptz`，前端按用户时区展示到分钟。
