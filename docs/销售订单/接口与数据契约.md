# 销售订单接口与数据契约

## 数据模型

- `dealer_sales_document` 保存客户、商家、地址、金额和业务状态。
- `dealer_sales_document_item` 保存产品、尺寸、配置和提交时冻结的 BOM、价格、折扣及邮费快照；`shipping_template_id` 结构化保存命中的邮费模板，用于精确引用判断。
- `dealer_sales_document_event` 保存关键操作流水。

所有业务时间由后端按 UTC 写入，PostgreSQL 初始化字段统一使用 `timestamptz`；前端按用户时区格式化展示。

状态流转为 `DRAFT -> QUOTED -> SUBMITTED -> COMPLETED`，草稿也可以直接进入 `SUBMITTED`。取消进入 `CANCELLED`。

付款、生产、物流分别使用 `paymentStatus`、`productionStatus`、`shipmentStatus`，不能用一个状态代替三个业务事实。

## 金额

`listAmount - discountAmount = productAmount`，`productAmount + shippingAmount + taxAmount = totalAmount`。第一版税费为零。

所有报价和提交动作均由后端重新调用现有产品价格试算，商家前端金额不作为可信数据。订单提交后只读取冻结快照。

## 接口

接口前缀为 `/dealer/sales-documents`：

- `GET /list`、`GET /{id}`：列表和详情。
- `GET /customer/{customerId}`：客户销售历史。
- `POST /`、`PUT /`、`POST /{id}/copy`、`DELETE /{ids}`：草稿维护。
- `GET /catalog/options`、`GET /catalog/{productId}/setup`：商家可见产品目录和业务选项。
- `POST /calculate-item`、`PUT /{id}/calculate`：单行和整单测算。
- `PUT /{id}/quote`、`PUT /{id}/reopen`、`PUT /{id}/submit`、`PUT /{id}/cancel`：销售单生命周期。
- `PUT /{id}/payment`、`PUT /{id}/production/start`、`PUT /{id}/production/complete`、`PUT /{id}/ship`、`PUT /{id}/deliver`：订单履约。
- `GET /{id}/pdf?type=QUOTE|ORDER|PRODUCTION`、`POST /{id}/export`、`POST /{id}/email`：PDF、Excel 和邮件输出。

报价、撤回、提交、取消、付款、开始生产、完成生产、发货和签收统一以 `salesDocumentId` 使用同一把业务锁。最终更新同时携带旧单据状态或旧履约状态作为条件；并发请求只有一个可以完成状态迁移，失败请求不写操作流水。重复提交已提交订单时返回已有订单号，重复确认已付款订单不覆盖原付款信息。

生产清单只允许平台履约人员生成，逐行读取 `bom_snapshot_json`，不重新查询当前配方或物料。客户报价和订单单据不输出 BOM、成本公式或内部折扣规则。

付款确认可绑定一个 OSS 付款凭证，主表保存 `payment_proof_media_id`。邮件失败只写失败流水，不改变销售单状态。

## 邮费与引用

- 面积区间统一为左闭右开：`minArea <= area < maxArea`；最后一段 `maxArea` 为空，表示不封顶。
- 销售明细同时保存 `shipping_template_id` 和完整邮费快照。删除邮费模板时按结构化 ID 精确判断引用，不解析或模糊匹配 JSON。
- 已提交订单只读取冻结的邮费与价格结果，模板后续停用或调价不改写历史单据。

## 权限

列表、编辑页和详情页按接口权限隐藏操作：新增、修改、删除、报价、提交、付款、生产、发货、签收、PDF、Excel 和邮件分别使用对应的 `dealer:sales:*` 权限。只有查询权限的用户只看到详情和只读数据，不展示点击后必然返回 `403` 的动作。
