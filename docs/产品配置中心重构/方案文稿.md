这版先作为**讨论版 V0.1**，重点解决：

> 技术科怎么维护源头数据，产品怎么给客户下单，BOM 怎么自动生成，基础物料和成品 SKU 怎么不乱。

# Dealer Portal 产品工程与配置中心完整方案 V0.1

## 1. 总体设计原则

你的系统不能按普通电商 SKU 来做，也不能按传统固定 BOM 来做。核心原则应该是：

```text
销售产品负责“客户买什么”
配置中心负责“客户怎么选”
工程中心负责“能不能做、怎么生产”
货品中心负责“真实物料和成品 SKU”
价格中心负责“怎么报价”
订单中心负责“冻结快照、防纠纷”
```

也就是说：

```text
销售产品 Product Model
  ↓
下单配置 Config Question / Answer
  ↓
工程模板 Engineering Template
  ↓
尺寸能力限制 / BOM 规则 / 包装规则
  ↓
订单配置
  ↓
生产 BOM 快照
```

# 2. 菜单结构建议

## 2.1 产品中心

产品中心解决的是：

> 卖什么产品，客户下单时看到什么，标品 SKU 怎么管理。

```text
产品中心
├── 销售产品
├── 标品 SKU
├── 产品分类
├── 下单配置模板
├── 下单选项配置
├── 销售展示配置
└── 产品发布检查
```

### 销售产品

例如：

```text
Custom Zebra Shades
Madison Zebra Shades
Custom Roller Shades
9.5 Zebra Shades
```

销售产品不是库存 SKU。
它是客户下单入口。

字段建议：

```text
product_code
product_name_cn
product_name_en
product_category
product_type: CUSTOM / READY_MADE / SEMI_CUSTOM
bound_engineering_template_id
bound_price_rule_id
bound_config_template_id
status
sort_order
```

---

### 标品 SKU

标品 SKU 是固定尺寸、固定颜色、可入库、可销售的成品。

例如：

```text
米色 25 英寸 × 72 高 9.5 斑马帘
白色 35 英寸 × 72 高 9.5 斑马帘
```

它应该绑定：

```text
销售产品
固定尺寸
固定配置
工程模板版本
标品 BOM 快照
```

标品 SKU 可以进 `item_master`，类型为：

```text
FINISHED_GOOD
```

---

### 下单配置模板

用于定义某个销售产品下单时要问哪些问题：

```text
控制方式
控制方向
安装方式
卷布方向
底杆颜色
罩壳颜色
电机型号
遥控器
是否包布
特殊说明
```

这里引用配置中心的问题/答案，不自己新建一套。

# 3. 配置中心

配置中心是你老系统“问题 + 答案 + 适用维度 + 答案绑定组件”的升级版。

它解决：

> 下单有哪些字段？有哪些答案？答案来自字典还是物料？选中答案后绑定什么物料、组件、规则？

```text
配置中心
├── 系统字典
├── 配置问题
├── 配置答案
├── 答案绑定
├── 联动规则
└── 适用范围规则
```

---

## 3.1 系统字典

适合放：

```text
拉珠 / 无拉 / 电机
左 / 右
内装 / 外装
正卷 / 反卷
是 / 否
```

这些是方式类、业务枚举，不是实物。

不适合放：

```text
白色拉珠
黑色铝材
杜亚电机
三通道遥控器
纸箱
螺丝
边盖
```

这些是真实采购物料，要进货品中心。

---

## 3.2 配置问题 Config Question

例如：

```text
控制方式 Control Type
控制方向 Control Side
安装方式 Mount Type
底杆颜色 Bottom Rail Color
电机型号 Motor Type
遥控器 Remote
```

字段建议：

```text
question_code
question_name_cn
question_name_en
input_type: RADIO / SELECT / CHECKBOX / TEXT / NUMBER
required
business_group
sort_order
apply_scope
apply_condition
visible_to_dealer
visible_to_factory
affects_price
affects_bom
affects_constraint
```

---

## 3.3 配置答案 Config Answer

答案要增加来源类型：

```text
DICT
ITEM
KIT
BOOLEAN
TEXT
NUMBER
```

示例：

### 控制方式

```text
问题：控制方式
答案：拉珠
source_type = DICT
source_ref = control_type.chain
```

### 底杆颜色

如果“铁灰底杆”对应真实采购物料：

```text
问题：底杆颜色
答案：铁灰 / Iron Gray
source_type = ITEM
source_ref = BOTTOM-RAIL-IRON-GRAY
```

### 安装包

```text
答案：标准安装包
source_type = KIT
source_ref = KIT-INSTALL-ZEBRA
```

---

## 3.4 答案绑定 Answer Binding

这是关键。你截图里的“选中后绑定组件”要升级。

一个答案可以绑定：

```text
物料 Item
组件包 Kit
显示后续问题
隐藏后续问题
价格规则
BOM 规则
尺寸限制规则
```

例如：

### 控制方式 = 拉珠

```text
显示：控制方向
显示：拉珠物料
隐藏：电机型号
隐藏：遥控器
绑定：拉珠系统组件包
触发：拉珠尺寸限制
```

### 控制方式 = 电机

```text
显示：电机型号
显示：遥控器
隐藏：拉珠颜色
绑定：电机组件包
触发：电机面积限制
```

### 底杆颜色 = 铁灰

```text
绑定：铁灰底杆
绑定：铁灰边盖
绑定：铁灰封头
```


## 4.1 统一 Item Master

底层必须统一：

```text
item_master
```

所有真实货品都进这里：

```text
面料
电机
拉珠
铝材
下杆
罩壳
遥控器
纸箱
螺丝
边盖
安装码
安装零件包
标品成品 SKU
```

字段建议：

```text
item_code
legacy_code
item_name_cn
item_name_en
item_type
category_id
base_unit
purchase_unit
inventory_unit
color
spec
brand
supplier_id
is_purchasable
is_stockable
is_sellable
is_bom_component
is_finished_good
status
```

---

## 4.2 基础物料和成品 SKU 要分开页面，但底层统一

你担心的点是对的：

* 完全混在一个页面，会很难维护。
* 完全分成不同数据源，后期统计会乱。

所以方案是：

```text
数据库统一 item_master
前端按类型分页面维护
```

比如：

```text
面料页面：只看 item_type = FABRIC
电机页面：只看 item_type = MOTOR
纸箱页面：只看 item_type = PACKAGING
标品 SKU 页面：只看 item_type = FINISHED_GOOD
```

这样既不乱，又能统一统计。

# 5. 组件包 / 安装零件包

组件包必须单独管理。

```text
工程中心 / 组件包
```

或者也可以放在货品中心下面：

```text
货品中心 / 组件包
```

它本质是：

> 一个虚拟 Kit，由多个基础物料组成。

例如：

```text
KIT-INSTALL-ZEBRA 标准斑马帘安装包
  ├── 安装码 × 4
  ├── 螺丝 × 8
  ├── 膨胀管 × 8
  └── 说明书 × 1
```

组件包字段：

```text
kit_code
kit_name_cn
kit_name_en
kit_type
status
```

组件包明细：

```text
kit_code
item_code
quantity
unit
remark
```

BOM 可以引用 Kit。

一期建议：

```text
生产单显示 Kit
后期需要精细库存时再展开 Kit 明细
```

# 6. 工程中心

工程中心是技术科主战场。

```text
工程中心
├── 工程模板
├── 生产属性
├── 尺寸能力限制
├── 扣减规则
├── BOM 规则
├── 组件包
├── 包装规则
├── 工程试算器
└── 工程版本发布
```

---

## 6.1 工程模板 Engineering Template

工程模板对应技术科维护的一套产品工程方案。

例如：

```text
9.5 斑马帘工程模板
大货斑马帘工程模板
遮阳卷帘工程模板
蜂巢帘工程模板
半定制斑马帘工程模板
```

字段建议：

```text
template_code
template_name
product_type
version
status: DRAFT / TESTING / ACTIVE / ARCHIVED
effective_from
effective_to
remark
```

销售产品和标品 SKU 都应该绑定工程模板。

---

## 6.2 生产属性

生产属性不是物料。

它是会影响生产方式和 BOM 的配置字段。

例如：

```text
控制方式
控制方向
安装方式
卷布方向
是否包布
电机方案
遥控方案
```

而下面这些不是生产属性，是物料：

```text
白色拉珠
黑色铝材
杜亚电机
1通道遥控器
纸箱
螺丝
边盖
```

生产属性来源于配置中心的字典/问题。

---

## 6.3 尺寸能力限制

这块专门解决：

> 超过一定尺寸不能用拉珠、不能无拉、不能某种电机。

例如：

```text
面料系列 = Earth Semi-Opaque
控制方式 = 无拉
Max Width = 86.6
Max Height = 82
说明 = 不能做无拉
```

字段建议：

```text
template_id
fabric_series
control_type
mount_type
min_width
max_width
min_height
max_height
max_area
action: DISABLE_OPTION / BLOCK / WARN
target_question
target_answer
message_cn
message_en
priority
status
```

前端用于禁用选项，后端用于最终拦截。

---

## 6.4 扣减规则

用于计算生产尺寸。

例如：

```text
内装宽度扣减 0.5 inch
外装不扣减
某产品高度扣减 0.25 inch
```

字段：

```text
template_id
mount_type
width_deduction
height_deduction
unit
effective_from
status
```

订单中必须冻结：

```text
客户尺寸
扣减规则
生产尺寸
```

---

## 6.5 BOM 规则

BOM 规则不是固定 BOM，而是：

```text
条件 + 物料/组件包 + 数量公式
```

BOM 行类型：

```text
固定件
条件件
公式件
包装件
组件包
```

字段建议：

```text
template_id
line_no
line_type
item_code / kit_code
condition_expr
quantity_expr
unit
loss_rate
rounding_rule
remark
status
```

示例：

```text
永远使用：说明书 × 1
当控制方式=拉珠：拉珠系统包 × 1
当控制方式=电机：电机 × 1
面料：生产宽 × 生产高
安装码：ceil(width / 24) + 1
```

---

## 6.6 包装规则

包装不是客户选项，一般由技术科配置。

例如：

```text
宽度 <= 30 inch → BOX-30
30 < 宽度 <= 60 inch → BOX-60
宽度 > 60 inch → BOX-LONG
```

字段：

```text
template_id
rule_name
condition_expr
box_item_code
quantity_expr
priority
status
```

一期建议：

```text
每个 Order Item 独立匹配纸箱
合箱优化二期再做
```

---

## 6.7 工程试算器

这是必须做的。

技术科配置完以后，要输入：

```text
产品模板
面料
宽度
高度
控制方式
安装方式
颜色/物料选择
```

输出：

```text
可用选项
禁用原因
扣减后尺寸
BOM 展开结果
包装结果
是否通过校验
```

没有试算器，技术科配置一定会出错。

# 9. 订单中心如何消费这些配置

订单不自己维护生产规则，只消费已发布配置。

流程：

```text
选择销售产品
  ↓
加载下单配置模板
  ↓
加载配置问题和答案
  ↓
输入尺寸
  ↓
执行尺寸能力限制
  ↓
过滤不可选答案
  ↓
用户选择配置
  ↓
计算价格
  ↓
执行扣减规则
  ↓
展开 BOM
  ↓
匹配包装
  ↓
生成 Quote / Order
  ↓
冻结配置快照、价格快照、BOM 快照
```

订单要冻结：

```text
产品快照
配置快照
尺寸快照
扣减快照
价格快照
BOM 快照
包装快照
状态历史
```

# 10. 技术科工作流

技术科日常应该这样操作：

```text
1. 维护基础物料
2. 维护组件包
3. 创建 / 复制工程模板
4. 配置模板支持的生产属性
5. 配置尺寸能力限制
6. 配置扣减规则
7. 配置 BOM 规则
8. 配置包装规则
9. 使用工程试算器验证
10. 发布工程模板版本
11. 产品绑定工程模板
12. 订单开始使用
```

技术科不要直接改订单，不要在订单里手填 BOM。

---

# 11. 销售 / 产品运营工作流

销售或产品运营负责：

```text
1. 创建销售产品
2. 绑定工程模板
3. 绑定价格规则
4. 配置下单问题
5. 决定哪些答案 Dealer 可见
6. 配置展示名称、排序、默认值
7. 配置是否加价
8. 产品发布检查
9. 上架销售产品
```

边界：

```text
技术科决定能不能做
运营决定给不给客户选
Dealer 决定这单实际选什么
```

# 14. 最关键的结论

你现在应该把系统核心定成：

```text
配置中心：管问题和答案
货品中心：管真实物料和成品 SKU
工程中心：管技术科规则和 BOM
产品中心：管销售产品和下单入口
价格中心：管报价
订单中心：消费配置并冻结快照
```

不要让订单自己维护 BOM。
不要让销售产品直接维护螺丝纸箱。
