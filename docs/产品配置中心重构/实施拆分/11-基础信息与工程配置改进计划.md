## 1. 当前结论

当前基础信息已经有底子：

```text
pc_product_category

pc_fabric_series
```

但基础信息还需要按 ERP 主数据底座补齐：

```text
物料资料必须成为统一大主表
面料、铝材、电机、拉珠、遥控器、太阳能板、配件、安装件、包装件、标准成品都进入 pc_material
不同物料类型通过 material_type 和物料属性区分
组件包只是物料组合，不是另一套物料体系
物料主表需要规格摘要 spec_summary，减少列表、导出、工程预览多表联查
```


## 2. 模块边界

当前阶段只做两块：

```text
基础信息
工程配置
```

产品配置中心处理方式：

```text
先保留现有销售产品、配置模板、问题、答案、求值器
暂停继续深入扩展
后续作为客户下单问题展示层，消费工程配置
```

判断：

```text
是的，当前真正要补的就是基础信息 + 工程配置两块。
基础信息负责主数据，工程配置负责技术科维护产品怎么组成、什么条件下能选、会带出什么。
产品配置中心不是不要，而是先放着，等工程配置稳定后再做映射。
```

### 2.1 菜单分配

建议菜单顺序：

```text
基础信息
  录入向导
  产品分类
  物料资料
  物料属性
  单位管理
  面料系列
  面料资料
  组件包
  资料资产

工程配置
  工程方案
  构成项设置
  可选范围
  能力规则
  带出规则
  标品固定配置
  工程配置预览
  工程配置检查

产品配置中心
  销售产品
  配置模板
  问题组
  配置问题
  配置答案
  配置求值器
```

菜单原则：

```text
基础信息先于工程配置。
工程配置先于产品配置中心。
产品配置中心不再承载工程规则，只承载客户问题展示和答案选择。
低频中间表不强行做一级菜单，能从工作台进入的就从工作台进入。
```

### 2.2 对现有功能的影响

| 现有功能 | 当前处理 |
| --- | --- |
| 基础资料页面 | 保留，补统一物料资料、规格摘要、ERP 主数据字段 |
| 录入向导 | 保留，作为基础信息快速录入入口 |
| 物料资料 | 保留并强化为统一大主表，不拆铝材、系统、配件、包装独立主表 |
| 面料系列 / 面料资料 | 保留，承载面料物理资料和原始可用范围 |
| 组件包 | 保留，表达系统包、安装包、配件包、包装包 |
| 产品配置中心 | 保留，不继续扩张；后续消费工程配置 |
| 配置求值器 | 暂保留，后续接工程配置预览 |
| 价格、发布、订单快照 | 本阶段不展开，只保留后续消费工程配置的方向 |

## 3. 基础信息改进目标

基础信息目标：

```text
为未来 ERP、工程配置、采购、库存提供统一主数据底座
但当前阶段不做采购订单、库存流水和 MRP
```

### 3.1 统一物料资料

`pc_material` 是核心大主表。

物料类型至少覆盖：

```text
FABRIC          面料
PROFILE         型材 / 铝材
BOTTOM_RAIL     下杆
TRACK           轨道
MOTOR           电机
CHAIN           拉珠 / 链条
REMOTE          遥控器
SOLAR_PANEL     太阳能板
ACCESSORY       配件
INSTALL_PART    安装件
PACKAGING       包装件
FINISHED_GOOD   标准成品
SERVICE         服务项，预留
```

不要为铝材、系统、配件、包装再建独立主表。

### 3.2 物料主字段

`pc_material` 建议补齐：

```text
material_code
material_name_cn
material_name_en
material_type
unit_code
purchase_unit_code
inventory_unit_code
usage_unit_code
supplier_code
supplier_name
vendor_item_no
primary_spec
spec_summary
primary_color
primary_weight
purchase_enabled
inventory_enabled
legacy_source
legacy_id
status
```

字段说明：

```text
primary_spec：标准规格，来自供应商或工厂。
spec_summary：规格摘要，标准规格 + 高频补充属性，用于列表、搜索、导出、工程配置预览。
pc_material_attribute：结构化属性明细，工程规则判断仍然引用结构化属性，不只依赖 spec_summary。
```

### 3.3 物料属性

差异属性进入 `pc_material_attribute`：

```text
面料：厚度、门幅、克重、材质、纹理
铝材：长度规格、颜色、截面规格、型号
电机：电压、协议、扭矩、管径
遥控器：通道数、协议
包装件：长、宽、高、包装类型
安装件：安装方式、适用系列
```

### 3.4 面料资料

面料仍然保留：

```text
pc_fabric_series
pc_fabric_profile
```

原因：

```text
面料有系列、门幅、厚度、双面料组合等高频工程配置属性。
面料本身仍然也是 pc_material 的一种。
```

### 3.5 组件包

组件包使用：

```text
pc_component
pc_component_item
```

组件包类型建议：

```text
SYSTEM_PACK       系统组件包
INSTALL_PACK      安装零件包
ACCESSORY_PACK    配件包
PACKAGING_PACK    包装包
```

组件包引用的明细仍然来自 `pc_material`。

## 6. 工程构成项

产品系列工程方案先定义构成项。

示例：

```text
蜂巢帘工程方案 V1

构成项：
主面料
副面料，可选
铝材
铝材颜色
系统
配件
安装零件包
包装方式
```

构成项来源示例：

```text
主面料 -> pc_material where material_type = FABRIC
铝材 -> pc_material where material_type in PROFILE / BOTTOM_RAIL / TRACK
系统 -> pc_material where material_type in MOTOR / CHAIN / SOLAR_PANEL
配件 -> pc_material where material_type = ACCESSORY
安装零件包 -> pc_component where component_type = INSTALL_PACK
包装方式 -> pc_material where material_type = PACKAGING
```

## 7. 工程能力规则

规则基于工程构成项和结构化属性。

示例：

```text
主面料厚度 + 副面料厚度 > 最大可卷厚度
  -> 阻断
```

```text
宽度 > 250cm
  -> 禁用系统：拉珠
```

```text
系统 = 电机
  -> 启用遥控器、太阳能板等配件选项
```

规则表达第一阶段使用受控 JSON，不做任意脚本。

## 8. 工程带出规则

带出规则表达满足条件后自动带出什么。

示例：

```text
系统 = 电机
  -> 带出电机组件包
  -> 带出电机说明书

安装方式 = 墙装
  -> 带出墙装安装零件包

宽度 > 180cm
  -> 带出加长纸箱
```

带出数量是默认数量，不代表真实生产用量。

## 9. 标品固定工程配置

标品和定制共用同一套工程方案。

```text
定制：构成项多数由客户选择，规则动态控制。
标品：构成项大多固定，只做预览和完整性检查。
```

示例：

```text
55寸五拉蜂巢帘
主面料 = 固定面料
系统 = 五拉
铝材 = 固定铝材
安装零件包 = 标准安装包
包装方式 = 标准纸箱
```

## 10. 工程配置预览

预览用于技术科验证方案。

输入：

```text
工程方案版本
产品系列
宽度
高度
已选构成项
标品 SKU，可选
```

输出：

```text
已解析构成项
可选项
禁用项
带出组件包
带出物料
带出资料
提示
阻断
缺失项
命中规则
```

## 11. 工程配置检查

检查内容：

```text
必选构成项是否有可选范围
构成项引用的物料和组件包是否存在且启用
规则引用的构成项是否存在
规则引用的属性是否存在
带出规则是否缺物料、缺组件包、缺资料
标品固定工程配置是否完整
预设检查样例是否通过
```

## 12. Wave 计划

### Wave 1：基础信息补强

```text
梳理 pc_material 字段
新增或调整 spec_summary
补采购单位、库存单位、使用单位等主数据字段
补 purchase_enabled / inventory_enabled 等基础标记
补物料类型字典
补物料属性维护入口
整理组件包类型
```

数据库产物：

```text
pc_material 字段补齐或语义调整
pc_material_attribute 继续作为结构化属性明细
pc_base_attribute 补物料属性定义
pc_component / pc_component_item 明确组件包类型和明细
基础信息菜单和字典调整
```

后端产物：

```text
物料资料 CRUD 增强
物料属性 CRUD 增强
组件包和组件明细 CRUD 增强
基础信息引用检查
spec_summary 同步策略
```

前端产物：

```text
物料资料列表和抽屉增强
物料属性维护入口
组件包明细维护入口
基础信息录入向导同步现有单独功能
```

### Wave 2：工程配置数据库和后端

```text
新增工程方案相关表
新增工程构成项和可选范围接口
新增工程能力规则和带出规则接口
新增标品固定工程配置接口
新增工程配置预览引擎

```

数据库产物：

```text
pc_engineering_plan
pc_engineering_plan_version
pc_engineering_item
pc_engineering_item_scope
pc_engineering_rule
pc_engineering_output_rule
pc_standard_sku_engineering

```

后端产物：

```text
EngineeringPlanController / Service
EngineeringItemController / Service
EngineeringRuleController / Service
EngineeringPreviewEngine

```

### Wave 3：工程配置前端

```text
工程方案页面
构成项配置页面
可选范围配置页面
规则配置页面
标品固定工程配置页面
工程配置预览页面
工程配置检查页面
```

前端设计原则：

```text
工程配置以工作台为主，不做很多孤立 CRUD 页面。
左侧选择工程方案和版本，中间维护构成项，右侧维护可选范围、能力规则、带出规则。
预览和检查要能让技术科快速看到缺什么、禁用了什么、带出了什么。
新增和编辑优先弹窗或抽屉，不把页面拉成无尽长表单。
```

### Wave 4：产品配置中心衔接，后置

```text
销售产品绑定工程方案版本
配置问题映射工程构成项
配置答案来源于构成项可选范围
配置求值器调用工程配置预览
```

Wave 4 不在当前阶段展开，只作为后续衔接方向。

### Wave 5：测试数据和浏览器验证

测试数据：

```text
至少 1 个产品系列：蜂巢帘或卷帘
至少 1 个工程方案版本
至少 6 类构成项：主面料、铝材、系统、配件、安装零件包、包装方式
至少 1 条双面料厚度限制
至少 1 条宽度禁用系统规则
至少 1 条系统选择后带出组件包规则
至少 1 个标品固定工程配置
```

浏览器验证：

```text
基础信息能维护物料、属性、组件包
工程方案能创建版本
构成项能维护可选范围
能力规则能保存并参与预览
带出规则能保存并参与预览
标品固定配置能预览
工程配置检查能提示缺失项和引用错误
页面不出现横向滚动、内容挤压、按钮遮挡和表单字段溢出
```

## 13. 当前实现是否满足要求

结论：

```text
不满足。
```

已经满足：

```text
基础资料有 CRUD 底子
产品配置中心有问题 / 答案 / 求值器骨架
资料资产和组件包方向可复用
```

还缺：

```text
基础物料主数据字段不够完整
spec_summary 口径未落实
工程方案不存在
工程构成项不存在
构成项可选范围不存在
工程能力规则不存在
工程带出规则不存在
标品固定工程配置不存在
工程配置预览和检查不存在
产品配置中心还没有消费工程配置
```

影响：

```text
如果不补工程配置，订单系统无法稳定判断某个产品在某个尺寸和选择下能不能做。
如果不补工程配置，技术科只能把规则塞进配置问题和答案，后续会变成无限联动。
如果基础物料不补 ERP 主数据字段，以后采购、库存、成本、工程引用都会返工。
```

## 14. 验收标准

```text
基础信息中能维护统一物料资料，物料有 spec_summary
面料、铝材、电机、拉珠、遥控器、太阳能板、配件、安装件、包装件都能作为物料维护
组件包能引用物料组成安装包、系统包、包装包
能为一个产品系列创建工程方案版本
能定义主面料、副面料、铝材、系统、配件、安装零件包、包装方式等构成项
能配置构成项可选范围
能配置双面料厚度组合限制
能配置宽度限制系统可选
能配置系统选择后带出组件包和资料
能配置标品 SKU 的固定工程配置
能运行工程配置预览
能运行工程配置检查
```
