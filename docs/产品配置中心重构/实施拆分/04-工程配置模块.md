# 04 - 工程配置模块

## 1. 模块定位

工程配置不是生产工艺，也不是 BOM。

工程配置解决：

```text
一个产品系列由哪些工程构成项组成
每个构成项可以从哪些基础资料中选择
什么条件下某个系统、配件、面料组合、包装方式能选或不能选
什么条件下需要带出组件包、安装零件包、包装件或资料
定制产品和标品 SKU 如何共用同一套工程方案
技术科如何预览和检查一个工程方案是否完整
```

工程配置不解决：

```text
裁切扣减
真实生产用量
生产工序
车间工艺
库存扣减
采购计划
MRP
销售报价
订单遥控器分组操作
```

这些后续模块可以消费工程配置，但不在当前模块实现。

## 2. 和基础信息的关系

基础信息提供统一资料池：

```text
pc_material            统一物料资料
pc_material_attribute  物料属性
pc_unit                单位
pc_fabric_series       面料系列
pc_fabric_profile      面料资料
pc_component           组件包
pc_component_item      组件包明细
pc_media_asset         资料资产
pc_media_binding       资料绑定
```

工程配置只引用基础信息，不再创建铝材资料、系统资料、配件资料、包装资料等独立资料源。

示例：

```text
构成项：主面料
来源：pc_material where material_type = FABRIC

构成项：铝材
来源：pc_material where material_type in PROFILE / BOTTOM_RAIL / TRACK

构成项：系统
来源：pc_material where material_type in MOTOR / CHAIN / CORDLESS / SOLAR_PANEL

构成项：安装零件包
来源：pc_component where component_type = INSTALL_PACK

构成项：包装方式
来源：pc_material where material_type = PACKAGING
或 pc_component where component_type = PACKAGING_PACK
```

## 3. 核心对象

建议表：

```text
pc_engineering_plan              工程方案
pc_engineering_plan_version      工程方案版本
pc_engineering_item              工程构成项
pc_engineering_item_scope        构成项可选范围
pc_engineering_rule              工程能力规则
pc_engineering_output_rule       工程带出规则
pc_standard_sku_engineering      标品固定工程配置
pc_engineering_check_case        工程配置检查样例
```

### 3.1 工程方案

工程方案挂在产品系列上。

```text
工程方案：蜂巢帘工程方案
适用系列：蜂巢帘
版本：V1
状态：草稿 / 启用 / 停用
```

同一个产品系列可以有多个版本，技术科维护草稿，发布时使用启用版本。

### 3.2 工程构成项

工程构成项定义这个系列由哪几类东西组成。

示例：

```text
蜂巢帘工程方案 V1

构成项：
1. 主面料
2. 副面料，可选
3. 铝材
4. 铝材颜色
5. 系统
6. 配件
7. 安装零件包
8. 包装方式
```

字段建议：

```text
item_code             构成项编码，例如 MAIN_FABRIC / SYSTEM / PACKAGING
item_name_cn          构成项中文名
item_type             FABRIC / MATERIAL / COMPONENT / OPTION / PACKAGE
source_type           MATERIAL / COMPONENT / FABRIC_SERIES / MATERIAL_ATTRIBUTE
required_flag         是否必选
multi_select_flag     是否多选
customer_selectable   后续是否可映射为客户选择问题
default_source_code   默认来源
sort_order            排序
```

### 3.3 构成项可选范围

构成项可选范围定义每个构成项可以选哪些基础资料。

示例：

```text
主面料：
  material_type = FABRIC
  series_code in FS_HONEYCOMB

系统：
  material_type in CHAIN / MOTOR / CORDLESS / SOLAR_PANEL

安装零件包：
  component_type = INSTALL_PACK
```

字段建议：

```text
scope_type       MATERIAL_TYPE / MATERIAL_CODE / FABRIC_SERIES / COMPONENT_TYPE / COMPONENT_CODE / ATTRIBUTE
scope_code       范围编码
include_flag     INCLUDE / EXCLUDE
condition_json   可选范围条件
sort_order
```

## 4. 工程能力规则

工程能力规则用于判断能不能选、要不要提示、要不要阻断。

规则只能引用工程方案中已经定义的构成项和基础信息属性，不能凭空写业务词。

规则类型：

```text
尺寸限制
厚度组合限制
系统可选限制
配件可选限制
包装匹配限制
必选补齐
提示
阻断
```

示例：

```text
规则：双面料厚度限制
条件：主面料.thickness + 副面料.thickness > 系列.max_combined_thickness
动作：阻断
提示：双面料组合厚度超限，当前系统无法卷入
```

```text
规则：宽度超过限制禁用拉珠
条件：width > 250cm
动作：禁用 构成项 SYSTEM 中的 CHAIN
提示：当前宽度不支持拉珠系统
```

```text
规则：电机系统显示遥控和太阳能
条件：SYSTEM = MOTOR
动作：启用 构成项 ACCESSORY 中的 REMOTE / SOLAR_PANEL
```

当前阶段规则表达可以先采用受控 JSON，不做任意脚本表达式。

## 5. 工程带出规则

工程带出规则用于表达满足条件后应该带出什么组件包、安装包、包装件或资料。

示例：

```text
条件：SYSTEM = MOTOR
带出：电机组件包
带出：电机安装说明
```

```text
条件：安装方式 = WALL
带出：墙装安装零件包
```

```text
条件：width > 180cm
带出：加长纸箱
```

字段建议：

```text
condition_json      条件
output_type         MATERIAL / COMPONENT / MEDIA
output_code         物料、组件包或资料编码
default_qty         默认数量
unit_code           单位
reason_text         带出原因
sort_order
```

default_qty 只表示默认带出数量，不代表真实生产用量。

## 6. 标品固定工程配置

标品和定制共用同一套工程方案。

定制产品：

```text
构成项多数由客户选择
规则动态控制可选项
```

标品 SKU：

```text
构成项大多固定
只做预览和完整性检查
```

示例：

```text
标品 SKU：55寸五拉蜂巢帘

固定配置：
主面料 = 米色 25 寸 9.5 斑马帘
系统 = 五拉
铝材 = 固定上轨 / 下杆
安装零件包 = 标准安装包
包装方式 = 标准纸箱
```

## 7. 工程配置预览

预览输入：

```text
productSeriesCode
engineeringPlanVersionId
salesProductCode，可选
standardSkuCode，可选
width
height
selectedItems
```

预览输出：

```text
resolvedItems          已解析构成项
availableOptions       可选项
disabledOptions        禁用项
outputComponents       带出组件包
outputMaterials        带出物料
outputMediaAssets      带出资料
warnings               提示
blockers               阻断
missingItems           缺失构成项
matchedRules           命中的工程规则
```

`matchedRules` 给技术科看规则命中原因，不给生产工艺使用。

## 8. 工程配置检查

发布或启用工程方案前必须检查：

```text
必选构成项是否都有可选范围
构成项引用的物料、组件包是否存在且启用
标品固定配置是否完整
工程规则引用的构成项是否存在
工程规则引用的物料或组件包是否存在
带出规则是否缺物料、缺组件包、缺资料
规则之间是否有明显冲突
预设检查样例是否能通过
```

## 9. 技术科工作流

```text
1. 在基础信息维护物料、属性、单位、组件包、资料资产
2. 创建产品系列工程方案
3. 定义工程构成项
4. 配置每个构成项的可选范围
5. 配置工程能力规则
6. 配置工程带出规则
7. 配置标品固定工程配置
8. 运行工程配置预览
9. 运行工程配置检查
```

## 10. 一期验收

```text
能为一个产品系列创建工程方案版本
能定义主面料、副面料、铝材、系统、配件、安装零件包、包装方式等构成项
能为构成项配置基于物料类型、面料系列、组件包类型的可选范围
能配置双面料厚度组合限制
能配置宽度超过限制禁用某个系统
能配置选择电机后带出电机组件包和资料
能配置标品 SKU 的固定工程配置
能运行工程配置预览并返回可选项、禁用项、带出项、提示和阻断
能运行工程配置检查并提示缺失和引用错误
```
