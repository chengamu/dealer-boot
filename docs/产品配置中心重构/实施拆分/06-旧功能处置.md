## 4. SQL 处置

可以直接重写：

```text
sql/postgresql/product_capability.sql
```

可以删除旧菜单初始化，按新菜单重建：

```text
基础资料
产品与配置
工程与 BOM
价格与发布
```

如果旧表命名误导，可以直接改表名。

## 5. 保留原则

保留：

```text
已经符合新业务边界的表
已经符合项目规范的 CRUD 代码
已有可复用的上传、权限、i18n、UTC、字典能力
```

删除或重写：

```text
假数据页面
只为了演示的 mock
泛泛的 Product Capability 大杂烩页面
前端自己推导价格 / BOM 的逻辑
重复 OSS 的附件逻辑
不符合新模块边界的菜单
```
