# 03 - 产品与配置模块

## 1. 模块目标

产品与配置模块解决：

```text
客户买什么
客户下单时看到什么
客户能选什么
答案来自字典、物料还是组件包
选择答案后触发哪些规则
```

不解决：

```text
最终价格
最终生产 BOM
库存和采购
```

## 2. 页面

一期页面：

```text
销售产品
标品 SKU
配置模板
问题组模板
配置问题
配置答案
配置求值器
```

## 3. 数据库

优先表：

```text
pc_sales_product
pc_standard_sku
pc_config_template
pc_config_template_version
pc_question_group
pc_config_question
pc_config_option
pc_config_rule
```

如果暂时不重命名：

```text
pc_product_model 先承载销售产品
pc_sales_variant 先承载销售变体 / 标品 SKU
```

## 4. 后端建议

Controller：

```text
ProductConfigController
```

Service：

```text
ProductConfigService
```

Engine：

```text
ConfigEvaluationEngine
```

`ConfigEvaluationEngine` 要从现在的必填校验升级为：

```text
返回可见问题
返回可选答案
返回禁用原因
返回必填校验
返回答案带出的组件
返回答案绑定的资料
返回需要工程试算的输入
```

## 5. 先做 CRUD

```text
销售产品
标品 SKU
配置模板
问题组模板
配置问题
配置答案
配置规则
```

## 6. 必须尽早做试算器

配置求值器不能后置太久。

最小输入：

```text
templateVersionId
width
height
selectedOptions
```

最小输出：

```text
visibleQuestions
availableOptions
disabledOptions
validations
autoComponents
mediaAssets
warnings
blockers
```

## 7. 一期验收

```text
能创建一个销售产品
能绑定一个配置模板
能维护问题和答案
答案能引用字典、物料或组件
输入尺寸和已选答案后，后端返回可见问题、可选答案和校验结果
```
