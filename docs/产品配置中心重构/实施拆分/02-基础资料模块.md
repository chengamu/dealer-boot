# 02 - 基础资料模块

## 1. 模块目标

基础资料解决：

```text
面料是什么
物料是什么
组件包是什么
附件资料在哪里
这些资料如何被产品、配置、工程、BOM 和订单引用
```

不解决：

```text
采购审批
库存扣减
完整 ERP 物料计划
生产排程
```

## 2. 页面

一期页面：

```text
面料设置
物料设置
组件包
配置字典
附件资料
单位管理
```

## 3. 数据库

优先表：

```text
pc_product_category
pc_unit
pc_item_master 或 pc_material
pc_material_attribute
pc_fabric_series
pc_fabric_profile
pc_component
pc_component_item
pc_base_attribute
pc_media_asset
pc_media_binding
```

面料建模边界：

```text
面料系列：承载规则和约束，例如厚度组合限制、可用宽幅、适用产品线、适用控制系统等
面料资料：承载系列下的具体颜色、材质、纹理、工厂型号等差异项
其他物料：电机、遥控器、PET胶条、安装码、配重条等不按类型单独建扩展表，统一走物料属性
```

物料属性建模边界：

```text
pc_material：物料主表，保存物料编号、名称、类型、单位、供应商、高频查询字段
pc_base_attribute：属性定义，例如规格、重量g、电压、控制协议、通道数、形状、颜色
pc_material_attribute：物料属性值，例如 H0004141 的规格=1.2，H0008007 的重量g=50、规格=3.5*12
不要为电机、遥控器、安装码、胶条、配重条等每个类型都建 profile 表
只有面料因为有系列规则和配置约束，才保留 pc_fabric_profile
```

面料相关字段必须覆盖：

```text
pc_fabric_series.default_thickness_unit
pc_fabric_series.max_thickness_diff
pc_fabric_series.max_combined_thickness
pc_fabric_series.thickness_rule_enabled
pc_fabric_series.width_rule_enabled
pc_fabric_series.available_widths
pc_fabric_series.min_width_value
pc_fabric_series.max_width_value
pc_fabric_series.width_unit
pc_fabric_profile.thickness_value
pc_fabric_profile.thickness_unit
pc_fabric_profile.color_code
pc_fabric_profile.material_composition
pc_fabric_profile.texture_type
pc_fabric_profile.finish_type
pc_fabric_profile.sample_book_no
pc_fabric_profile.vendor_item_no
pc_fabric_profile.gsm_value
pc_fabric_profile.supplier_code
pc_fabric_profile.supplier_name
pc_fabric_profile.purchase_unit_code
pc_fabric_profile.inventory_unit_code
pc_fabric_profile.sales_unit_code
pc_fabric_profile.legacy_attribute_text
pc_material_attribute.attribute_code
pc_material_attribute.attribute_value
pc_material.attribute_summary
```

说明：

```text
面料系列负责维护大多数约束
系列上需要维护可用门幅、最小/最大门幅、厚度组合限制等规则
单个面料资料主要区分颜色、材质、纹理、工厂型号等
面料资料可以记录自己的厚度属性，但厚度组合规则挂在系列上
业务主编号只保留一个：material_code；面料作为物料的一种，不再单独设计面料主编号
旧表里“名称、颜色、门幅、型号、供应商、平方克重”等竖向描述必须拆成结构化字段
旧订单系统编号不单独建业务字段，统一进入 legacy_source / legacy_id
样册编号、供应商编码、供应商料号属于外部关联编号，可以保留字段，但不能替代 material_code
例如两块/两层面料组合时，最大厚度差不能超过多少，或总厚度不能超过多少
```

字段冗余原则：

```text
性能和后续分析优先，允许主动冗余
不要为了纯范式化导致列表页、试算器、发布检查、订单快照大量联表
冗余字段由 Service 在新增/修改/导入时同步
重要对象保留 code + name 快照，例如 material_code、material_name_cn、series_code、series_name_cn、supplier_name、unit_code
冗余字段不作为唯一身份判断，唯一主编号仍然以 material_code 等主业务编号为准
不同类型物料的零散属性进入 pc_material_attribute，常用展示字段可同步到 pc_material.attribute_summary
```

## 5. OSS 边界

附件资料必须复用 OSS。

```text
文件本体：sys_oss / bocoo-common-oss
上传接口：/system/oss/upload
下载接口：/system/oss/download/{ossId}
资料资产：pc_media_asset，只保存 ossId 和业务资料信息
资料绑定：pc_media_binding，保存绑定对象和用途
```

禁止重做文件系统。

## 7. 一期验收

```text
能新增一个面料系列
能新增一个具体面料
能维护面料厚度，并在系列上设置两块/两层面料的厚度组合限制
能新增一个电机或遥控器物料
能给 PET 胶条、安装码、配重条、电机、遥控器维护不同属性，而不是新增类型专属表
能新增一个组件包
能上传一个色卡图到 OSS
能把色卡图绑定到面料或配置答案
能维护单位、单位精度和基础换算
能维护配置字典属性，例如颜色、涂层、纹理、控制系统、罩壳分类
能写入至少 5 个产品测试样本，覆盖面料、控制系统、电机/遥控、通用物料属性、组件包、附件资料
停用资料或物料时能提示引用
```
