# 02 - 基础资料模块

## 1. 模块目标

基础资料解决：

```text
统一物料资料是什么
物料有哪些标准规格和补充属性
物料如何被工程配置按类型引用
组件包由哪些物料组成
资料资产在哪里
这些资料如何被未来 ERP、工程配置、产品配置中心共同引用
```

不解决：

```text
采购审批
库存扣减
完整 ERP 物料计划
生产排程
销售报价
产品尺寸能力判断
客户下单问题配置
```

优先表：

统一物料资料边界：

```text
pc_material 是基础信息的核心大主表。
面料、铝材、电机、拉珠、遥控器、太阳能板、轨道、下杆、配件、安装件、包装件、标准成品都进入 pc_material。
不同物料通过 material_type 区分，不为铝材、系统、配件、包装分别拆独立主数据表。
差异属性进入 pc_material_attribute，高频展示和导出字段可以冗余到 pc_material。
```

面料建模边界：

```text
面料系列：承载面料物理资料和原始可用范围，例如默认厚度、厚度差提示、可用门幅、最小/最大门幅
面料资料：承载系列下的具体颜色、材质、纹理、工厂型号等差异项
其他物料：电机、遥控器、PET胶条、安装码、配重条等不按类型单独建扩展表，统一走物料属性
```

面料系列不承载工程生产规则：

```text
不要在面料系列里维护“某产品能不能做”
不要在面料系列里维护“某控制方式是否禁用”
不要在面料系列里维护“超尺寸是否需要双电机”
不要在面料系列里维护“生产尺寸扣减公式”
不要在面料系列里维护“生产用料规则”
```

这些规则必须进入工程配置模块。

物料属性建模边界：

```text
pc_material：物料主表，保存物料编号、名称、类型、单位、供应商、高频查询字段
pc_base_attribute：属性定义，例如规格、重量g、电压、控制协议、通道数、形状、颜色
pc_material_attribute：物料属性值，例如 H0004141 的规格=1.2，H0008007 的重量g=50、规格=3.5*12
不要为电机、遥控器、安装码、胶条、配重条等每个类型都建 profile 表
只有面料因为有系列规则和配置约束，才保留 pc_fabric_profile
```

说明：

```text
面料系列只负责维护面料自身物理资料和原始限制
系列上可以维护可用门幅、最小/最大门幅、厚度组合提示
单个面料资料主要区分颜色、材质、纹理、工厂型号等
面料资料可以记录自己的厚度属性，但厚度组合规则挂在系列上
业务主编号只保留一个：material_code；面料作为物料的一种，不再单独设计面料主编号
旧表里“名称、颜色、门幅、型号、供应商、平方克重”等竖向描述必须拆成结构化字段
旧订单系统编号不单独建业务字段，统一进入 legacy_source / legacy_id
样册编号、供应商编码、供应商料号属于外部关联编号，可以保留字段，但不能替代 material_code
例如两块/两层面料组合时，最大厚度差不能超过多少，或总厚度不能超过多少
```

物料主表必须覆盖：

```text
material_code        物料编码，唯一业务主编号
material_name_cn     中文名称
material_name_en     英文名称
material_type        物料类型
unit_code            默认单位
purchase_unit_code   采购单位
inventory_unit_code  库存单位
usage_unit_code      工程使用单位
supplier_code        供应商编码快照
supplier_name        供应商名称快照
vendor_item_no       供应商料号
primary_spec         标准规格
spec_summary         规格摘要：标准规格 + 高频补充属性
primary_color        主颜色
primary_weight       主重量/克重
purchase_enabled     是否可采购，当前只做主数据字段
inventory_enabled    是否入库，当前只做主数据字段
status               启用状态
```

规格字段边界：

```text
primary_spec 保存供应商或工厂标准规格。
spec_summary 保存标准规格 + 高频补充属性拼接后的可读摘要，用于列表、搜索、导出、工程配置预览，减少多表联查。
pc_material_attribute 仍然是结构化属性明细，工程规则判断不能只依赖 spec_summary。
```

价格字段边界：

```text
pc_material / pc_fabric_profile / pc_fabric_series 不维护销售单价
基础资料阶段不做报价规则、经销商价、运费、加价项
后续如果需要采购成本或参考成本，应单独进入成本/采购域或成本字段设计，不和销售价格混用
客户报价统一进入价格中心，由 pc_price_plan / pc_price_rule_item 等表维护
```

字段冗余原则：

```text
性能和后续分析优先，允许主动冗余
不要为了纯范式化导致列表页、试算器、发布检查、订单快照大量联表
冗余字段由 Service 在新增/修改/导入时同步
重要对象保留 code + name 快照，例如 material_code、material_name_cn、series_code、series_name_cn、supplier_name、unit_code
冗余字段不作为唯一身份判断，唯一主编号仍然以 material_code 等主业务编号为准
不同类型物料的零散属性进入 pc_material_attribute，常用展示字段可同步到 pc_material.attribute_summary
当前字段名如果仍保留 attribute_summary，后续应统一改名或语义调整为 spec_summary。
```

## 5. OSS 边界

附件资料必须复用 OSS。

```text
文件本体：sys_oss / bocoo-common-oss
上传接口：/system/oss/upload
下载接口：/system/oss/download/{ossId}
资料资产：pc_media_asset，只保存 ossId 和业务资料信息
资料绑定：pc_media_binding，保存绑定对象和用途
```

禁止重做文件系统。

## 7. 一期验收

```text
能新增一个面料系列
能新增一个具体面料
能维护面料厚度，并在系列上设置两块/两层面料的厚度组合限制
能新增一个电机或遥控器物料
能给 PET 胶条、安装码、配重条、电机、遥控器维护不同属性，而不是新增类型专属表
能新增一个组件包
能上传一个色卡图到 OSS
能把色卡图绑定到面料或配置答案
能维护单位、单位精度和基础换算
能维护配置字典属性，例如颜色、涂层、纹理、控制系统、罩壳分类
能写入至少 5 个产品测试样本，覆盖面料、控制系统、电机/遥控、通用物料属性、组件包、附件资料
停用资料或物料时能提示引用
```
