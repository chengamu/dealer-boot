生成日期：2026-06-11

本文只作为重构执行入口，不再承载所有细节。

当前项目状态：

```text
项目未上线
没有客户数据
属于首次正式编码前的重构
以前第一期做过的产品配置中心功能可以改、删、重命名、重建
```

但是仍然要遵守：

```text
不修改生产配置
不修改无关模块
不引入重依赖
不重做 OSS / 权限 / i18n / UTC 等通用能力
```

## 1. 执行方式

不要再按一个大文档推进。

后续按模块一块一块做：

```text
先改数据库脚本
再改后端包和接口
再改前端页面
最后跑一个产品线闭环
```

每一块都要能独立验收，不能只写骨架。

## 3. 一期主线

一期只先打通一条产品线，例如卷帘或斑马帘：

```text
面料资料
  -> 物料 / 组件
  -> 附件资料 OSS 绑定
  -> 销售产品
  -> 配置模板
  -> 工程模板
  -> BOM 预览
  -> 价格试算
  -> 发布检查
  -> 发布包
  -> 订单配置 / 价格 / BOM / 资料快照
```

---

## 4. 第一批建议

第一批不要铺满所有页面，先做：

```text
1. 数据库脚本重整
2. 基础资料：面料设置、物料管理、组件管理、资料资产
3. 产品与配置：销售产品、配置模板、问题答案
4. 工程与 BOM：工程模板、尺寸限制、BOM 预览
5. 价格与发布：价格试算、发布检查、订单快照
```

每批完成后再继续下一批。

---

## 5. 禁止再出现的问题

```text
不能把 CRUD 骨架当功能完成
不能把试算器放到后面无限延期
不能让前端自己拼配置、价格、BOM
不能让附件资料重做 OSS
不能把所有功能堆在一个 Product Center 大页面
不能把订单规则写死在订单模块
```
