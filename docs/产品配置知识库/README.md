# 产品配置知识库规范

本文是基础信息、产品工程设计、产品配置中心相关功能的文档入口。新增或重构这些功能时，先按本文判断是否需要补充知识库文档。

知识库目标不是把所有细节写成文档，而是让业务人员能顺手维护，让开发能明确接口和数据契约，让后续 AI 代办能力有稳定上下文。

## 适用范围

适用于以下模块：

- 基础信息
- 产品工程设计
- 产品配置中心
- 后续接入这些模块的 AI 代办能力

不适用于纯 UI 微调、临时验证脚本、内部实现细节整理。

## 文档分级

### P0 核心录入功能

满足任一条件时，必须补充两份文档：

- 独立菜单页面。
- 有新增、编辑、删除、启停、发布、检查、预览等业务动作。
- 有独立后端接口。
- 涉及数据库主表或核心关联表。
- 后续希望 AI 自动录入、维护或检查。
- 用户需要理解业务顺序才能正确使用。

必须补充：

- `操作手册.md`
- `接口与数据契约.md`

如果测试过程中发现当前页面体验问题，必须单独补充：

- `体验问题清单.md`

体验问题不写入正式操作手册，不写入稳定接口契约。

### P1 辅助维护功能

满足任一条件时，至少补充一份功能说明或操作手册：

- 隐藏菜单但有完整页面、接口和权限。
- 从主功能跳转进入的低频维护页。
- 工作台里的专用维护抽屉。
- 对核心录入有辅助作用，但不是第一入口。

建议补充：

- `操作手册.md` 或 `功能说明.md`

### P2 展示和跳转功能

满足以下条件时，不要求单独建目录，写入模块总览即可：

- 纯展示卡片。
- 纯统计指标。
- 只负责跳转到其他页面。
- 没有独立业务录入动作。
- 没有独立数据契约。

### 不需要知识库文档

以下内容不单独补文档：

- 按钮位置、间距、列宽、颜色等纯 UI 调整。
- i18n 文案修正。
- 表格字段显示顺序调整。
- 无业务语义变化的代码重构。
- 测试用临时脚本。

## 每个功能的文档口径

### 操作手册

给业务人员、技术科和测试人员阅读，重点回答：

- 这个功能解决什么业务问题。
- 谁使用这个功能。
- 使用前要先准备哪些基础资料。
- 推荐录入顺序是什么。
- 字段怎么填，哪些字段不能乱填。
- 常见错误怎么处理。
- 如何确认录入成功。

操作手册只描述目标状态和正确用法，不记录当前实现缺陷。

### 接口与数据契约

给开发、测试和 AI 工具调用阅读，重点回答：

- 页面路由、菜单、权限码。
- 主要接口。
- 请求和响应结构。
- 数据库主表、关联表和关键字段。
- 选择器数据来源。
- 状态值、字典值、单位来源；产品业务枚举必须说明来自 `pc_product_dict_type` / `pc_product_dict_item`，具体单位必须说明来自 `pc_unit`。
- 校验规则。
- 示例 payload。
- 常见错误码或错误场景。

接口与数据契约只描述稳定设计、接口和数据要求，不记录某一轮测试发现的临时问题。

### 体验问题清单

给产品、开发、测试和 Codex 协作使用，重点记录：

- 测试发现的问题。
- 用户截图或反馈来源。
- 期望行为。
- 影响范围。
- 修复优先级。
- 当前状态。
- 修复后用户确认结果。

体验问题清单是临时工作清单，不是用户手册。问题修复并确认后，由 Codex 负责更新状态；稳定后可以归档或移除已确认问题。

## AI 代办能力要求

如果功能后续需要支持“聊天框自动录入”，文档必须包含 AI 代办能力说明。

AI 代办说明至少包含：

- 用户意图。
- 必填槽位。
- 可推断字段。
- 需要追问的字段。
- 保存前确认摘要。
- 调用接口。
- 成功后返回给用户的确认信息。
- 失败时的可读提示。

示例：

```text
用户意图：新增物料资料
必填槽位：物料编码、中文名称、物料类型、工程使用单位
可推断字段：状态默认启用，采购启用默认否，库存启用默认否
需要追问：缺少物料类型时必须追问
保存前确认：展示物料编码、名称、类型、单位
调用接口：POST /product-capability/material
```

## 更新原则

- 先更新文档，再实现容易跑偏的复杂功能。
- 文档描述以当前确认后的业务口径为准，不保留已废弃方案影响判断。
- 不为了文档而文档；只有 P0/P1 功能需要独立文档。
- 文档可以先写最小可用版本，但不能漏业务入口、接口契约和 AI 代办判断。
- 功能行为变更时，同步更新对应知识库文档。
- 测试发现的问题写入 `体验问题清单.md`，不要污染 `操作手册.md` 和 `接口与数据契约.md`。
- 修复问题后，由 Codex 更新体验问题状态；用户确认后再标记为已确认或归档。
