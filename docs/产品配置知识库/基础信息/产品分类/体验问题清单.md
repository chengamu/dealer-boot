# 产品分类体验问题清单

本文记录产品分类页面测试过程中发现的体验问题。正式使用说明见 `操作手册.md`，稳定接口和数据要求见 `接口与数据契约.md`。

| 状态 | 含义 |

## 待修复

| 编号 | 问题 | 来源 | 期望行为 | 优先级 | 影响范围 |
| --- | --- | --- | --- | --- | --- |
| PC-CAT-001 | 列表现在是普通平铺 grid，看不出父子关系 | 用户浏览器标注：产品分类列表区域 | 改成树形表格，参考系统部门管理 | P0 | 产品分类维护、后续分类选择 |
| PC-CAT-002 | 新增或编辑时显示 `父级ID` 数字输入框 | 用户浏览器标注：编辑产品分类抽屉 | 改成 `上级分类` 树选择器，不让用户填写内部 ID | P0 | 新增分类、编辑分类 |
| PC-CAT-003 | 编辑已有分类时，上级分类显示内部 ID | 用户浏览器标注：`父级ID = 100001` | 显示上级分类名称，保存时由系统提交 ID | P0 | 编辑分类 |
| PC-CAT-004 | 表格默认展示 `层级` 字段，对业务人员低价值 | 用户浏览器标注：层级表头 | 默认隐藏层级；必要时放到详情或系统信息区 | P1 | 列表可读性 |
| PC-CAT-005 | 新增子分类入口不顺手 | 用户测试反馈 | 支持从父分类行直接新增子分类，自动带出上级分类 | P1 | 连续录入分类 |
| PC-CAT-006 | 新增或编辑时显示 `路径` 输入框，用户不知道该填什么 | 用户测试反馈：路径我没看懂 | 路径由系统根据上级分类和分类编码自动生成，详情只读展示即可 | P0 | 新增分类、编辑分类 |
