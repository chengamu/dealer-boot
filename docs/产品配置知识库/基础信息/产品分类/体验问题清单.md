## 已修复待确认

| 编号 | 问题 | 修复说明 | 验证方式 |
| --- | --- | --- | --- |
| PC-CAT-001 | 列表现在是普通平铺 grid，看不出父子关系 | 当前 `ProductBaseInfoPage.vue` 的 `category` 配置已启用 tree grid：`parentKey=parentId`、`rowKey=categoryId`。 | 重新打开 `/product-master/categories`，确认分类按树形层级展示。 |
| PC-CAT-002 | 新增或编辑时显示 `父级ID` 数字输入框 | 当前分类表单使用 `tree-select` 和分类 options 加载上级分类，不再让用户手填普通数字。 | 新增/编辑分类，确认字段显示为“上级分类”选择器。 |
| PC-CAT-003 | 编辑已有分类时，上级分类显示内部 ID | 当前上级分类使用分类选项 label 展示编码和名称，保存时提交内部 ID。 | 编辑已有子分类，确认上级分类显示可读名称。 |
| PC-CAT-004 | 表格默认展示 `层级` 字段，对业务人员低价值 | 当前 `categoryLevel` 设置为 `form: false, table: false`，不在列表和表单默认展示。 | 打开分类列表和编辑抽屉，确认不默认展示层级字段。 |
| PC-CAT-006 | 新增或编辑时显示 `路径` 输入框，用户不知道该填什么 | 当前 `categoryPath` 在表单中只读，且不在列表默认展示；后端按父级分类和编码生成路径。 | 新增/编辑分类，确认路径不可编辑；保存后详情路径由系统生成。 |
