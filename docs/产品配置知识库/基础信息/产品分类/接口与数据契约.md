## 数据表

| 表名 | 用途 |
| --- | --- |
| `pc_category` | 产品分类主表，保存分类树节点 |
| `pc_sales_product` | 销售产品保存分类编码、名称快照 |
| `pc_engineering_plan` | 工程方案保存产品系列或分类快照 |

## 关键字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `category_id` | bigint | 是 | 产品分类内部 ID，用户不应直接填写 |
| `tenant_id` | bigint | 是 | 租户 ID |
| `parent_id` | bigint | 否 | 父分类 ID，应由上级分类选择器写入 |
| `category_code` | varchar(80) | 是 | 产品分类编码，同一租户下未删除数据唯一 |
| `category_name_cn` | varchar(200) | 是 | 产品分类中文名称 |
| `category_name_en` | varchar(200) | 否 | 产品分类英文名称 |
| `business_type` | varchar(80) | 否 | 业务口径类型，第一阶段可为空 |
| `category_level` | integer | 否 | 分类层级，系统推导字段，不建议普通用户维护 |
| `category_path` | varchar(500) | 否 | 分类路径编码，由系统根据上级分类和分类编码生成，不建议普通用户维护 |
| `status` | varchar(20) | 是 | `ENABLED` 或 `DISABLED` |
| `sort_order` | integer | 否 | 同级排序 |
| `remark` | varchar(500) | 否 | 备注 |
| `del_flag` | varchar(1) | 是 | 逻辑删除标记 |

## 前端业务展示要求

产品分类应该按树形结构展示，而不是普通平铺表格。

建议展示列：

| 列 | 说明 |
| --- | --- |
| 分类名称 | 树形主列，显示层级缩进 |
| 分类编码 | 辅助识别 |
| 分类英文名称 | 英文环境辅助 |
| 状态 | 启用 / 停用 |
| 排序 | 同级排序 |
| 操作 | 详情、编辑、删除、引用检查 |

不建议默认展示：

- `父级ID`
- `层级`
- `路径`
- `tenant_id`
- `del_flag`
- 审计字段

## 表单业务要求

新增和编辑表单应该展示：

| 页面字段 | 数据来源 | 说明 |
| --- | --- | --- |
| 上级分类 | `/product-capability/categories/tree` | 树选择器，显示分类名称，不显示 ID |
| 分类编码 | 用户录入 | 稳定业务编码 |
| 分类名称 | 用户录入 | 中文名称 |
| 分类英文名称 | 用户录入 | 英文名称 |
| 排序 | 用户录入 | 同级排序 |
| 备注 | 用户录入 | 维护说明 |

不应该让用户直接填写：

- `parentId`
- `categoryLevel`
- `categoryPath`
- `categoryId`

路径生成建议：

```text
顶级分类路径 = 当前分类编码
子级分类路径 = 上级分类路径 + "/" + 当前分类编码
```

示例：

```text
WINDOW_COVERING/ROLLER_SHADE/MOTORIZED_ROLLER_SHADE
```

## 选择器来源

| 页面字段 | 数据来源 | 过滤条件 |
| --- | --- | --- |
| 上级分类 | `/product-capability/categories/tree` | 默认只展示启用分类；编辑时排除自己和自己的下级 |
| 销售产品分类 | `/product-capability/categories/options` 或 `/tree` | 默认只展示启用分类 |
| 工程方案产品系列 | `/product-capability/categories/options` 或 `/tree` | 默认只展示启用分类 |

## 状态和字典

| 字段 | 来源 | 可选值 |
| --- | --- | --- |
| `status` | 系统通用状态 | `ENABLED`、`DISABLED` |

## 校验规则

- `categoryCode` 必填。
- `categoryNameCn` 必填。
- 同一租户下未删除分类的 `categoryCode` 不能重复。
- 不能选择自己作为上级分类。
- 不能选择自己的下级作为上级分类。
- 删除前应检查下级分类和业务引用。
- 有引用的分类建议停用，不建议删除。

## 请求示例

新增二级分类：

```json
{
  "parentId": 100001,
  "categoryCode": "HONEYCOMB_SHADE",
  "categoryNameCn": "蜂巢帘",
  "categoryNameEn": "Honeycomb Shade",
  "status": "ENABLED",
  "sortOrder": 15,
  "remark": "蜂巢帘产品系列"
}
```

修改状态：

```http
PUT /product-capability/categories/change-status/100003/DISABLED
```

## 响应示例

分类树节点：

```json
{
  "categoryId": 100001,
  "parentId": 0,
  "categoryCode": "WINDOW_COVERING",
  "categoryNameCn": "窗饰产品",
  "categoryNameEn": "Window Covering",
  "categoryLevel": 1,
  "categoryPath": "WINDOW_COVERING",
  "status": "ENABLED",
  "sortOrder": 1
}
```

说明：当前后端 `/tree` 返回列表数据，前端应根据 `parentId` 组装树；或后续后端直接返回 `children` 结构。

## 常见错误

| 错误场景 | 后端或页面表现 | 用户提示 |
| --- | --- | --- |
| 分类编码重复 | 保存失败 | 分类编码已存在，请换一个编码或编辑已有分类 |
| 父级分类填错 | 分类挂到错误节点 | 请从上级分类树里选择，不要手填 ID |
| 删除被引用分类 | 删除失败或影响后续资料 | 该分类已被引用，建议停用 |
| 选择自己的下级作为父级 | 形成循环层级 | 上级分类不能选择自己或自己的下级 |

## AI 工具调用契约

### 意图

```text
新增产品分类
编辑产品分类
停用产品分类
查询分类引用
```

### 必填槽位

- `categoryCode`
- `categoryNameCn`
- `parentCategory` 或 `isRootCategory`

### 可推断字段

- `status = ENABLED`
- `sortOrder` 可按同级最大排序值加 10 建议。
- `categoryNameEn` 可留空，后续人工补充。

### 禁止暴露字段

- `categoryId`
- `parentId`
- `tenantId`
- `categoryLevel`
- `categoryPath`
- `delFlag`
- `createBy`
- `createTime`
- `updateBy`
- `updateTime`

### 保存前确认格式

```text
将新增产品分类：
- 上级分类：窗饰产品
- 分类编码：HONEYCOMB_SHADE
- 分类名称：蜂巢帘
- 状态：启用
是否确认保存？
```
