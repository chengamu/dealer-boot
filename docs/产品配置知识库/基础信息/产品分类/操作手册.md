# 产品分类操作手册

## 功能定位

产品分类用于维护产品能力体系里的分类树，是基础信息的入口层级。

它不是销售产品，也不是具体物料。它负责把物料管理和后续已确认业务模块归到正确业务范围里，方便录入、筛选、授权、检查和后续 AI 代办识别上下文。

业务顺序：

```text
产品分类 -> 物料管理 -> 配方管理 / 已确认的后续业务模块
```

## 使用对象

- 主要使用人：技术科、产品资料维护人员。
- 辅助使用人：实施人员、测试人员。
- 不建议使用人：普通销售、客户、非资料维护人员。

## 这个功能维护什么

产品分类应该维护成树形结构，例如：

```text
窗饰产品
  基础卷帘样本
  电动卷帘样本
  蜂巢帘
  柔纱帘样本
  户外遮阳样本
  轨道配件样本
```

一级分类用于表达大业务域，二级或三级分类用于表达具体产品系列或样本分类。

## 使用前准备

使用本功能前，需要先明确：

- 业务上有哪些产品大类。
- 哪些分类要作为后续业务的产品系列使用。
- 哪些分类只是辅助归档，不应该被后续业务直接引用。
- 分类编码命名规则，例如英文大写和下划线。

## 推荐录入顺序

1. 先维护一级分类，例如 `WINDOW_COVERING / 窗饰产品`。
2. 再在一级分类下维护二级分类，例如 `HONEYCOMB_SHADE / 蜂巢帘`。
3. 后续在物料管理、配方管理或已确认业务模块中引用这些分类。

## 页面入口

- 菜单：基础信息 / 产品分类
- 路由：`/product-master/categories`

## 字段说明

| 字段 | 是否必填 | 怎么填写 | 注意事项 |
| --- | --- | --- | --- |
| 分类编码 | 是 | 使用稳定业务编码，例如 `HONEYCOMB_SHADE` | 保存后会被后续模块引用，不建议随意修改 |
| 分类名称 | 是 | 中文业务名称，例如 `蜂巢帘` | 给技术科和业务人员识别 |
| 分类英文名称 | 否 | 英文名称，例如 `Honeycomb Shade` | 用于英文环境和后续跨语言展示 |
| 上级分类 | 否 | 从分类树里选择上级分类 | 不应该手填父级 ID |
| 路径 | 否 | 系统根据上级分类和分类编码自动生成 | 不应该手填；详情中只读展示即可 |
| 状态 | 是 | 启用或停用 | 停用后不应再作为新资料选择项 |
| 排序 | 否 | 数字越小越靠前 | 只影响展示顺序 |
| 备注 | 否 | 说明用途或维护口径 | 不要写临时沟通内容 |

## 常用操作

### 新增分类

1. 点击新增。
2. 选择上级分类。如果是一级分类，则选择顶级或留空，具体以页面设计为准。
3. 填写分类编码、中文名称、英文名称。
4. 设置排序。
5. 保存。

### 编辑分类

用于修正名称、英文名、排序和备注。

如果分类已经被配方管理或后续业务引用，修改编码或层级会影响业务识别，必须先做引用检查。

### 停用分类

停用适用于不再新增引用的分类。

停用后，历史数据仍应保留，但新增物料、配方或后续业务资料时不应再选择该分类。

### 删除分类

删除前必须确认没有下级分类和业务引用。

有引用时不建议删除，应优先停用。

## 对后续业务的影响

产品分类会影响：

- 物料管理：用于物料归类和检索。
- 配方管理：配方主表会保存分类编码和分类名称快照。
- 后续已确认业务模块：使用分类辅助筛选、授权和数据归档。
- AI 代办：AI 需要根据用户说的“蜂巢帘、卷帘、柔纱帘”等词识别分类上下文。

## 常见问题

| 问题 | 原因 | 处理方式 |
| --- | --- | --- |
| 不知道父级 ID 填什么 | 页面暴露了内部字段 | 后续应改成上级分类树选择器 |
| 分类列表看不出父子关系 | 当前是平铺表格 | 后续应改成树形表格 |
| 层级字段看不懂 | 层级是系统推导信息 | 不应作为普通用户关注字段 |
| 路径字段看不懂 | 路径是系统根据分类树生成的内部定位信息 | 不应让用户填写，详情只读展示即可 |
| 删除失败 | 分类被后续资料引用 | 改为停用，或先处理引用 |

## 如何确认维护成功

- 分类能在树里显示到正确父节点下面。
- 排序位置符合预期。
- 启用分类能在后续物料、配方或已确认业务模块的选择器中使用。
- 停用分类不再出现在新增业务选择器中。

## AI 代办能力

### 支持的意图

```text
意图名称：新增产品分类
用户说法：帮我在窗饰产品下面新增一个蜂巢帘分类
```

```text
意图名称：停用产品分类
用户说法：把户外遮阳样本这个分类停用
```

```text
意图名称：查询分类引用
用户说法：这个蜂巢帘分类现在被哪些地方用了
```

### 必填槽位

- 分类编码。
- 分类中文名称。
- 上级分类或是否顶级分类。

### 可推断字段

- 状态默认启用。
- 排序可根据同级分类末尾自动建议。
- 分类英文名称可由用户补充，也可以先留空。

### 需要追问

- 用户只说“新增蜂巢帘”，但没有说明上级分类时，需要追问挂在哪个分类下面。
- 用户提供的分类名称已存在时，需要追问是新建、编辑还是停用已有分类。
- 用户要求删除已被引用的分类时，需要提示风险并建议停用。

### 保存前确认摘要

```text
将新增产品分类：
- 上级分类：窗饰产品
- 分类编码：HONEYCOMB_SHADE
- 分类名称：蜂巢帘
- 状态：启用
是否确认保存？
```
