# 基础字典操作手册

## 功能定位

基础字典用于维护产品配置业务里的简单下拉枚举，例如业务类型、资料类型、工程规则类型和配置输入类型。

物料类型和属性分组在“物料类型”模块维护；具体单位在“单位管理”维护。

## 录入前准备

基础字典分两层：

- 字典类型：例如 `product_business_type`、`engineering_rule_type`。
- 字典项：例如 `PURCHASE`、`DISABLE`、`BLOCK`。

“上级字典项”只用于同一个字典类型内部存在父子层级或级联关系的场景。多数基础字典不需要上级字典项，应保持为空。

## 推荐维护顺序

1. 进入 `基础信息 / 基础字典`。
2. 先确认左侧字典类型是否存在。
3. 选择字典类型后，在右侧维护字典项。
4. 新增字典项时填写业务值、中文标签、英文标签、排序和状态。
5. 停用不再使用的字典项，停用后业务下拉不再返回该项。

## 关键字段说明

| 字段 | 说明 |
| --- | --- |
| 类型编码 | 稳定业务编码，前端和后端按它读取 options。 |
| 类型中文名称 / 英文名称 | 字典类型展示名称。 |
| 业务域 | `BASE`、`ENGINEERING`、`CONFIG`，用于区分基础信息、工程配置、产品配置。 |
| 字典值 | 业务保存值，必须稳定。 |
| 中文标签 / 英文标签 | 下拉展示文案。 |
| 上级字典项 | 可选字段，只在有父子层级的字典类型中使用。 |
| 系统内置 | 种子内置项，不能删除。 |
| 可编辑 | 控制是否允许维护基础信息。 |
| 状态 | `ENABLED` 返回到下拉，`DISABLED` 不返回到下拉。 |

## 常见错误

- 物料类型和属性分组到“物料类型”模块维护。
- 具体单位到“单位管理”维护。
- 产品业务枚举不要写到 `sys_dict_type` / `sys_dict_data`。
- 不要删除系统内置字典项；如业务不再使用，优先停用。
- 修改字典值前先确认已有业务数据是否保存了该值。

## AI 代办能力

### 支持的意图

```text
意图名称：新增字典项
用户说法：在资料类型里加一个安装说明，值是 INSTALL_GUIDE。
```

```text
意图名称：停用字典项
用户说法：把配置输入类型里的旧选项停用。
```

### 必填槽位

- 字典类型。
- 字典值。
- 中文标签。

### 可推断字段

- 英文标签未提供时不自动翻译，需追问或留空。
- 状态默认启用。
- 业务域可从已选字典类型继承。

### 需要追问

- 用户没有说明字典类型时必须追问。
- 字典值缺失时必须追问。
- 用户要新增字典类型但没有说明业务域时必须追问。

### 保存前确认摘要

```text
将新增字典项：
- 字典类型：product_asset_type
- 字典值：INSTALL_GUIDE
- 中文标签：安装说明
- 状态：启用
是否确认保存？
```
