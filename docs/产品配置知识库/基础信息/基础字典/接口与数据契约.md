## 校验规则

- 字典类型编码在同一租户下唯一。
- 同一字典类型下字典项值唯一。
- 启用状态资料不能直接编辑，先停用再编辑。
- 被业务引用的字典类型或字典项不能删除。
- `parent_value` 只表示同一字典类型内部的上级字典项。

## AI 工具调用契约

### 意图

```text
意图名称：新增字典类型
用户说法：新增一个资料类型字典
适用页面：基础信息 / 基础字典
调用接口：POST /product-capability/product-dict-types
权限要求：product:dict:add
```

```text
意图名称：新增字典项
用户说法：在资料类型里加一个安装说明，值是 INSTALL_GUIDE
适用页面：基础信息 / 基础字典
调用接口：POST /product-capability/product-dict-items
权限要求：product:dict:add
```

### 必填槽位

新增字典类型：

- `typeCode`：类型编码。
- `typeNameCn`：类型中文名称。
- `businessDomain`：业务域，取值 `BASE`、`ENGINEERING`、`CONFIG`。

新增字典项：

- `typeCode` 或可唯一识别的字典类型。
- `itemValue`：字典值。
- `itemLabelCn`：中文标签。

### 可推断字段

- 字典项状态默认 `ENABLED`。
- 字典项业务域跟随所属字典类型。
- `parentValue` 默认空，除非用户明确说明父子层级。

### 需要追问

- 字典类型无法唯一定位。
- 新增字典类型时缺少业务域。
- 新增字典项时缺少字典值或中文标签。
- 用户要求修改系统内置或不可编辑资料。

### Payload 边界

- AI 不提交 `tenantId`、`createBy`、`createTime`、`updateBy`、`updateTime`、`delFlag`。
- AI 不让用户填写内部 ID。
- 编辑前必须先调用对应 `edit-check`。
- AI 不把产品业务枚举写入系统字典接口。

### 禁止暴露字段

- `dictTypeId`
- `dictItemId`
- `tenantId`
- `createBy`
- `createTime`
- `updateBy`
- `updateTime`
- `delFlag`

### 保存前确认格式

```text
将新增字典项：
- 字典类型：product_asset_type
- 字典值：INSTALL_GUIDE
- 中文标签：安装说明
- 英文标签：未填写
- 状态：启用
是否确认保存？
```

### 失败提示

| 场景 | 用户提示 |
| --- | --- |
| 字典类型缺失 | 请说明要维护哪个字典类型。 |
| 字典类型不存在 | 没找到这个字典类型，请先新增字典类型。 |
| 字典值缺失 | 请提供稳定的字典值。 |
| 字典值重复 | 该字典类型下已存在相同字典值，请换一个值或编辑已有字典项。 |
| 系统内置不可删除 | 这是系统内置字典，不能删除；如不再使用，请停用。 |
