# 基础属性操作手册

## 功能定位

基础属性用于定义“物料可以维护哪些结构化属性”。它不是产品分类，也不是产品字典。

```text
产品字典 -> 定义下拉值
基础属性 -> 定义属性字段
物料属性 -> 给某个物料填写属性值
产品工程设计 -> 后续用这些属性做可选范围、能力规则和检查
```

例如：

| 物料 | 基础属性定义 | 物料属性值 |
| --- | --- | --- |
| 遥控器 | 通道数 | 1 路、6 路 |
| 面料 | 厚度 | 0.55 mm |
| 面料 | 颜色 | 米白 |
| 电机 | 系统类型 | 奥克电机、杜亚电机 |

基础属性解决的是“字段标准化”问题。没有它，物料属性会变成每个人随便写字段名，后续工程规则无法稳定判断。

## 使用对象

- 技术科：定义哪些物料属性需要结构化维护。
- 资料维护人员：在“物料属性”里按照这些定义填写具体物料的属性值。
- 开发和测试：确认工程规则、预览、检查读取的属性字段是否稳定。
- AI 代办能力：自动录入物料时根据基础属性判断还缺哪些属性值。

## 使用前准备

使用本功能前，需要先维护：

- 产品字典：物料类型、属性分组等业务枚举。
- 单位管理：长度、重量、面积等具体单位。
- 物料资料：后续在“物料属性”里给具体物料填值。

## 推荐录入顺序

1. 先确认属性适用于哪个物料类型，例如面料、遥控器、电机。
2. 再确认属性值类型，例如文本、数字、开关、选项。
3. 如果是数字型测量属性，再选择默认单位。
4. 保存基础属性定义。
5. 到“物料属性”页面给具体物料维护属性值。

## 页面入口

- 菜单：基础信息 / 基础属性
- 路由：`/product-master/base-attributes`

## 字段说明

| 字段 | 是否必填 | 怎么填写 | 注意事项 |
| --- | --- | --- | --- |
| 属性分组 | 是 | 从属性分组字典选择，例如 `FABRIC`、`CONTROL`、`SIZE`。 | 不建议手填自由文本。 |
| 属性编码 | 是 | 稳定英文编码，例如 `THICKNESS`、`CHANNEL_COUNT`。 | 保存后不要随意修改，工程规则会引用。 |
| 属性名称 | 是 | 中文展示名，例如“厚度”“通道数”。 | 给业务人员看。 |
| 属性英文名称 | 否 | 英文展示名。 | 用于英文环境。 |
| 值类型 | 是 | 从当前固定选项中选择：文本、数字、开关、枚举。 | 决定“物料属性”页面显示什么录入控件。 |
| 默认单位 | 否 | 仅数字型测量属性需要，例如厚度用 `MM`。 | 颜色、系统类型、通道数通常不需要单位。 |
| 适用物料类型 | 是 | 选择一个或多个物料类型。 | 不建议手填编码。 |
| 扩展配置 | 否 | 预留给高级配置。 | 普通业务人员不应直接填写 JSON。 |
| 状态 | 是 | 启用或停用。 | 停用后不应再作为新物料属性选项。 |
| 排序 | 否 | 数字越小越靠前。 | 影响录入展示顺序。 |
| 备注 | 否 | 说明属性用途。 | 建议写给维护人员看的业务说明。 |

## 常用操作

### 新增

1. 点击新增。
2. 选择属性分组和适用物料类型。
3. 填写属性编码、中文名称和值类型。
4. 如果值类型是数字且确实有计量意义，再选择默认单位。
5. 保存。

### 编辑

已经被物料属性或工程规则引用的属性，不建议修改属性编码和值类型。需要调整展示名称、排序、备注时可以编辑。

### 删除或停用

如果属性已经被物料属性引用，优先停用，不要删除。删除前需要做引用检查。

### 查询

推荐按属性分组、属性编码、属性名称和状态查询。

## 常见问题

| 问题 | 原因 | 处理方式 |
| --- | --- | --- |
| 为什么有字典还要基础属性？ | 字典只管下拉值，基础属性定义物料有哪些字段。 | 例如 `product_material_type` 里有“面料”，但“面料厚度”这个字段要由基础属性定义。 |
| 为什么颜色不应该选单位？ | 颜色是文本或选项，不是计量值。 | 值类型选文本或选项，默认单位留空。 |
| 通道数要不要单位？ | 通道数是数量属性，但通常业务展示为“路”，不是采购/库存单位。 | 第一阶段可以不填单位，或后续补充专用展示单位。 |
| 扩展 JSON 怎么填？ | 普通维护不应该直接写 JSON。 | 需要通过结构化控件或高级配置面板维护。 |

## 如何确认维护成功

- 基础属性列表能查到新增属性。
- “物料属性”页面选择物料后，可以选择该属性并填写对应类型的值。
- 工程规则后续选择属性条件时，可以看到该属性。

## AI 代办能力

### 支持的意图

```text
意图名称：新增基础属性
用户说法：给遥控器加一个通道数属性，数字类型。
```

### 必填槽位

- 属性分组
- 属性编码
- 属性中文名称
- 值类型
- 适用物料类型

### 可推断字段

- 状态默认启用。
- 排序可以按当前分组最大排序值递增。
- 数字型属性如果用户没有说明单位，不自动乱选单位。

### 需要追问

- 属性编码缺失时需要追问。
- 数字型属性是否需要默认单位不明确时需要追问。
- 选项型属性的可选值来源不明确时需要追问。

### 保存前确认摘要

```text
将新增基础属性：
属性分组：CONTROL
属性编码：CHANNEL_COUNT
属性名称：通道数
值类型：数字
适用物料类型：遥控器
默认单位：无
```
