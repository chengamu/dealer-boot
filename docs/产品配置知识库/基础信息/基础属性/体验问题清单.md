| PC-ATTR-001 | 页面没有解释“基础属性”与产品字典、产品分类、物料属性的关系，业务人员不知道这个功能做什么。 | 基础属性页面人工测试 | 页面标题区说明：基础属性是物料属性字段定义，实际属性值在“物料属性”维护，后续工程规则会使用。 | P0 | 基础属性、物料属性、工程规则 |
| PC-ATTR-002 | “属性分组”是普通输入框，业务人员需要手写 `CONTROL`。 | 基础属性编辑抽屉人工测试 | 改为产品字典选择器，例如 `product_attribute_group`；展示中文名称，保存稳定编码。 | P1 | 基础属性录入、AI 代办 |
| PC-ATTR-003 | “适用物料类型”是普通输入框，业务人员需要手写 `REMOTE`。 | 基础属性编辑抽屉人工测试 | 改为物料类型多选，来源 `product_material_type`；展示中文标签，保存物料类型编码集合。 | P1 | 基础属性录入、物料属性筛选 |
| PC-ATTR-004 | “默认单位”对所有值类型都展示，颜色等非计量属性也能选米、个、千克。 | 基础属性编辑抽屉人工测试 | 仅值类型为数字且属性具备计量语义时显示默认单位；单位按长度、重量、面积等类型过滤。 | P1 | 基础属性录入、物料属性录入 |
| PC-ATTR-005 | 值类型显示为英文枚举，页面没有解释文本、数字、开关、选项分别影响什么。 | 基础属性编辑抽屉人工测试 | 值类型使用业务化中文标签，并提示它会决定“物料属性”录入控件。 | P2 | 基础属性录入 |
| PC-ATTR-006 | 直接暴露“扩展 JSON”，业务人员需要写 JSON。 | 基础属性编辑抽屉人工测试 | 普通模式隐藏 raw JSON；需要扩展配置时提供结构化配置器或高级模式，至少要有 JSON 校验和解析提示。 | P0 | 基础属性录入、数据质量 |
| PC-ATTR-007 | 抽屉字段缺少业务分组，所有字段平铺，难以判断哪些必填、哪些高级。 | 基础属性编辑抽屉人工测试 | 分为“字段定义”“适用范围”“录入控制”“高级配置/备注”。 | P2 | 基础属性录入体验 |
| PC-ATTR-008 | 页面没有告诉用户这个属性会被哪些功能使用。 | 基础属性页面人工测试 | 详情或帮助区展示引用场景：物料属性、工程能力规则、工程检查、AI 自动录入。 | P2 | 基础属性、工程设计 |
| PC-ATTR-009 | 基础属性没有承接老 ERP“类型参数设置”的心智，用户不知道它就是按物料类型定义参数模板。 | 老 ERP 截图对比分析 | 页面和文档应改口径为“物料属性定义”；支持按物料类型维护字段模板，例如面料显示颜色、门幅、厚度，铝材显示规格、颜色、厚度，系统显示系统类型、规格等。 | P0 | 基础属性、物料资料、物料属性 |
