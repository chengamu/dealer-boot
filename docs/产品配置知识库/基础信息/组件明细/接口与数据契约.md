# 组件明细接口与数据契约

## 页面与权限

| 项目 | 内容 |
| --- | --- |
| 菜单名称 | 基础信息 / 组件明细 |
| 前端路由 | `/product-master/component-items`、`/product-master/components?tab=componentItem` |
| 前端组件 | `admin-ui/src/pages/product-center/base/ProductBaseInfoPage.vue` 的 `componentItem` 配置 |
| 通用 grid 组件 | `admin-ui/src/pages/product-center/components/ProductEntityGridPage.vue` |
| 批量录入组件 | `admin-ui/src/pages/product-center/components/ProductComponentItemBatchEntry.vue` |
| 前端 API | `admin-ui/src/api/product-capability/component.ts` |
| Controller | `ProductComponentController` |
| Service | `ProductComponentItemService` |
| 权限码 | `product:component-item:list`、`product:component-item:add`、`product:component-item:edit`、`product:component-item:remove`、`product:component-item:reference` |

## 后端接口

| 动作 | 方法 | 路径 | 权限 |
| --- | --- | --- | --- |
| 列表 | GET | `/product-capability/component-items/list` | `product:component-item:list` |
| 选项 | GET | `/product-capability/component-items/options` | `product:component-item:list` |
| 详情 | GET | `/product-capability/component-items/{id}` | `product:component-item:list` |
| 修改前检查 | GET | `/product-capability/component-items/{id}/edit-check` | `product:component-item:edit` |
| 新增 | POST | `/product-capability/component-items` | `product:component-item:add` |
| 修改 | PUT | `/product-capability/component-items` | `product:component-item:edit` |
| 启停 | PUT | `/product-capability/component-items/change-status/{id}/{status}` | `product:component-item:edit` |
| 删除 | DELETE | `/product-capability/component-items/{ids}` | `product:component-item:remove` |
| 引用检查 | GET | `/product-capability/component-items/{id}/references` | `product:component-item:reference` |

## 数据表

| 表名 | 用途 |
| --- | --- |
| `pc_component` | 组件包主表 |
| `pc_component_item` | 组件明细表，保存组件包和物料的组成关系 |
| `pc_material` | 明细引用的统一物料主档 |
| `pc_unit` | 明细数量单位来源 |

## 关键字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `componentItemId` | number | 修改时必填 | 组件明细 ID |
| `componentId` | number | 否 | 组件包 ID |
| `componentCode` | string | 是 | 组件包编码 |
| `materialId` | number | 否 | 物料 ID |
| `materialCode` | string | 是 | 物料编码 |
| `materialNameCn` | string | 否 | 物料中文名称快照 |
| `qtyFormula` | string | 否 | 数量公式文本 |
| `defaultQty` | number | 否 | 默认数量 |
| `unitCode` | string | 否 | 单位编码 |
| `requiredFlag` | string/boolean | 否 | 是否必需，保存时可转为 `1` / `0` |
| `sortOrder` | number | 否 | 排序 |
| `status` | string | 是 | `ENABLED` / `DISABLED` |

## 选择器来源

| 页面字段 | 数据来源 | 过滤条件 |
| --- | --- | --- |
| 组件编码 | `GET /product-capability/components/options` | `status=ENABLED` |
| 物料编码 | `GET /product-capability/materials/options` | 默认 `status=ENABLED`，可按物料类型过滤 |
| 单位编码 | `GET /product-capability/units/options` | 页面初始化读取启用单位 |

## 状态和字典

| 字段 | 来源 | 可选值 |
| --- | --- | --- |
| `status` | 基础资料状态规则 | `ENABLED`、`DISABLED` |
| `requiredFlag` | 页面布尔控件 | `1`、`0` 或布尔值 |

## 校验规则

- 新增组件明细必须提供组件编码和物料编码。
- 普通修改前调用 `edit-check`。
- 后端普通修改会校验当前状态；启用状态不可普通编辑。
- 批量录入保存时逐行调用新增或修改接口。

## 请求示例

```json
{
  "componentCode": "PACK_MOTOR",
  "materialCode": "MOTOR_001",
  "materialNameCn": "标准电机",
  "defaultQty": 1,
  "unitCode": "PCS",
  "requiredFlag": "1",
  "sortOrder": 10,
  "status": "ENABLED"
}
```

## 响应示例

```json
{
  "code": 200,
  "msg": "操作成功"
}
```

## 常见错误

| 错误场景 | 后端提示 | 用户提示 |
| --- | --- | --- |
| 主键缺失 | `{gen.validation.pk.required}` | 请选择要操作的组件明细。 |
| 启用状态普通修改 | 当前资料已启用，不可直接修改 | 请先停用后再编辑。 |
| 缺少组件或物料 | 参数校验失败 | 请补充组件编码和物料编码。 |

## AI 工具调用契约

### 意图

```text
新增组件明细
编辑组件明细
停用组件明细
查询组件明细
```

### 调用接口和权限

| 意图 | 方法 | 路径 | 权限 |
| --- | --- | --- | --- |
| 新增组件明细 | POST | `/product-capability/component-items` | `product:component-item:add` |
| 编辑组件明细 | PUT | `/product-capability/component-items` | `product:component-item:edit` |
| 停用组件明细 | PUT | `/product-capability/component-items/change-status/{id}/{status}` | `product:component-item:edit` |
| 查询组件明细 | GET | `/product-capability/component-items/list` | `product:component-item:list` |

### 必填槽位

- componentCode
- materialCode

### 可选槽位

- qtyFormula
- defaultQty
- unitCode
- requiredFlag
- sortOrder
- status

### 可推断字段

- `status` 默认 `ENABLED`。
- `requiredFlag` 默认 `0`。
- `defaultQty` 默认 1。
- `unitCode` 可从物料管理的 `usageUnitCode` 带出。

### 需要追问

- 组件编码缺失。
- 物料编码缺失。
- 组件或物料存在多个候选。

### Payload 边界

- 只调用已有后端接口，不直接写数据库。
- 不要求用户提供内部 ID、审计字段、租户字段或 JSON。
- 批量新增时仍按多次单条接口调用，除非后续补充后端批量接口。

### 禁止暴露字段

- componentItemId
- componentId
- materialId
- createBy
- createTime
- updateBy
- updateTime

### 保存前确认格式

```text
请确认组件明细：
- 组件编码：componentCode
- 物料编码：materialCode
- 默认数量：defaultQty unitCode
- 是否必需：requiredFlag
确认后保存。
```

### 失败提示

| 场景 | 给用户的提示 |
| --- | --- |
| 组件不存在 | 没找到该组件包，请先维护组件包。 |
| 物料不存在 | 没找到该物料，请先维护物料管理。 |
| 当前资料不可编辑 | 当前组件明细已启用，请先停用后再编辑。 |
