# 资料资产操作手册

## 功能定位

资料资产用于维护产品相关图片、规格书、安装说明、色卡、参考资料等业务资料 metadata，以及这些资料绑定到物料等业务对象的关系。

```text
系统 OSS -> 资料资产 -> 资料绑定 -> 物料 / 后续发布检查
```

文件本体复用系统 OSS，本功能不重新实现文件存储。

## 使用对象

- 主要使用人：产品资料维护人员、技术科。
- 辅助使用人：测试人员、实施人员。
- 不建议使用人：只处理报价或订单的用户。

## 使用前准备

使用本功能前，需要先维护：

- 基础字典：资料类型 `product_asset_type`。
- 业务对象：物料管理。
- OSS 配置：系统上传接口可用。

## 推荐录入顺序

1. 先维护业务对象，例如物料。
2. 在物料详情中上传附件，或先在资料资产中维护资产 metadata。
3. 建立资料绑定，指定目标类型、目标编码、用途、可见性和排序。
4. 后续发布检查或销售展示按绑定关系读取资料。

## 页面入口

- 菜单：基础信息 / 资料资产、基础信息 / 资料绑定。
- 路由：`/product-master/media-assets`、`/product-master/media-bindings`。
- 也可在支持附件的业务页面抽屉中上传附件。

## 字段说明

### 资料资产

| 字段 | 是否必填 | 怎么填写 | 注意事项 |
| --- | --- | --- | --- |
| 资产编码 | 是 | 稳定唯一编码 | 通用上传会自动生成 `ASSET_目标类型_目标ID_时间戳`。 |
| 中文名称 | 是 | 文件或业务资料名称 | 便于业务人员识别。 |
| 英文名称 | 否 | 英文资料名 | 可与中文名称相同。 |
| 资料类型 | 否 | 从基础字典选择 | 例如图片、文件等。 |
| 用途 | 否 | 安装说明、规格书、色卡、参考资料 | 当前页面固定枚举。 |
| 语言 | 否 | 语言编码 | 通用上传使用当前界面语言。 |
| 可见性 | 否 | 内部 / 公开 | 默认内部。 |
| OSS ID | 否 | 上传后返回 | 文件本体在 `sys_oss`。 |
| URL | 否 | 上传后返回或手填 | 用于打开资料。 |
| 版本号 | 否 | 数字 | 通用上传默认 1。 |
| 状态 | 是 | 启用 / 停用 | 启用后不可普通编辑。 |

### 资料绑定

| 字段 | 是否必填 | 怎么填写 | 注意事项 |
| --- | --- | --- | --- |
| 资产 ID | 是 | 资料资产 ID | 当前页面为数字字段，后续可优化为选择器。 |
| 资产编码 | 是 | 资料资产编码 | 用于识别绑定资料。 |
| 目标类型 | 是 | 业务对象类型 | 当前基础信息附件统一填 `MATERIAL`。 |
| 目标 ID | 否 | 业务对象 ID | 通用附件上传会写入。 |
| 目标编码 | 是 | 业务对象编码 | 当前基础信息附件使用物料编码。 |
| 用途 | 否 | 安装说明、规格书、色卡、参考资料 | 与资产用途保持一致。 |
| 可见性 | 否 | 内部 / 公开 | 默认内部。 |
| 语言 | 否 | 语言编码 | 便于多语言资料区分。 |
| 发布必需 | 否 | 是 / 否 | 后续发布检查可使用。 |
| 排序 | 否 | 数字 | 同一目标下展示顺序。 |
| 状态 | 是 | 启用 / 停用 | 停用后不应进入新增业务使用。 |

## 常用操作

### 上传附件

在物料等支持附件的编辑抽屉中点击上传：

1. 页面调用 `/system/oss/upload` 上传文件本体。
2. 上传成功后新增资料资产。
3. 再新增资料绑定。
4. 页面刷新附件列表。

当前上传链路不是后端事务接口。如果资料资产或绑定创建失败，可能留下孤立 OSS 文件或孤立资料资产，需要人工测试确认实际表现。

### 手工新增资料资产

手工维护资料资产时，需要填写资产编码、名称、类型、用途、OSS ID 或 URL 等信息。

### 手工新增资料绑定

手工绑定时，需要填写资产、目标类型、目标编码、用途、可见性和排序。

### 编辑

资料资产编辑前会调用修改前检查。资料绑定当前没有单独 edit-check 接口，后续需要确认是否补齐。

### 删除或停用

资料资产已被绑定时不建议删除，应优先停用。资料绑定可删除，但需要确认是否会影响发布检查和销售展示。

### 查询

资料资产推荐按资产编码、名称、类型、用途和状态查询。资料绑定推荐按资产编码、目标类型、目标编码和状态查询。

## 常见问题

| 问题 | 原因 | 处理方式 |
| --- | --- | --- |
| 上传后看不到附件 | 可能 OSS 上传成功但资产或绑定创建失败 | 到资料资产和资料绑定页分别检查。 |
| 打开资料失败 | URL 缺失或 OSS 不可访问 | 检查 `ossId`、URL 和 OSS 配置。 |
| 不知道目标类型怎么填 | 目标类型对应业务对象 | 当前基础信息附件填 `MATERIAL`。 |

## 如何确认维护成功

- 资料资产列表能查询到资产。
- 资料绑定列表能查询到目标对象对应绑定。
- 在物料详情中能看到并打开附件。

## AI 代办能力

### 支持的意图

```text
意图名称：绑定资料资产
用户说法：把安装说明 ASSET_INSTALL_001 绑定到物料 MATERIAL_001，作为安装说明。
```

### 调用入口

- 页面：`/product-master/media-bindings`
- 接口：`POST /product-capability/media-bindings`
- 权限：`product:asset:bind`

### 必填槽位

- assetCode
- targetType
- targetCode

### 可选槽位

- usageType
- visibility
- languageCode
- requiredForPublish
- sortOrder
- status

### 可推断字段

- 状态默认启用。
- 可见性默认内部。
- 语言可使用当前用户语言。
- 用途可从用户说法推断为安装说明、规格书、色卡或参考资料。

### 需要追问

- 缺少资产编码。
- 缺少目标类型或目标编码。
- 目标类型不属于已支持业务对象。

### 失败提示

| 场景 | 给用户的提示 |
| --- | --- |
| 资料资产不存在 | 请先上传或维护资料资产。 |
| 目标对象不存在 | 请确认要绑定的物料是否已维护。 |
| 上传失败 | 文件上传失败，请检查文件大小和 OSS 配置。 |

### 保存前确认摘要

```text
将绑定资料：
- 资产：assetCode
- 目标：targetType / targetCode
- 用途：usageType
- 可见性：visibility
确认后保存。
```
