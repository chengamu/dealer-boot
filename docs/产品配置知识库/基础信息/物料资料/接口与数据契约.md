## 数据表

| 表名 | 用途 |
| --- | --- |
| `pc_material` | 统一物料主档，保存面料、铝材、系统、配件、包装件、标准成品等基础资料 |
| `pc_material_attribute` | 物料属性值表，保存某条物料在某个属性定义下的具体值 |
| `pc_base_attribute` | 物料属性表，定义某类物料有哪些参数字段 |
| `pc_unit` | 单位表，供应采购单位、库存单位、工程使用单位和值单位选择 |
| `pc_product_dict_type` | 产品业务字典类型 |
| `pc_product_dict_item` | 产品业务字典项 |

## 选择器来源

| 页面字段 | 数据来源 | 过滤条件 |
| --- | --- | --- |
| 物料类型 | 产品字典项 | `dictTypeCode = product_material_type`，启用 |
| 面料系列 | `pc_fabric_series` | `status = ENABLED`，`materialType = FABRIC`；仅 `materialType = FABRIC` 时显示 |
| 业务类型 | 产品字典项 | `dictTypeCode = product_business_type`，启用 |
| 主单位 / 采购单位 / 库存单位 / 工程使用单位 | `pc_unit` | 启用 |
| 类型属性定义 | `pc_base_attribute` | `status = ENABLED`，`material_types` 包含当前 `material_type` |
| 类型属性值单位 | `pc_unit` | 启用；通常仅数字属性显示 |
| 状态 | 通用状态枚举 | `ENABLED` / `DISABLED` |

## 状态和字典

| 字段 | 来源 | 可选值 |
| --- | --- | --- |
| `material_type` | 产品字典 `product_material_type` | 例如 `FABRIC`、`HARDWARE`、`TRACK`、`MOTOR`、`REMOTE`、`CHAIN`、`INSTALL_PART`、`PACKAGING`、`FINISHED_GOOD` |
| `business_type` | 产品字典 `product_business_type` | 以字典维护为准 |
| `price_currency_code` | 当前阶段固定默认 | `CNY` |
| `status` | 通用状态 | `ENABLED` / `DISABLED` |

具体单位值来自 `pc_unit`，不走产品字典。

## 校验规则

- `material_code`、`material_name_cn`、`material_type` 必填。
- `material_code` 在同一租户下由接口服务层校验唯一。
- 相同物料中文名称、主规格、供应商编码或供应商名称的物料由接口服务层判重。
- `material_type = FABRIC` 时必须选择面料系列；后端会回填 `fabric_series_id`、`fabric_series_code`、`fabric_series_name_cn`。
- `material_type != FABRIC` 时后端会清空面料系列快照字段。
- `price_currency_code` 为空时默认 `CNY`。
- `spec_summary` 为空时，后端当前按 `attribute_summary`、`primary_spec` 顺序补默认值。
- `purchase_unit_code`、`inventory_unit_code`、`usage_unit_code` 为空时，后端当前默认使用 `unit_code`。
- `purchase_enabled`、`inventory_enabled` 为空时，后端当前默认 `false`。
- 保存物料时如果带 `attributeList`，后端会同步保存到 `pc_material_attribute`。
- `attributeList` 中没有 `attribute_code` 的行会被忽略。
- 类型属性值应和属性定义的 `value_type` 对应：
  - `TEXT` / `ENUM` 使用 `value_text`。
  - `NUMBER` 使用 `value_number`，可带 `value_unit_code`。
  - `BOOLEAN` 使用 `value_bool`。
- 删除物料前应检查引用；有引用时优先停用。
- 普通修改只允许修改停用状态物料；启用状态物料需要先停用，或后续走独立超级修改。
- 修改按钮打开编辑抽屉前应先调用修改前检查接口，避免用户填写完成后才被保存接口拒绝。
- 超级修改是独立权限和独立接口，本阶段物料资料已落地；不混入普通修改接口。

## 修改边界

物料资料遵守 [基础资料状态与修改规则](../基础资料状态与修改规则.md)。

修改前检查返回示例：

```json
{
  "editable": false,
  "reason": "当前资料已启用，不可直接修改，请先停用后再编辑。",
  "status": "ENABLED",
  "impactSummary": []
}
```

## 常见错误

| 错误场景 | 后端提示 | 用户提示 |
| --- | --- | --- |
| 物料编码为空 | 校验失败 | 请填写物料编码 |
| 物料名称为空 | 校验失败 | 请填写物料中文名称 |
| 物料类型为空 | 校验失败 | 请选择物料类型 |
| 编码重复 | 物料编码已存在 | 当前物料编码已存在，请检查是否编辑已有物料 |
| 删除被引用物料 | 引用检查不通过 | 当前物料已被物料属性、组件明细或工程配置引用，建议停用 |
| 类型属性不显示 | 没有匹配属性定义 | 请先维护当前物料类型对应的物料属性 |

| 意图 | 方法 | 路径 | 权限 |

### 必填槽位

- `materialCode`
- `materialNameCn`
- `materialType`

### 可选槽位

- `unitCode`
- `usageUnitCode`
- `primarySpec`
- `specSummary`
- `supplierName`
- `vendorItemNo`
- `purchaseUnitPrice`
- `costUnitPrice`
- `attributeList`

### 可推断字段

- `status = ENABLED`
- `priceCurrencyCode = CNY`
- `purchaseEnabled` 和 `inventoryEnabled` 可按物料类型给建议，但保存前必须确认。
- `usageUnitCode` 可按物料类型建议，例如面料建议米或平方米，配件建议个，系统包建议套。

### 需要追问

- 用户没有说明物料类型时，必须追问。
- 当前物料类型要求的关键类型属性缺失时，需要追问。
- 用户要求修改启用物料时，必须先调用修改前检查；不可普通修改时提示停用或超级修改。
- 用户要求删除已被引用物料时，需要提示风险并建议停用。

### Payload 边界

- 不要求用户提供 `materialId`、`attributeValueId`、租户字段或审计字段。
- 类型属性只通过 `attributeList` 传稳定属性编码和值，不让用户填写 JSON。
- 普通编辑只走普通修改接口；超级修改必须由用户明确表达且账号具备权限。

### 禁止暴露字段

- `materialId`
- `attributeValueId`
- `tenantId`
- `createBy`
- `createTime`
- `updateBy`
- `updateTime`
- `delFlag`

### 保存前确认格式

```text
将保存物料资料：
- 物料编码：{materialCode}
- 物料名称：{materialNameCn}
- 物料类型：{materialTypeLabel}
- 标准规格：{primarySpec}
- 工程使用单位：{usageUnitCode}
- 供应商：{supplierName}
- 供应商料号：{vendorItemNo}
- 采购单价：{purchaseUnitPrice} {priceCurrencyCode}
- 成本单价：{costUnitPrice} {priceCurrencyCode}

```

### 失败提示

| 场景 | 给用户的提示 |
| --- | --- |
| 物料编码重复 | 物料编码已存在，请换一个编码，或者打开已有物料继续编辑。 |
| 物料类型缺失 | 请先说明物料类型，例如面料、铝材、电机或配件。 |
| 关键类型属性缺失 | 当前物料类型还缺少必要属性，请补充后再保存。 |
| 启用物料不可普通修改 | 当前物料已启用，不可普通修改；请先停用，或确认是否走超级修改。 |
| 物料已被引用 | 当前物料已被物料属性、组件明细或工程配置引用，建议停用，不建议删除。 |
| 无权限 | 当前账号没有维护物料资料的权限，请联系管理员授权。 |
