# 基础信息实现对照清单

本文按 2026-06-21 当前代码和知识库文档整理，用于后续测试、修复和继续补齐基础信息模块。它不是操作手册；稳定用法仍以各功能目录下的 `操作手册.md` 和 `接口与数据契约.md` 为准。

## 分层边界

当前基础信息的定位保持为“维护资料是什么”：

- 基础信息维护产品分类、产品字典、单位、物料属性定义、面料系列、物料资料、组件包、组件明细和资料资产。
- 工程配置维护产品怎么组成、什么条件下可用或带出。
- 产品配置维护客户下单时怎么问、怎么选。

不要把工程能力规则、尺寸可做性、带出规则继续塞回物料资料或配置答案。

## 已落地的通用保护

| 规则 | 当前实现 |
| --- | --- |
| 启用资料不可普通编辑 | 前端编辑前调用 `edit-check`，后端 `ProductServiceSupport.assertNormalEditable` 兜底 |
| 删除前引用检查 | 前端删除前调用 `references`，后端返回 `allowed` / `referenceCount` / `referenceSummaries` |
| 新业务选择器默认取启用资料 | 页面 option loader 通常传 `status: ENABLED`；产品字典 `type/{dictTypeCode}` 固定只取启用项 |
| 超级修改独立入口 | 当前只在物料资料启用 `superUpdate`，权限为 `product:base:superEdit` |
| 资料附件复用 OSS | 通用 grid 上传到 `/system/oss/upload`，再写入资料资产和资料绑定 |

## 当前已校准口径

| 项目 | 处理结果 |
| --- | --- |
| 产品分类、单位 | 使用当前实体 Service |
| 物料属性定义权限 | 使用 `product:base-attribute:*` |
| 产品分类表名 | 使用 `pc_category` |
| 产品分类层级和路径 | 后端根据 `parentId` 维护 `categoryLevel` / `categoryPath` |
| 产品字典上级字典项 | 当前仅 `product_category_level` 展示 `parentValue`，普通字典类型隐藏并清空 |
| 物料属性定义值类型 | 当前前端固定为 `TEXT`、`NUMBER`、`BOOLEAN`、`ENUM`，不是产品字典来源 |
| 单位类型 | 当前 SQL 种子和产品字典使用 `COUNT` 表示数量类单位，前端从 `product_unit_type` 读取 |
| 物料资料默认值 | 后端为空时补 `specSummary`、采购/库存/工程单位、采购/库存启用标记和币种默认值 |
| 面料物料模型 | 面料是 `pc_material` 中 `material_type = FABRIC` 的物料；面料系列快照保存在 `pc_material.fabric_series_*` 字段 |
| 组件包模型 | `pc_component` 只保存组件包主档；物料组成统一走 `pc_component_item` |
| 物料唯一性 | 物料编码、名称 + 规格 + 供应商由接口服务层校验，不新增业务复合数据库唯一约束 |

## 下一轮建议核查

| 优先级 | 模块 | 核查点 |
| --- | --- | --- |

| P1 | 面料系列 / 物料资料 | 面料系列前置、`FABRIC` 物料绑定系列和附件绑定目标是否符合最新文档 |
| P1 | 资料资产 | 附件上传、资产创建、绑定创建的失败回滚策略需要验证 |
