# 基础信息实现对照清单

本文记录当前基础信息闭环的实现口径，用于测试、修复和后续补测。稳定用法仍以各功能目录下的 `操作手册.md` 和 `接口与数据契约.md` 为准。

## 分层边界

当前基础信息定位为“维护基础分类、枚举、单位和物料主数据是什么”：

- 基础信息维护产品分类、基础字典、单位、厂家、物料类型、物料属性、物料管理和资料资产绑定。
- 工程配置维护产品怎么组成、什么条件下可用或带出。
- 产品配置维护客户下单时怎么问、怎么选。
- 未重写确认的采购、库存、生产处理单位换算、批次、余料和领退料。

| 能力 | 前端入口 | 后端入口 | 当前状态 |
| --- | --- | --- | --- |

| 厂家管理 | `ProductBaseInfoPage.vue` 的 `manufacturer` 配置 | `ProductBaseInfoController` + `ProductManufacturerService` | 厂家编号和名称维护；物料按厂家下拉保存快照 |
| 物料类型 | `ProductMaterialTypePage.vue` | `ProductBaseInfoController` + `ProductMaterialTypeGroupService` / `ProductMaterialTypeService` | 已有属性分组和物料类型双栏维护 |
| 物料属性 | `ProductBaseInfoPage.vue` 的 `baseAttribute` 配置 | `ProductBaseInfoController` + `ProductBaseAttributeService` | 已有 CRUD、启停、编辑检查、引用检查；属性按分组维护 |

## 已落地的通用保护

| 规则 | 当前实现 |
| --- | --- |
| 已确认资料不可普通编辑 | 前端编辑前调用 `edit-check`，后端兜底 |
| 删除前引用检查 | 前端删除前调用 `references`，后端删除前再次校验 |
| 新业务选择器默认取可用资料 | options 查询默认取 `status = ENABLED` |
| 超级修改独立入口 | 当前在物料管理启用，权限为 `product:base:superEdit` |
| 物料类型驱动物料属性 | 物料类型归属属性分组，物料属性按属性分组加载 |
| 厂家驱动物料快照 | 物料选择厂家后保存厂家 ID、编号和名称快照 |
| 表单防误关闭 | 物料新增/编辑/超级修改抽屉有未保存确认保护 |
| 变更流水 | 物料新增、修改、超级修改、审核、取消审核写入 `pc_change_log` |

## 当前物料口径

| 项目 | 当前规则 |
| --- | --- |
| 状态 | `DISABLED` 展示为未审核，`ENABLED` 展示为已审核 |
| 新增默认 | 新增物料默认未审核 |
| 审核 | 审核走 `change-status/{id}/ENABLED`，记录 `audit_by` 和 `audit_time` |
| 取消审核 | 取消审核走 `change-status/{id}/DISABLED`，保留最后审核记录 |
| 必填 | 物料编码、物料名称、物料类型、规格、主单位 |
| 唯一性 | 物料编码唯一；物料名称 + 型号 + 规格 + 厂家 ID 唯一 |
| 规格型号 | `model`、`spec` 是结构化字段，`spec_model_text` 后端自动生成 |
| 厂家 | 物料表单选择厂家，后台保存 `manufacturer_id`、`manufacturer_code`、`manufacturer_name` |
| 类型属性 | 保存物料时同步保存 `attributeList` 到 `pc_material_attribute` |
| 删除 | 被物料属性值、资料绑定等业务对象引用时不可删除 |
| 停用/取消审核 | 通常允许，表示不再给新增业务选择；不应仅因有引用就阻断 |
