# 物料管理接口与数据契约

## 页面与权限

| 项目 | 内容 |
| --- | --- |
| 菜单名称 | 基础信息 / 物料管理 |
| 前端路由 | `/product-master/materials` |
| 前端组件 | `product-center/base/ProductEntityGridPage.vue` 的 `material` 配置 |
| 权限码 | `product:base:*` |

## 后端接口

| 动作 | 方法 | 路径 | 权限 |
| --- | --- | --- | --- |
| 列表 | GET | `/product-capability/materials/list` | `product:base:list` |
| options | GET | `/product-capability/materials/options` | `product:base:list` |
| 详情 | GET | `/product-capability/materials/{id}` | `product:base:list` |
| 修改前检查 | GET | `/product-capability/materials/{id}/edit-check` | `product:base:edit` |
| 新增 | POST | `/product-capability/materials` | `product:base:add` |
| 修改 | PUT | `/product-capability/materials` | `product:base:edit` |
| 超级修改 | PUT | `/product-capability/materials/super-update` | `product:base:superEdit` |
| 审核 / 取消审核 | PUT | `/product-capability/materials/change-status/{id}/{status}` | `product:base:edit` |
| 删除 | DELETE | `/product-capability/materials/{ids}` | `product:base:remove` |
| 引用检查 | GET | `/product-capability/materials/{id}/references` | `product:base:reference` |
| 变更记录 | GET | `/product-capability/change-logs/list` | `product:base:reference` |

物料属性值有独立接口和隐藏台账，普通维护入口在物料管理新增、编辑、超级修改抽屉里同步保存。

物料管理页面选中一条物料后，可通过“变更记录”查看该物料的新增、修改、超级修改、审核和取消审核流水。

## 可检索性与引用关系

业务编码或单号：

- `material_code`

搜索展示字段：

- `material_code`
- `material_name_cn`
- `material_name_en`
- `spec`
- `material_type_code`
- `material_type_name_cn`
- `attribute_group_code`
- `attribute_group_name_cn`
- `manufacturer_code`
- `status`
- `updateTime`

主要引用关系：

- 物料属性值通过 `material_id` / `material_code` 关联物料。
- 资料资产绑定可通过 `targetType = MATERIAL`、`targetId = materialId` 关联物料。
- 配方设置中的原料池和选项物料通过物料编码引用物料。
- 后续已确认业务模块应以 `material_code` 作为全局检索和业务关联入口。

权限过滤依据：

- `tenantId`
- `product:base:list` / `product:base:reference`
- `status`
- `delFlag`

详情入口：

| 入口 | 内容 |
| --- | --- |
| 前端路由 | `/product-master/materials` |
| 后端详情接口 | `GET /product-capability/materials/{id}` |

说明：搜索结果只作为入口和摘要展示；物料状态、属性值、引用检查和审计信息以业务数据库和详情接口为准。

## 校验规则

- `material_code`、`material_name_cn`、`material_type_code`、`spec`、`unit_code` 必填。
- 物料类型必须来自 `pc_material_type`。
- 保存物料时，后端根据物料类型回填 `material_type_*` 和 `attribute_group_*` 快照字段。
- 保存物料时，后端根据厂家回填 `manufacturer_id`、`manufacturer_code`、`manufacturer_name` 快照字段。
- 新增物料默认 `status = DISABLED`，页面展示为未审核。
- 审核时写入 `status = ENABLED`，并记录 `audit_by`、`audit_time`。
- 取消审核时写入 `status = DISABLED`，保留最后审核记录。
- 新增、修改、超级修改、审核、取消审核成功后写入 `pc_change_log`。
- `material_code` 由接口服务层校验唯一。
- `material_name_cn + model + spec + manufacturer_id` 由接口服务层校验唯一。
- 未选择厂家时，按 `material_name_cn + model + spec` 判断唯一。
- `spec_model_text` 由后端生成；页面不提供手工录入。
- 页面列表默认将 `material_name_cn`、`material_name_en` 拆成两列展示。
- 页面列表默认展示 `spec`，规格允许多行展示完整内容；`spec_model_text` 保留为系统生成字段，不作为默认列、默认查询条件或表单输入项。
- 保存物料时如果带 `attributeList`，后端同步保存到 `pc_material_attribute`。
- 删除物料前必须检查引用；有引用时拒绝删除。
- 已审核物料不能普通编辑；取消审核后可普通编辑。

## 常见错误

| 错误场景 | 后端提示 | 用户提示 |
| --- | --- | --- |
| 物料编码未填 | `product.material.codeRequired` | 请填写物料编码。 |
| 物料中文名称未填 | `product.material.nameRequired` | 请填写物料中文名称。 |
| 规格未填 | `product.material.specRequired` | 请填写规格。 |
| 物料类型未填 | `product.material.materialTypeRequired` | 请选择物料类型。 |
| 主单位未填 | `product.material.unitRequired` | 请选择主单位。 |
| 物料类型不存在 | `product.material.materialTypeNotFound` | 请选择有效的物料类型。 |
| 物料编码重复 | `product.material.codeExists` | 物料编码已存在，请换一个编码或编辑已有物料。 |
| 自然键重复 | `product.material.naturalKeyExists` | 已存在相同名称、型号、规格和厂家的物料。 |
| 删除已引用物料 | `product.material.hasReferences` | 该物料已被引用，请取消审核而不是删除。 |
| 已审核物料普通修改 | `product.material.auditedEditDenied` | 已审核物料不可普通修改，请先取消审核。 |

### 必填槽位

新增物料必须补齐：

- `materialCode`：物料编码。
- `materialNameCn`：物料中文名称。
- `materialTypeCode` 或可唯一识别的物料类型名称。
- `spec`：规格。
- `unitCode` 或可唯一识别的主单位。

普通编辑和超级修改必须先定位唯一物料，优先使用 `materialCode`。无法唯一定位时必须追问。

### 可选槽位

- `materialNameEn`：物料英文名称。
- `model`：型号。
- `colorName`：颜色。
- `weightValue`：克重/重量。
- `secondaryUnitCode`：副单位。
- `unitPrice`：单价；AI 不主动猜测，只保存用户明确提供的值。
- `salesPrice`：销售价；AI 不主动猜测，只保存用户明确提供的值。
- `manufacturerId`、`manufacturerCode` 或可唯一识别的厂家名称。
- `manufacturerItemNo`：厂家物料编码。
- `remark`：备注。
- `attributeList`：类型属性值。

### 可推断字段

- 新增物料默认 `status = DISABLED`，页面展示为未审核。
- `material_type_id`、`material_type_code`、`material_type_name_cn` 由后端根据物料类型回填。
- `attribute_group_id`、`attribute_group_code`、`attribute_group_name_cn` 由后端根据物料类型所属属性分组回填。
- `manufacturer_code`、`manufacturer_name` 由后端根据厂家回填。
- `spec_model_text` 由后端根据型号和规格生成。

### 需要追问

- 缺少物料编码、物料中文名称、物料类型、规格或主单位。
- 用户说的物料类型、单位或厂家无法唯一匹配。
- 当前物料类型有关键类型属性，但用户未提供对应属性值。
- 用户要求编辑已审核物料，但没有明确要求超级修改。
- 用户要求删除已引用物料时，应提示不能删除并建议取消审核。

### Payload 边界

- AI 只提交业务字段，不提交 `tenantId`、`createBy`、`createTime`、`updateBy`、`updateTime`、`delFlag`。
- AI 不让用户填写内部 ID；只有在后台通过编码或名称解析出唯一对象后，才可在 payload 中携带对应 ID。
- AI 不直接提交 `specModelText`、`auditBy`、`auditTime`。
- AI 不猜测价格、采购、库存、生产工艺等未确认业务结果。
- 普通编辑前必须先调用 `edit-check`；不可编辑时不调用普通 `PUT`。
- 超级修改必须由用户明确提出受控修正，并且账号具备 `product:base:superEdit`。

### 禁止暴露字段

- `materialId`
- `tenantId`
- `attributeGroupId`
- `createBy`
- `createTime`
- `updateBy`
- `updateTime`
- `delFlag`

### 失败提示

| 场景 | 用户提示 |
| --- | --- |
| 物料编码缺失 | 请提供物料编码。 |
| 物料中文名称缺失 | 请提供物料中文名称。 |
| 物料类型缺失或无法匹配 | 请说明物料类型，或先到物料类型维护可用类型。 |
| 规格缺失 | 请提供规格。 |
| 主单位缺失或无法匹配 | 请说明主单位，或先到单位管理维护可用单位。 |
| 厂家无法匹配 | 没找到这个厂家，请确认厂家名称，或先到厂家管理维护。 |
| 物料编码重复 | 物料编码已存在，请换一个编码，或者打开已有物料继续编辑。 |
| 自然键重复 | 已存在相同名称、型号、规格和厂家的物料，请确认是否编辑已有物料。 |
| 已审核不可普通编辑 | 该物料已审核，请先取消审核；如需受控修正，请使用超级修改。 |
| 无超级修改权限 | 当前账号没有超级修改权限，请联系管理员授权或改走取消审核后编辑。 |
| 删除已引用物料 | 该物料已被引用，不能删除；可以取消审核让它不再进入新增业务选择。 |
