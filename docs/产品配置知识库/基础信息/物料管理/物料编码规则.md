# 物料编码规则

## 编码结构

正式物料编码由四段组成，不带短横线：

```text
DL(2) + LX(2) + HX(6) + 厂家编号(3)
```

| 段 | 长度 | 含义 |
| --- | --- | --- |
| `DL` | 2 | 物料大类编码 |
| `LX` | 2 | 物料类型编码 |
| `HX` | 6 | 核心属性编码 |
| 厂家编号 | 3 | 来自厂家管理的 `manufacturer_code` |

示例展示可以使用短横线方便阅读：

```text
ML-JL-ZG3000-001
```

系统正式保存时不带短横线：

```text
MLJLZG3000001
```

## 厂家编号

厂家编号来自厂家管理，是 3 位业务编号。

| 编号 | 用途 |
| --- | --- |
| `001` 到 `899` | 真实厂家 |
| `900` | 通用厂家、暂无厂家、自制件 |
| `901` 到 `999` | 预留通用或特殊来源 |

初始化脚本只内置 `900 通用厂家/暂无厂家`。真实厂家由业务人员在厂家管理中录入。

## 与物料管理的关系

当前物料管理仍由用户录入物料编码，系统只校验唯一性。本规则用于统一编码口径，为后续“编码建议/生成器”和 AI 表格录入做准备。

物料管理中：

- “厂家”来自厂家管理。
- “厂家编号”属于厂家主数据，不在物料表单中重复录入。
- “厂家物料编码”是厂家自己的料号，存入 `manufacturer_item_no`，不参与系统物料编码唯一性。
