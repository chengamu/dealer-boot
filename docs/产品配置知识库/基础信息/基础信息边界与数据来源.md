# 基础信息边界与数据来源

本文提炼当前基础信息的长期边界和可复用数据来源。具体页面、接口和字段以各功能目录的 `操作手册.md`、`接口与数据契约.md` 为准。

## 模块目标

基础信息解决：

- 统一物料管理是什么。
- 物料有哪些标准规格和补充属性。
- 物料如何被工程配置按类型引用。
- 组件包由哪些物料组成。
- 资料资产在哪里，以及如何绑定到业务对象。
- 这些资料如何被未来 ERP、工程配置、产品配置和订单快照共同引用。

基础信息不解决：

- 采购审批、库存扣减、生产排程。
- 销售报价、价格规则、经销商价、运费和加价项。
- 产品尺寸能力判断、生产用料规则、带出规则。
- 客户下单问题配置和选项联动。

## 建模边界

- `pc_material` 是基础信息核心主表。面料、铝材、电机、拉珠、遥控器、太阳能板、轨道、下杆、配件、安装件、包装件、标准成品都进入统一物料。
- 不为电机、遥控器、安装码、胶条、配重条等每个类型建立独立 profile 表；差异属性进入 `pc_material_attribute`。
- `pc_base_attribute` 定义属性，例如规格、重量、电压、控制协议、通道数、形状、颜色。
- 面料因为有系列资料和原始限制，保留 `pc_fabric_series`；具体面料 SKU 是 `pc_material` 中 `material_type_code = FABRIC` 的物料。
- 面料系列只维护系列主数据和原始限制，不维护工程生产规则。
- 工程能力、尺寸可做性、带出规则、生产用料、发布检查和试算器规则进入工程配置或后续对应模块。

## 资料资产边界

文件本体复用系统 OSS：

- 文件本体：`sys_oss` / `bocoo-common-oss`
- 上传接口：`/system/oss/upload`
- 下载接口：`/system/oss/download/{ossId}`

基础信息只维护业务资料资产和绑定关系：

- `pc_media_asset` 保存 `ossId`、资料类型、名称、来源等业务信息。
- `pc_media_binding` 保存资料绑定对象和用途。

不要为产品配置中心重做文件系统或附件存储。

## OFBiz 数据来源

老 OFBiz 系统只作为开发和测试样本来源，不作为新系统正式模型。

适合复用为测试数据：

- 产品主数据、产品分类、可配置产品。
- 配置问题、配置选项、选项绑定物料或子产品。
- 产品图片、说明、文档类资料的附件索引。
- 报价和订单中的产品、数量、尺寸、房间、价格字段。

需要脱敏或暂不导入：

- 客户、经销商、用户、地址、联系方式。
- 报价和订单备注。
- 可能包含客户现场信息的图片、PDF、图纸。

不直接复用为新模型：

- OFBiz 通用实体主键和状态模型。
- OFBiz Content / DataResource 作为新附件主表。
- OFBiz ProductAssoc 作为完整工程配置规则。
- OFBiz Quote / OrderHeader 作为新订单模型。

推荐策略：

1. 抽取少量真实产品线作为 seed/test data。
2. 生成新系统基础资料、配置模板、资料资产 metadata 和试算样本。
3. 文件本体可访问时导入 OSS；不可访问时只保留 legacy metadata。
4. 工程配置规则、价格规则和发布检查按新系统目标重新建模。
