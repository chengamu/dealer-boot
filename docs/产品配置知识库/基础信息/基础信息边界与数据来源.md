# 基础信息边界与数据来源

本文定义当前基础信息的稳定边界。具体页面、接口和字段以各功能目录的 `操作手册.md`、`接口与数据契约.md` 为准。

## 模块目标

基础信息解决：

- 产品分类如何作为工程和后续业务的基础分类来源。
- 基础枚举如何统一维护。
- 单位如何统一选择和展示。
- 物料类型如何按属性分组管理。
- 物料属性模板如何按分组复用。
- 物料主档如何被配方管理和后续已确认业务模块引用。
- 资料资产如何绑定到当前基础资料对象。

基础信息不解决：

- 产品工程结构、配方、用料规则和尺寸能力。
- 未重写确认的报价、价格规则、运费和加价项。
- 未重写确认的采购审批、库存批次、余料扣减、生产排程。
- 客户下单问题配置和选项联动。
- 文件本体存储和附件上传服务。

## 当前模块

| 模块 | 业务定位 | 主要数据 |
| --- | --- | --- |
| 产品分类 | 产品、工程和后续配置的基础分类来源 | `pc_product_category` |
| 基础字典 | 产品业务内的简单枚举 | `pc_product_dict_type`、`pc_product_dict_item` |
| 单位管理 | 统一单位选择和展示 | `pc_unit` |
| 厂家管理 | 物料供货来源和厂家编号来源 | `pc_manufacturer` |
| 物料类型 | 属性分组和物料类型两级主数据 | `pc_material_type_group`、`pc_material_type` |
| 物料属性 | 按属性分组维护物料扩展字段模板 | `pc_base_attribute` |
| 物料管理 | 统一物料主档和物料属性值 | `pc_material`、`pc_material_attribute` |
| 资料资产 | 资料 metadata 和物料附件绑定 | `pc_media_asset`、`pc_media_binding` |

## 建模边界

- `pc_material` 是基础信息核心主表。面料、铝材、电机、拉珠、遥控器、太阳能板、轨道、下杆、配件、安装件、包装件、标准成品都进入统一物料。
- 不为每个物料类型建立独立 profile 表；稳定核心字段进 `pc_material`，类型扩展字段进 `pc_material_attribute`。
- `pc_material_type_group` 表示属性分组，例如面料、铝材、控制系统、五金配件、包装参数。
- `pc_material_type` 表示具体物料类型，例如面料、电机、遥控器、拉珠、轨道、下杆、包装件。
- `pc_base_attribute` 定义某个属性分组下有哪些属性，例如颜色、纹理、电压、协议、通道数、材质、安装方式。
- 物料新增时选择物料类型，系统根据物料类型所属属性分组加载属性模板。

## 资料与文件边界

文件本体复用系统 OSS：

- 文件本体：`sys_oss` / `bocoo-common-oss`
- 上传接口：`/system/oss/upload`
- 下载接口：`/system/oss/download/{ossId}`

基础信息只保存当前基础资料对象的业务绑定。当前稳定闭环是物料附件绑定，不重做文件系统或附件存储。

## 外部数据来源

ERP、历史表格和人工整理资料只作为录入来源。导入前需要整理成当前模型：

1. 先确认单位、物料类型和物料属性模板。
2. 再录入物料编码、名称、型号、规格、单位、单价、销售价、厂家等核心字段。
3. 类型专属信息进入物料属性值。
4. 批量表格整理后续可接入 AI 辅助，但当前不作为基础信息闭环的必做功能。
