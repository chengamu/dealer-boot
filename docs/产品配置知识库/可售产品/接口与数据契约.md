# 可售产品接口与数据契约

## 模块定位

可售产品是客户下单前可选择的产品入口。它绑定一个已生效的配方版本，但不是订单成功后生成的具体尺寸 SKU。

业务链路：

1. 基础信息维护物料、单位、分类和字典。
2. 配方管理审核通过，生成生效配方版本。
3. 可售产品绑定生效配方版本。
4. 价格设置维护该可售产品的面料价格和附加费规则。
5. 后续订单成功后再生成具体尺寸 SKU、用料和价格快照。

## 页面路由

| 页面 | 路由 | 说明 |
| --- | --- | --- |
| 可售产品 | `/product-pricing/sale-products` | 标准 grid，维护可售产品主档 |
| 价格设置 | `/product-formula/price-settings?saleProductId={id}` | 从当前可售产品进入价格设置工作台 |

## 接口清单

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/product-pricing/sale-products/list` | 分页查询可售产品 |
| `GET` | `/product-pricing/sale-products/options` | 查询可售产品选择器 |
| `GET` | `/product-pricing/sale-products/{id}` | 查询可售产品详情 |

| `PUT` | `/product-pricing/sale-products` | 修改可售产品 |

| `PUT` | `/product-pricing/sale-products/change-status/{id}/{status}` | 启用或停用可售产品 |
| `GET` | `/product-pricing/sale-products/{id}/references` | 查询价格规则引用 |

## 主表契约

表：`pc_sale_product`

| 字段 | 说明 |
| --- | --- |
| `sale_product_id` | 主键 |
| `tenant_id` | 租户 |
| `sale_product_code` | 可售产品编号 |
| `sale_product_name` | 可售产品名称 |
| `category_id/category_code/category_name_cn` | 来源于绑定配方的产品分类快照 |
| `product_type_code/product_type_name_cn` | 来源于绑定配方的产品类型快照 |
| `formula_id/formula_code/formula_name` | 绑定配方快照 |
| `formula_version_id/formula_version_no/formula_version_label` | 当前生效配方版本 |
| `min_width_inch/min_height_inch/max_width_inch/max_height_inch` | 来源于绑定配方的尺寸范围 |
| `size_summary` | 尺寸摘要 |
| `price_status` | 价格状态：`NOT_READY`、`READY`、`WARNING` |
| `status` | 可售状态：`ENABLED`、`DISABLED` |
| `sort_order/remark` | 排序和备注 |

## 业务规则

- 新增和修改必须选择已生效且存在版本快照的配方。
- 分类、产品类型、尺寸范围和配方版本字段由后端按配方快照冗余，不由用户手填。
- 可售产品编号在当前租户内唯一。
- 相同配方版本和可售产品名称不能重复。
- 启用前必须完成价格校验，且 `price_status = READY`。
- 启用后不可普通编辑，可停用。
- 已维护价格规则的可售产品不能删除。
