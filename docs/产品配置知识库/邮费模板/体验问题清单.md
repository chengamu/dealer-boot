# 邮费模板体验问题清单

## 当前已确认

- 邮费模板使用标准 grid 列表和抽屉表单。
- 邮费模板采用面积区间固定价格，不提供邮费公式编辑器。
- 币种来自系统标准币种表，不走产品字典，也不允许自由文本。
- 页面不展示费用类别或自由触发条件；规则以“系统 = 电机 / 系统 ≠ 电机”的中文适用条件表达。
- 模板状态在 grid 中使用独立开关控制，新增和修改抽屉不录入状态。
- 面积区间使用平方英尺，最大面积为空表示不封顶。
- 模板编码由系统自动生成；默认模板和排序不在当前界面展示。
- 邮费明细支持 Excel 批量导入，导入结果先进入抽屉草稿，保存后才落库。
- 邮费独立于价格设置；同币种启用新模板时自动替换旧模板。
- 客户报价确认时已保存邮费结果和来源规则快照，避免每周调价影响历史报价。
- 销售订单提交时已保存邮费结果和来源规则快照，并结构化保存邮费模板 ID。

## 后续观察

- 是否需要按国家或承运商扩展多个模板维度。
- 是否需要在价格校验问题中定位到具体邮费模板规则。

## 暂不处理

- 本模块不做实际物流渠道报价。
- 本模块已接入客户报价和销售订单测算，但不负责支付处理，也不接承运商 API。
