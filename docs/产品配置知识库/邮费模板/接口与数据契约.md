# 邮费模板接口与数据契约

## 模块定位

邮费模板维护美国销售场景下可复用的面积区间固定邮费。它独立于产品价格设置；客户报价和销售订单均按价格币种读取当前启用模板，并冻结实际命中的模板、规则和邮费。

页面是标准 grid：模板状态只在列表中单独启停，新增和修改抽屉不录入状态；新建模板默认停用，邮费明细完整后再从列表启用。

## 页面路由

| 页面 | 路由 | 说明 |
| --- | --- | --- |
| 邮费模板 | `/product-formula/shipping-templates` | 标准 grid，维护模板和面积分段固定邮费 |

## 接口清单

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/product-pricing/shipping-templates/list` | 分页查询邮费模板 |
| `POST` | `/product-pricing/shipping-templates/export` | 按当前查询条件导出邮费模板 |
| `GET` | `/product-pricing/shipping-templates/options` | 查询模板选项；传 `status=ENABLED` 获取当前有效模板 |
| `GET` | `/product-pricing/shipping-templates/{id}` | 查询模板详情和规则 |
| `POST` | `/product-pricing/shipping-templates` | 新增模板 |
| `PUT` | `/product-pricing/shipping-templates` | 修改模板 |
| `DELETE` | `/product-pricing/shipping-templates/{ids}` | 删除停用模板；启用模板不可删除 |
| `PUT` | `/product-pricing/shipping-templates/change-status/{id}/{status}` | 启用或停用模板 |
| `POST` | `/product-pricing/shipping-templates/import-rules` | 解析 Excel 邮费明细，返回抽屉草稿数据 |
| `POST` | `/product-pricing/shipping-templates/import-template` | 下载邮费明细导入模板 |

## 表结构契约

### `pc_shipping_template`

| 字段 | 说明 |
| --- | --- |
| `shipping_template_id` | 主键 |
| `template_code/template_name` | 模板编码和名称；编码由后端自动生成，用户不可编辑 |
| `currency_code` | 币种，来源为标准币种表 `sys_currency`，默认 `USD` |
| `default_flag` | 内部保留字段，当前页面不维护，也不参与有效模板选择 |
| `status` | `ENABLED`、`DISABLED` |
| `sort_order/remark` | 排序为内部保留字段，当前录入界面不展示；备注可维护 |

### `pc_shipping_template_rule`

| 字段 | 说明 |
| --- | --- |
| `shipping_template_rule_id` | 主键 |
| `shipping_template_id` | 所属模板 |
| `fee_code/fee_name` | 邮费适用条件的稳定业务码：`MANUAL` 表示“系统未选择电机”，`MOTORIZED` 表示“系统选择电机” |
| `min_area_sqft/max_area_sqft` | `numeric(18,4)`；面积区间使用平方英尺，采用左闭右开；最大面积为空表示不封顶 |
| `fee_amount` | `numeric(18,4)`；固定邮费来源值，允许为 `0`，不能小于 `0`；正式金额按币种精度冻结 |
| `sort_order/remark` | 排序和备注 |

## 业务规则

- 模板编码由 Service 自动生成并校验唯一，新增和修改接口都不接受用户改写编码。
- 租户字段由登录请求上下文注入，新增和修改接口不接受前端指定租户。
- 操作人员维护的是适用条件，不是手工选择“带电场景”：页面显示“系统 = 电机（带电）”和“系统 ≠ 电机（不带电）”。
- 启用模板必须同时覆盖“系统选择电机”和“系统未选择电机”两类条件。
- 同一适用条件下面积区间不能重叠。
- 每类适用条件的面积区间必须从 `0` 连续覆盖到不限，不能有空档；最大面积为空的不封顶规则必须是最后一段。
- 区间统一按左闭右开匹配：`minArea <= area < maxArea`。例如 `0-20` 不包含 `20`，面积正好为 `20` 时命中 `20-40`；最后一段最大面积为空时没有上界。
- 邮费模板不维护计算公式；每条明细只维护适用条件、面积区间和固定邮费。
- Excel 导入只替换当前抽屉中的未保存明细，用户检查并点击保存后才写入数据库。
- 同一租户、同一币种只能有一个启用模板；启用新模板时系统自动停用同币种旧模板。
- 启用模板不可编辑、不可删除；每周调价应复制或新增停用模板，维护完成后再启用替换旧模板。
- 邮费不进入价格设置，不参与可售产品价格就绪校验。
- 客户报价按价格币种、订单面积和是否选择电机命中当前启用模板规则，单件邮费乘数量后进入报价合计；报价明细保存模板 ID，内部价格快照保存命中规则和最终邮费。
- 销售订单直接复制确认报价中的邮费模板 ID、命中规则和最终邮费快照，不重新匹配当前模板。
- 币种使用系统标准数据接口 `/system/standard/currencies`，不使用产品字典或自由文本；后端统一保存为大写币种码。

## Grid 交互

- 查询：模板编码、模板名称、状态。
- 工具栏：新增、修改、删除、导出、显示/隐藏查询和刷新。
- 列表：模板编码、模板名称、币种、价格行数、状态、更新时间和操作。
- 状态使用行内开关并二次确认；新增、修改抽屉不出现状态字段。

## 可检索性与引用关系

- 稳定业务编码为 `template_code`，搜索展示模板名称、币种、价格行数、状态和更新时间。
- 列表和详情按租户、删除标记及邮费模板权限过滤；页面入口为 `/product-formula/shipping-templates`。
- 模板规则通过 `shipping_template_id` 归属模板；适用条件使用稳定业务码 `MANUAL/MOTORIZED`，中文名称只用于展示。
- 邮费模板不归属于某个可售产品或价格设置；报价服务按币种读取平台租户当前启用模板，并在报价明细和内部价格快照中冻结命中结果。
- 被任何未删除报价明细或销售订单明细引用的邮费模板不可删除；引用判断按结构化 `shipping_template_id` 精确查询，不使用 JSON 模糊匹配。
