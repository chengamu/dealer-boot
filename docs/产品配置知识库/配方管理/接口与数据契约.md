# 配方管理接口与数据契约

## 模块范围

配方管理当前包含五个业务模块：配方档案、配方原料、业务配置项、配方模拟、配方审核。

条件编辑器、公式编辑器、物料选择弹窗和用量设置弹窗只是模块内辅助能力，不单独作为业务模块。

## 页面路由

| 模块 | 页面 | 说明 |
| --- | --- | --- |
| 配方档案 | `/product-formula/formulas` | 配方列表、新增、编辑、版本、引用检查 |
| 配方原料 | `/product-formula/formulas/materials` | 原料池、生产备注、用量规则 |
| 业务配置项 | `/product-formula/formulas/options` | 客户可选项、选项值、关联物料、限制规则 |
| 配方模拟 | `/product-formula/formulas/simulation` | 独立模拟页，可从配方列表带 `formulaId` 进入，也可在页内切换配方 |
| 配方审核 | `/product-formula/reviews` | 审核列表和审核详情 |

### 配方审核

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/product-formula/reviews/list` | 查询审核列表 |
| `GET` | `/product-formula/reviews/{reviewId}` | 查询审核详情 |
| `PUT` | `/product-formula/reviews/{reviewId}/approve` | 审核通过并冻结版本 |
| `PUT` | `/product-formula/reviews/{reviewId}/reject` | 驳回 |

审核列表查询条件由 `ProductFormulaReviewBo` 承载：

| 字段 | 说明 |
| --- | --- |
| `formulaCode` | 按提交审核时的配方编号模糊查询 |
| `formulaName` | 按提交审核时的配方名称模糊查询 |
| `versionLabel` | 按版本展示号模糊查询，例如 `V1` |
| `submitBy` | 按提交人模糊查询 |
| `validationStatus` | 按提交审核时校验状态查询 |

审核列表固定查询待审核快照，即 `versionStatus = PENDING_REVIEW`。状态展示用于让用户理解当前审核池含义，不作为页面筛选项。配方编号和配方名称来自 `pc_formula_version.formula_snapshot_json` 中的 `formulaCode`、`formulaName`；版本号、提交人和校验状态来自 `pc_formula_version` 字段。

审核列表展示字段：

| 字段 | 来源 |
| --- | --- |
| 配方编号 | `formula_snapshot_json.formulaCode` |
| 配方名称 | `formula_snapshot_json.formulaName` |
| 当前版本 | `version_label` |
| 状态 | `version_status`，当前列表固定为待审核 |
| 提交人 | `submit_by` |
| 提交时间 | `submit_time`，前端展示到分钟 |
| 校验状态 | `validation_status` |

审核列表按钮：

| 按钮 | 权限 | 说明 |
| --- | --- | --- |
| 详情 | `product:formula:review` | 查看待审核版本快照 |
| 审核通过 | `product:formula:approve` | 审核通过并冻结生效版本 |
| 驳回 | `product:formula:reject` | 填写驳回原因并退回可编辑状态 |
| 审核记录 | `product:formula:reference` | 查看当前配方的提交、审核、驳回等变更流水 |

审核页交互：

- 顶部工具栏按钮作用于当前选中行。
- 表格行内直接展示“审核通过”和“驳回”两个核心动作。
- 双击行进入审核详情。
- “审核记录”抽屉复用 `GET /product-capability/change-logs/list`，参数为 `bizModule = FORMULA`、`bizType = FORMULA`、`bizId = formulaId`。

审核详情页读取 `GET /product-formula/reviews/{reviewId}`。详情页从 `formula_snapshot_json` 展示配方主信息，从 `setup_snapshot_json` 展示物料明细、用量规则数量和选项带出物料数量，并展示当前审核快照的状态、校验状态、审核人、审核时间和驳回原因。

## 配方原料契约

`pc_formula_material` 是配方原料池主行。

| 字段 | 说明 |
| --- | --- |
| `formula_material_id` | 原料行主键 |
| `formula_id` | 所属配方 |
| `material_id/material_code/material_name_cn` | 选入物料 |
| `spec_model_text` | 规格型号冗余展示 |
| `attribute_group_code/attribute_group_name_cn` | 物料属性分组 |
| `material_type_code/material_type_name_cn` | 物料类型 |
| `unit_code` | 物料主单位 |
| `loss_rate` | 损耗率，在原料池行内维护；模拟时在基础用量后按百分比加成，用量设置不重复录入 |
| `default_flag` | 默认带出标记，当前前端主表格不作为核心展示字段 |
| `required_flag` | 生产必带：命中配置时生产必须包含，不等于客户可见 |
| `usage_mode` | 用量方式，当前收敛为 `FIXED/FORMULA`；复杂公式不在物料池主表格展开 |
| `fixed_usage_qty` | 固定用量；固定值模式下用于生成固定用量规则 |
| `production_remark` | 生产备注，在原料池行内维护 |
| `status/sort_order` | 状态与排序 |

配方原料页面统计由前端根据聚合数据即时计算：

| 指标 | 计算口径 |
| --- | --- |
| 属性分组卡片 | 按 `materials.attribute_group_code` 统计行数，展示面料、铝材、系统、配件、零件包、包装等当前物料分组 |
| 总数 | `materials.length` |
| 未设 | 当前物料行没有可用固定用量，或公式规则缺少默认规则/有效公式时计入；界面短文案为“未设”，完整含义为“未设用量” |
| 规则 | `restrictions.length`，表示业务配置项里的限制规则数量，不表示原料校验失败 |

`pc_formula_usage_rule` 是配方原料的用量规则。

| 字段 | 说明 |
| --- | --- |
| `formula_material_id/material_code` | 规则所属原料 |
| `condition_type` | `DEFAULT/OPTION_VALUE/EXPRESSION` |
| `condition_option_code/condition_value_code` | `OPTION_VALUE` 条件引用的配置项和值 |
| `condition_expression/condition_text/condition_key` | 表达式条件、展示文本和去重键 |
| `usage_mode` | `FIXED/FORMULA` |
| `fixed_usage_qty` | 固定用量 |
| `length_formula/width_formula/height_formula/weight_formula/usage_formula` | 原料用量相关公式；订单变量只提供宽(in)、长(in)及系统派生的宽(cm)、长(cm)、面积(m²) |
| `default_rule_flag` | 默认规则标记 |

当前前端用量设置主要使用 `DEFAULT` 和 `EXPRESSION`；后端仍保留 `OPTION_VALUE` 条件类型，用于表示“某配置项 = 某选项值”的结构化条件。

用量规则的“用量结果”只表达公式校验是否通过，不作为最终计算数量展示。最终用量在配方模拟中按命中规则、内部变量和 `loss_rate` 统一计算。

## 内部变量契约

内部变量是配方内可复用的计算结果，用于简化用量公式。它不是全局主数据，也不是业务配置项；随配方草稿维护，审核通过后进入版本快照。

`pc_formula_variable` 是内部变量主表。

| 字段 | 说明 |
| --- | --- |
| `variable_id` | 变量主键 |
| `formula_id` | 所属配方 |
| `variable_key` | 系统生成的隐藏引用标识，同一变量生命周期内稳定，公式内部保存为 `var_variable_key` |
| `variable_code` | 变量编码，同一配方内唯一，用于人工识别、搜索和复制，不作为公式唯一引用 |
| `variable_name` | 变量名称，用于技术科识别和前端外层摘要展示；允许重复，公式内部不靠名称定位 |
| `sort_order/remark` | 排序和备注 |

`pc_formula_variable_rule` 是变量取值规则表。

| 字段 | 说明 |
| --- | --- |
| `variable_id/variable_key/variable_code` | 所属变量；保存时优先按 `variable_key` 关联，`variable_code` 只做展示和兼容识别 |
| `condition_expression/condition_text` | 取值条件表达式和展示文本；默认规则可为空 |
| `value_type` | `FIXED/FORMULA` |
| `fixed_value` | 固定值 |
| `formula_expression/formula_text` | 公式值内部表达式和展示文本 |
| `default_rule_flag` | 默认规则标记，每个变量必须有一条默认兜底 |

内部变量规则：

- 同一配方内变量编码唯一；变量名称仅用于技术科识别和公式展示，可以重复。
- 每个变量必须至少有一条默认规则。
- 取值条件优先匹配，未命中时使用默认规则。
- 变量公式可引用订单变量和其它内部变量，内部保存格式为 `var_variable_key`；界面展示和录入可使用变量名称。
- 不允许引用不存在的变量，不允许循环引用。
- 删除变量前必须检查是否被其它变量公式或用量公式引用，被引用则不可删除。
- 配方模拟时先计算内部变量，再计算物料用量。
- 内部变量的取值条件只决定变量取什么值，不决定物料或用量规则是否生效。
- 物料是否使用、某条用量公式是否生效，必须放在 `pc_formula_usage_rule` 的使用条件中。

## 表达式契约

公式表达式只允许数字、变量、括号、`+ - * /` 和受控函数，不支持对象访问、`if`、`case` 或三元表达式。

公式变量当前支持。下单端输入美国单位 `in`，`cm` 和 `m²` 由系统自动派生：

| 内部变量 | 中文别名 | 说明 |
| --- | --- | --- |
| `orderWidthIn` | 订单宽(in)、订单宽、宽 | 模拟页输入 |
| `orderLengthIn` | 订单长(in)、订单长、长 | 模拟页输入 |
| `orderWidthCm` | 订单宽(cm)、宽cm | `orderWidthIn * 2.54` |
| `orderLengthCm` | 订单长(cm)、长cm | `orderLengthIn * 2.54` |
| `orderAreaM2` | 订单面积(m²)、订单面积、面积 | `orderWidthCm * orderLengthCm / 10000` |
| `var_variable_key` | 内部变量名称 | 来源于当前配方内部变量池；界面显示变量名称，保存时转换为隐藏稳定引用 |

受控函数当前支持：

| 内部函数 | 中文展示 | 说明 |
| --- | --- | --- |
| `round()` | `四舍五入()` | 可传精度，例如 `round(orderWidthCm / 3, 2)` |
| `ceil()` | `向上取整()` | 可传精度，例如 `ceil(orderWidthCm / 10.5, 2)` |
| `floor()` | `向下取整()` | 可传精度，例如 `floor(orderLengthCm / 10, 2)` |

条件表达式额外支持：

| 内部变量 | 说明 |
| --- | --- |
| `productType` | 配方产品类型 |
| `fabric` | 当前选择的 `FABRIC` 配置项值 |
| `option_配置项编码` | 当前选择的配置项值，例如 `option_SYSTEM` |
| `material_配置项编码_属性编码` | 当前配置项值关联物料上的结构化属性，例如 `material_FABRIC_THICKNESS` |
| `store`、`optionValue` | 校验器保留变量；当前模拟链路未作为主要输入 |

表达式必须保存后由后端再次校验。前端校验只用于即时提示。后端必须校验表达式引用是否仍存在：配置项变量必须来自当前配方配置项，配置项值必须属于被引用配置项，物料属性变量必须来自当前配置项值已关联的原料池物料。

条件编辑器中的“配置项组合”以当前配方配置项树为数据源，不要求用户手写 `option_` 变量名。编辑态通过前端 `clientKey` 定位草稿节点，保存和校验时落到稳定的 `optionCode/valueCode`。

限制规则、用量条件和变量取值条件可以读取配置项值关联物料上的结构化属性。属性定义来自基础信息的 `pc_base_attribute`，具体数值来自物料资料的 `pc_material_attribute`。例如“面料厚度 + 副面料厚度 > 12”读取的是订单当前选择的面料和副面料所关联物料的 `THICKNESS` 数值；“罩壳重量 > 2.5”读取的是罩壳关联物料的 `WEIGHT` 数值。界面必须通过条件编辑器生成内部变量，不要求用户手写 `material_` 变量名。

## 业务配置项契约

| 表 | 说明 |
| --- | --- |
| `pc_formula_option` | 配置项定义，例如系统、电机型号、遥控型号 |
| `pc_formula_option_value` | 配置项可选值，例如拉珠、无拉、电机 |
| `pc_formula_option_material` | 某个选项值自动带出的生产物料 |
| `pc_formula_restriction` | 限制规则，例如宽度过大时禁用某个选项 |

配置项和值都维护中英文名称：

| 字段 | 说明 |
| --- | --- |
| `option_name_cn/option_name_en` | 配置项中文名和英文名；中文给工厂端识别，英文给美国客户下单展示 |
| `value_name_cn/value_name_en` | 选项值中文名和英文名；中文给工厂端识别，英文给美国客户下单展示 |

当前页面结构：

- 顶部统计展示配置项数、选项值数、关联物料数、限制规则数和待完善数；“规则”统计指限制规则，不等同于系统错误。
- 限制规则放在选项树上方，是全局风险规则区。
- 下方工作区左侧是选项树，右侧是当前选中配置项的设置和选项值表格。

配置项来源类型：

| 来源类型 | 用法 | 选项值维护 |
| --- | --- | --- |
| `MATERIAL_POOL` | 从当前配方原料池选入一组物料作为可选值，例如面料、罩壳 | 通过“导入选项值”批量选择原料池物料；中文名来自物料，不在表格里手工改 |
| `PRODUCT_DICT` | 来源于产品字典的业务枚举，当前页面作为来源类型保存 | 通过选项值表格维护值，后续如接入字典选择器仍必须落到选项值 |
| `BOOLEAN` | 是/否、包/不包、静音/不静音等纯选择 | 手工维护选项值，不强制关联物料 |
| `MANUAL` | 技术科手工维护的业务选项，例如系统、遥控型号 | 手工维护选项值，可按需关联原料池物料 |

配置项树规则：

- 根配置项默认展示，`visibility_mode = ALWAYS`。
- 选中某个值后新增下级配置项，会保存为 `visibility_mode = CONDITIONAL`，并记录父配置项和值。
- 树节点支持新增一级、新增下级、删除、上移、下移；配置项节点和值节点都可排序。
- 配置项名称和值名称都分中英文；中文服务技术科和工厂识别，英文服务美国客户下单展示。
- `optionCode/valueCode` 是保存后的业务编码；前端编辑态不依赖用户输入顺序做关联。

选项值和关联物料规则：

- 选项值可以不关联物料，也可以关联一个或多个当前配方原料池物料。
- 来源物料池的末级选项值必须至少关联一个原料池物料；该类选项值通常由导入原料池物料自动生成并自动建立关联。
- 布尔、手工或产品字典来源的选项值可以作为纯选择值，不强制关联物料，例如“包布 = 不包”。
- 任意来源的选项值如果主动关联物料，关联物料必须来自当前配方原料池，不能手写基础物料编码绕过原料池。
- 有下级配置项的父级值可以只负责展开后续选择，不要求自身直接关联物料。

限制规则：

- 限制条件统一使用条件编辑器生成表达式，保存为 `condition_type = EXPRESSION`。
- 目标支持整单、某个配置项、某个配置项值；目标为空表示整单。
- `DISABLE` 表示禁用或拦截，`WARN` 表示提醒。
- 限制规则只表达超宽、超重或特殊组合的风险；正常分支展示由配置项树的 `visibility_mode = CONDITIONAL` 表达。

前端草稿身份规则：

- 配方选项页编辑的是整页草稿，新增、删除、拖动、批量导入和恢复本地草稿都必须使用前端隐藏身份字段关联行数据。
- `optionCode`、`valueCode` 是业务编码，保存后给后端和后续模拟、审核、快照使用；前端不能把 `optionCode` 或 `OPTION_${数量+1}` 当成编辑态唯一身份。
- 前端草稿使用隐藏 `clientKey` 区分配置项、选项值和关联物料。配置项值、关联物料、限制规则、当前选中树节点都应通过 `clientKey` 做编辑态关联。
- 条件编辑器里的结构化选择也属于编辑态关联：配置项组合、选项物料属性和限制目标选择必须先用 `clientKey` 定位草稿节点，再生成或转换为后端表达式中的 `optionCode/valueCode`。
- 保存前由前端统一整理 payload：补齐缺失业务编码、修正草稿内重复编码、把 `clientKey` 关联转换为后端需要的 `optionCode/valueCode`，并剔除前端临时字段。
- 后端仍以数据库主键和业务编码为事实来源，不持久化前端 `clientKey`。
- 删除配置项或选项值前必须清理引用。被下级配置项、默认值、限制规则、用量规则、内部变量取值规则或关联物料引用时，前端应阻止明显删除，后端保存选项必须再次拒绝非法引用。

配置项正常分支由显示条件表达：根配置项始终显示；子配置项通过 `visibility_mode = CONDITIONAL` 和父配置项值控制展示。

限制规则只做禁用或提醒，不承担正常分支显示。限制条件统一使用表达式，目标可以是整单，也可以是某个配置项值。

`pc_formula_restriction` 当前核心字段：

| 字段 | 说明 |
| --- | --- |
| `restriction_name` | 限制说明，给技术科识别规则用途 |
| `condition_type` | 当前前端统一保存为 `EXPRESSION` |
| `condition_expression/condition_text` | 后端表达式和前端展示文本，例如 `orderWidthIn > 98` 或 `option_CONTROL_SYSTEM == "PULL_CHAIN"` |
| `action_type` | `DISABLE/WARN`，禁用或提醒 |
| `target_option_code/target_value_code` | 目标配置项和值；为空表示作用于整单 |
| `message_text` | 模拟或后续下单时展示的提示文本 |
| `status/sort_order` | 状态与排序 |

## 配方模拟契约

`ProductFormulaSimulationBO` 当前接口输入。接口字段为 `orderWidth/orderHeight`，业务语义分别为订单宽(in)、订单长(in)；`cm` 和 `m²` 不由用户提交，由模拟引擎自动派生到表达式上下文。

| 字段 | 说明 |
| --- | --- |
| `orderWidth` | 订单宽(in)，模拟页输入 |
| `orderHeight` | 订单长(in)，模拟页输入 |
| `orderQuantity` | 模拟数量，默认 `1`，必须大于 `0` |
| `room` | 房间或备注，用于模拟样例识别 |
| `selectedOptionValues` | 已选择配置项值，key 为配置项编码；单选保存一个值，多选用英文逗号分隔 |
| `saveResult` | 是否保存模拟结果，当前页面不作为主要入口 |
| `remark` | 备注 |

模拟引擎派生表达式上下文：

| 表达式变量 | 来源 |
| --- | --- |
| `orderWidthIn` | `orderWidth` |
| `orderLengthIn` | `orderHeight` |
| `orderWidthCm` | `orderWidth * 2.54` |
| `orderLengthCm` | `orderHeight * 2.54` |
| `orderAreaM2` | `orderWidthCm * orderLengthCm / 10000` |

模拟引擎会按以下顺序执行：

1. 校验订单宽、订单长和数量。
2. 校验订单尺寸是否在配方档案的最小/最大宽高范围内。
3. 根据当前已选配置项值计算可见配置项；可见且必填的配置项必须选择。
4. 校验已选择值必须属于当前配置项的启用选项值。
5. 先带出原料池中启用且 `required_flag = true` 的生产必带物料。
6. 再按已选配置项值匹配 `pc_formula_option_material`，带出选项值关联的生产物料。
7. 生成表达式上下文，包含订单变量、配置项变量、当前选项值关联物料属性变量和内部变量。
8. 校验限制规则；命中 `DISABLE` 限制时模拟失败，返回限制消息。
9. 按物料用量规则计算 BOM 明细、单件金额和总金额。

`ProductFormulaSimulationVO` 当前返回：

| 字段 | 说明 |
| --- | --- |
| `formulaId` | 配方 ID |
| `status` | `PASS/FAIL/NOT_VALIDATED` |
| `message` | i18n 消息 key 或限制提示 |
| `simulationTime` | 模拟时间 |
| `orderWidth/orderHeight/orderQuantity/room` | 本次模拟输入回显 |
| `selectedOptionValues` | 本次模拟选项回显 |
| `items` | BOM 明细 |
| `singleAmount` | 单件金额 |
| `totalAmount` | `singleAmount * orderQuantity` |

`ProductFormulaSimulationItemVO` 当前返回：

| 字段 | 说明 |
| --- | --- |
| `formulaMaterialId/materialId` | 配方原料行和基础物料 ID |
| `materialCode/materialNameCn/specModelText` | 物料编码、名称和规格 |
| `attributeGroupNameCn/materialTypeNameCn` | 属性分组和物料类型 |
| `unitCode` | 主单位编码，前端按单位资料展示名称 |
| `usageQty/lossRate` | 计算用量和损耗率 |
| `unitPrice/salesPrice/amount` | 成本单价、销售价和金额 |
| `usageSummary` | 命中的用量规则摘要 |
| `productionRemark` | 生产备注 |

模拟校验接口 `PUT /product-formula/formulas/{id}/simulation/validate` 会写入 `pc_formula.simulation_validation_status/message/time`，并同步汇总校验状态。该接口只允许草稿或驳回状态调用；调用前必须先通过配方原料和业务配置项校验。

配方模拟页控件规则：

| 配置项来源/模式 | 控件 |
| --- | --- |
| `source_type = BOOLEAN` 或 `selection_mode = SWITCH` | 开关 |
| `selection_mode = SINGLE` | 可搜索下拉 |
| `selection_mode = MULTIPLE` | 可搜索多选下拉；提交到 `selectedOptionValues` 时以英文逗号分隔 |
| `source_type = MATERIAL_POOL` | 可搜索下拉或多选下拉；选项内展示物料摘要；如果接口提供物料图片地址则展示真实缩略图，否则展示项目默认物料缩略图，不使用文字假图占位 |

除布尔/开关外，模拟页不把少量选项渲染成按钮块，也不为单选项打开选择弹窗；它应保持下单式录入体验。配置项按根配置项分组展示，子配置项跟随触发它的父配置项所在分组展示。

## 校验规则

- 配方编号在同租户未删除数据中唯一，由 Service 层校验。
- 配方名称、产品分类、产品类型和尺寸范围组合在同租户未删除数据中唯一，由 Service 层校验。
- 产品分类必须存在且启用。
- 产品类型必须来自启用的产品字典项。
- 配方原料、业务配置项和模拟只能在草稿或驳回状态保存。
- 原料池只能选择启用物料，且同一配方不能重复选择同一物料。
- 选项值关联物料必须来自当前配方原料池。
- 来源物料池的末级选项值必须至少关联一个原料池物料；如果该值下面还有下级配置项，可以只负责展开后续选择。布尔、手工或产品字典来源的选项值可以作为纯选择值，不强制关联物料。
- 配方原料存在公式规则时，每个物料必须有一条默认规则。
- 固定值模式只允许一条固定用量规则。
- 表达式条件和用量公式保存前必须通过后端表达式校验。
- 限制规则、用量规则和变量取值规则中的表达式引用必须存在；已删除或改名的配置项、选项值、关联物料属性不能通过保存或校验。
- 业务配置项编码在同一配方下唯一。
- 业务配置项值在同一配置项下唯一。
- 条件展示的配置项不能引用自己，条件值必须属于被引用的配置项。
- 限制规则条件使用表达式，表达式变量必须来自订单变量、当前业务配置项或当前配方原料池。
- 限制规则目标为空表示整单限制；目标为配置项值时，配置项和值必须属于当前配方。
- 配方模拟确认通过前，配方原料和业务配置项必须已校验通过。
- 配方模拟必须校验订单宽、订单长、数量、尺寸范围、可见必填配置项、选项值合法性、禁用限制和 BOM 明细非空。
- 提交审核前必须完成配方汇总校验；审核通过后生成版本快照。

## 版本快照

审核通过时生成 `pc_formula_version`。版本快照必须冻结配方主档、配方原料、用量规则、内部变量和变量取值规则、业务配置项、选项值、选项关联物料、限制规则、校验结果和审核信息。

审核通过后的报价、订单、生产快照应引用 `formula_version_id`，避免草稿修改影响历史业务。
