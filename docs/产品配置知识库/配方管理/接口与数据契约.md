# 配方管理接口与数据契约

## 模块范围

配方管理当前包含五个业务模块：配方档案、配方原料、业务配置项、配方模拟、配方审核。

条件编辑器、公式编辑器、物料选择弹窗和用量设置弹窗只是模块内辅助能力，不单独作为业务模块。

## 页面路由

| 模块 | 页面 | 说明 |
| --- | --- | --- |
| 配方档案 | `/product-formula/formulas` | 配方列表、新增、编辑、版本、引用检查 |
| 配方原料 | `/product-formula/formulas/materials` | 原料池、生产必带、生产备注、用量规则 |
| 业务配置项 | `/product-formula/formulas/options` | 客户可选项、选项值、关联物料、异常限制 |
| 配方模拟 | `/product-formula/formulas/{id}/simulation` | 按订单宽高和配置选项模拟 BOM |
| 配方审核 | `/product-formula/reviews` | 审核列表和审核详情 |

## 主表契约

`pc_formula` 是配方档案主表。

| 字段 | 说明 |
| --- | --- |
| `formula_id` | 配方主键 |
| `formula_code` | 配方编号，程序接口控制唯一 |
| `formula_name` | 配方名称 |
| `category_id/category_code/category_name_cn` | 产品分类冗余字段 |
| `product_type_code/product_type_name_cn` | 产品类型，来源于产品字典 |
| `min_width_inch/min_height_inch` | 最小宽高，当前按英寸保存 |
| `max_width_inch/max_height_inch` | 最大宽高，当前按英寸保存 |
| `current_version_id/current_version_no/current_version_label` | 当前生效版本 |
| `draft_version_no` | 草稿版本预览号 |
| `latest_validation_status/latest_validation_message/latest_validation_time` | 最近汇总校验结果 |
| `material_validation_status/material_validation_message/material_validation_time` | 最近原料校验结果 |
| `option_validation_status/option_validation_message/option_validation_time` | 最近选项校验结果 |
| `simulation_validation_status/simulation_validation_message/simulation_validation_time` | 最近模拟校验结果 |
| `status` | `DRAFT/PENDING_REVIEW/REJECTED/EFFECTIVE/STOPPED` |

## 配方原料契约

`pc_formula_material` 是配方原料池主行。

| 字段 | 说明 |
| --- | --- |
| `formula_material_id` | 原料行主键 |
| `formula_id` | 所属配方 |
| `material_id/material_code/material_name_cn` | 选入物料 |
| `spec_model_text` | 规格型号冗余展示 |
| `attribute_group_code/attribute_group_name_cn` | 物料属性分组 |
| `material_type_code/material_type_name_cn` | 物料类型 |
| `unit_code` | 物料主单位 |
| `default_flag` | 默认带出标记，当前前端主表格不作为核心展示字段 |
| `required_flag` | 生产必带：命中配置时生产必须包含，不等于客户可见 |
| `usage_mode` | 用量方式，当前收敛为 `FIXED/FORMULA` |
| `production_remark` | 生产备注 |
| `status/sort_order` | 状态与排序 |

`pc_formula_usage_rule` 是配方原料的用量规则。

| 字段 | 说明 |
| --- | --- |
| `formula_material_id/material_code` | 规则所属原料 |
| `condition_type` | `DEFAULT/OPTION_VALUE/EXPRESSION` |
| `condition_option_code/condition_value_code` | `OPTION_VALUE` 条件引用的配置项和值 |
| `condition_expression/condition_text/condition_key` | 表达式条件、展示文本和去重键 |
| `usage_mode` | `FIXED/FORMULA` |
| `fixed_usage_qty` | 固定用量 |
| `length_formula/width_formula/height_formula/weight_formula/usage_formula` | 长度、宽度、高度、重量和通用用量公式 |
| `loss_rate` | 损耗率 |
| `default_rule_flag` | 默认规则标记 |

当前前端用量设置主要使用：

- `DEFAULT`：默认规则，作为没有命中其他条件时的兜底。
- `EXPRESSION`：表达式条件，例如订单宽高或业务配置项值判断。

后端仍保留 `OPTION_VALUE` 条件类型，用于表示“某配置项 = 某选项值”的结构化条件。

## 表达式契约

公式表达式只允许数字、变量、括号和 `+ - * /`，不支持函数、对象访问、`if`、`case` 或三元表达式。

公式变量当前支持：

| 内部变量 | 中文别名 | 说明 |
| --- | --- | --- |
| `orderWidth` | 订单宽、宽 | 模拟页输入 |
| `orderHeight` | 订单高、订单厚、高、厚度 | 模拟页输入 |
| `orderLength` | 订单长、长 | 当前模拟中按订单宽带入 |
| `orderArea` | 订单面积、面积 | 当前按订单宽乘订单高生成 |
| `orderWeight` | 订单重量、重量 | 当前模拟中默认 `0` |

条件表达式额外支持：

| 内部变量 | 说明 |
| --- | --- |
| `productType` | 配方产品类型 |
| `fabric` | 当前选择的 `FABRIC` 配置项值 |
| `option_配置项编码` | 当前选择的配置项值，例如 `option_SYSTEM` |
| `material_属性分组编码_materialType` | 当前配方原料池中某属性分组的物料类型条件，例如面料.物料类型 |
| `material_属性分组编码_materialCode` | 当前配方原料池中某属性分组的物料编码条件，当前界面暂不前置 |
| `material_属性分组编码_materialName` | 当前配方原料池中某属性分组的物料名称条件，当前界面暂不前置 |
| `store`、`optionValue` | 校验器保留变量；当前模拟链路未作为主要输入 |

表达式必须保存后由后端再次校验。前端校验只用于即时提示。

条件编辑器中的“原料池条件”以当前配方原料池为数据源，属性分组和物料类型均从已选入原料池的物料快照聚合，不要求用户手写内部变量或全系统物料编码。

## 业务配置项契约

| 表 | 说明 |
| --- | --- |
| `pc_formula_option` | 配置项定义，例如系统、电机型号、遥控型号 |
| `pc_formula_option_value` | 配置项可选值，例如拉珠、无拉、电机 |
| `pc_formula_option_material` | 某个选项值自动带出的生产物料 |
| `pc_formula_restriction` | 异常限制，例如宽度过大时禁用某个选项 |

配置项正常分支由显示条件表达：根配置项始终显示；子配置项通过 `visibility_mode = CONDITIONAL` 和父配置项值控制展示。

异常限制只做禁用或提醒，不承担正常分支显示。

## 配方模拟契约

`ProductFormulaSimulationBO` 当前输入：

| 字段 | 说明 |
| --- | --- |
| `orderWidth` | 订单宽 |
| `orderHeight` | 订单高 |
| `selectedOptionValues` | 已选择配置项值，key 为配置项编码 |
| `saveResult` | 是否保存模拟结果 |
| `remark` | 备注 |

模拟引擎会根据生产必带原料和已选选项值关联物料生成模拟 BOM，并用当前用量规则计算用量。

## 校验规则

- 配方编号在同租户未删除数据中唯一，由 Service 层校验。
- 配方名称、产品分类、产品类型和尺寸范围组合在同租户未删除数据中唯一，由 Service 层校验。
- 产品分类必须存在且启用。
- 产品类型必须来自启用的产品字典项。
- 配方原料、业务配置项和模拟只能在草稿或驳回状态保存。
- 原料池只能选择启用物料，且同一配方不能重复选择同一物料。
- 选项值关联物料必须来自当前配方原料池。
- 配方原料存在公式规则时，每个物料必须有一条默认规则。
- 固定值模式只允许一条固定用量规则。
- 表达式条件和用量公式保存前必须通过后端表达式校验。
- 业务配置项编码在同一配方下唯一。
- 业务配置项值在同一配置项下唯一。
- 条件展示的配置项不能引用自己，条件值必须属于被引用的配置项。
- 异常限制只能引用当前配方存在的配置项和值。
- 提交审核前必须完成配方汇总校验；审核通过后生成版本快照。

## 版本快照

审核通过时生成 `pc_formula_version`。版本快照必须冻结：

- 配方主档快照
- 配方原料快照
- 用量规则快照
- 业务配置项快照
- 选项值、选项关联物料、异常限制快照
- 校验结果和审核信息

审核通过后的报价、订单、生产快照应引用 `formula_version_id`，避免草稿修改影响历史业务。
