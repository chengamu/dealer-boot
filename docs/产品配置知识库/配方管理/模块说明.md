# 配方管理模块说明

## 总体口径

配方管理用于技术科把基础信息中的产品分类、产品字典、单位、物料、物料类型和物料属性组织成可校验、可模拟、可审核、可冻结版本的产品配方。

配方管理不重复维护基础资料。基础物料、单位、厂家、物料类型和物料属性仍回到基础信息维护；配方里只做选择、组织、规则配置、模拟和审核冻结。

## 模块清单

| 模块 | 前端入口 | 后端入口 | 主要职责 |
| --- | --- | --- | --- |
| 配方档案 | `FormulaPage.vue` | `ProductFormulaController` | 维护配方编号、名称、产品分类、产品类型、尺寸范围、状态、校验状态和版本入口 |
| 配方原料 | `FormulaMaterialPage.vue` | `ProductFormulaMaterialController`、`ProductFormulaSetupService`、`ProductFormulaUsageRuleService` | 维护原料池、生产必带、生产备注和用量规则 |
| 业务配置项 | `FormulaOptionPage.vue` | `ProductFormulaOptionController` | 维护客户下单可选项、选项值、选项值关联生产物料和异常限制 |
| 配方模拟 | `FormulaSimulationPage.vue` | `ProductFormulaSimulationController` | 按订单宽(in)、订单长(in)和配置项选择模拟 BOM、用量和校验结果 |
| 配方审核 | `FormulaReviewPage.vue`、`FormulaReviewDetailPage.vue` | `ProductFormulaReviewController` | 审核、驳回、审核通过后冻结版本快照 |
| 版本快照 | 配方档案版本入口 | `ProductFormulaReviewLifecycle`、`ProductFormulaSnapshotJson` | 冻结配方主档、原料、内部变量、选项、限制和用量规则 |
| 变更记录 | 通用变更记录入口 | `pc_change_log` | 记录新增、修改、校验、提交审核、审核、驳回、停用等关键动作 |

## 辅助能力

以下能力不是独立模块，也不作为单独菜单：

- 物料选择弹窗：用于从物料管理选入配方原料，或从当前原料池中选择选项值关联物料。
- 用量设置弹窗：配方原料行内工具，用于维护该物料的固定用量或公式规则。
- 条件编辑器：用于维护用量规则的适用条件，当前支持默认规则和表达式条件。
- 公式编辑器：用于维护原料用量相关公式，并做表达式语法校验。
- 内部变量池：配方内可复用的计算结果，不作为独立菜单；在公式选择器右侧 grid 中维护和引用，订单变量固定在底部快捷区。内部变量只表达取值，不表达物料或用量规则是否生效。

## 业务边界

- 配方档案决定“这是什么配方”。
- 配方原料决定“生产可能用哪些物料、怎么算用量”。
- 内部变量决定“哪些中间计算结果需要复用，避免公式重复写复杂取值”。
- 业务配置项决定“客户下单能选什么、选了之后带出哪些生产物料”。
- 配方模拟决定“按一组样例订单是否能算出合理 BOM”。
- 配方审核决定“当前草稿是否冻结成可被后续报价、订单引用的版本”。

## 状态口径

配方主档使用单状态：

| 状态 | 含义 |
| --- | --- |
| `DRAFT` | 草稿，可编辑、可删除、可配置原料和选项 |
| `PENDING_REVIEW` | 待审核，不可普通编辑 |
| `REJECTED` | 驳回，可重新编辑并校验 |
| `EFFECTIVE` | 生效，审核通过后进入；不可普通编辑，可停用 |
| `STOPPED` | 停用，不再重新启用；继续使用时新建或复制草稿重新审核 |

## 版本口径

- 草稿数据保存在配方主档、原料、内部变量、业务配置项、异常限制和用量规则表中。
- 审核通过时生成 `pc_formula_version` 冻结快照。
- 后续报价、订单和生产快照只应引用审核通过的配方版本，不直接引用草稿。
