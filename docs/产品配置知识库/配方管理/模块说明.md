# 配方管理模块说明

## 总体口径

配方管理用于技术科把基础信息中的产品分类、产品字典、单位、物料、物料类型和物料属性组织成可校验、可模拟、可审核、可冻结版本的产品配方。

配方管理不重复维护基础资料。基础物料、单位、厂家、物料类型和物料属性仍回到基础信息维护；配方里只做选择、组织、规则配置、模拟和审核冻结。

## 模块清单

| 模块 | 前端入口 | 后端入口 | 主要职责 |
| --- | --- | --- | --- |
| 配方档案 | `FormulaPage.vue` | `ProductFormulaController` | 维护配方编号、名称、产品分类、产品类型、尺寸范围、状态、校验状态和版本入口 |
| 配方工作台 | `FormulaWorkbenchPage.vue` | `ProductFormulaController`、既有原料/选项/模拟/审核接口 | 汇总配方修订入口、配置步骤、版本历史和版本快照；从这里进入原料、配置项、模拟和待审核详情 |
| 配方原料 | `FormulaMaterialPage.vue` 包装 `FormulaSetupPage.vue` | `ProductFormulaMaterialController`、`ProductFormulaSetupService`、`ProductFormulaUsageRuleService` | 维护原料池、损耗率、生产备注和用量规则 |
| 业务配置项 | `FormulaOptionPage.vue` 包装 `FormulaSetupPage.vue` | `ProductFormulaOptionController` | 维护客户下单可选项、选项值、选项值关联生产物料和限制规则 |
| 配方模拟 | `FormulaSimulationPage.vue` | `ProductFormulaSimulationController` | 按订单宽(in)、订单高(in)和配置项选择模拟 BOM、用量和校验结果 |
| 配方审核 | `FormulaReviewPage.vue`、`FormulaReviewDetailPage.vue`、`FormulaReviewRecordDrawer.vue` | `ProductFormulaReviewController`、`ProductFormulaReviewQueryService`、`ProductFormulaReviewRecordQueryService`、`ProductFormulaReviewLifecycle` | 查询待审核快照、审核、驳回、审核通过后冻结版本快照，并查看审核记录 |
| 版本快照 | 配方档案版本入口 | `ProductFormulaReviewLifecycle`、`ProductFormulaSnapshotJson` | 冻结配方主档、原料、内部变量、选项、限制和用量规则 |
| 变更记录 | 配方列表通用变更记录入口 | `pc_change_log` | 记录配方档案、原料、选项等普通新增、修改、保存、校验等操作流水；不作为配方审核页审核记录数据源 |

## 辅助能力

以下能力不是独立模块，也不作为单独菜单：

- 物料选择弹窗：用于从物料管理选入配方原料，或从当前原料池中选择选项值关联物料。
- 用量设置弹窗：配方原料行内工具，用于维护该物料的固定用量或公式规则。
- 条件编辑器：用于维护用量规则的适用条件，当前支持默认规则和表达式条件；弹窗使用受控宽度和紧凑左右布局，避免大面积空白。
- 公式编辑器：用于维护原料用量相关公式，并做表达式语法校验；左侧写公式和看校验结果，右侧内部变量池只做查看和插入。
- 内部变量池：配方内可复用的计算结果，不作为独立菜单；在用量设置中统一维护，公式选择器右侧只用于查看和插入变量，订单变量固定在底部快捷区。内部变量只表达取值，不表达物料或用量规则是否生效。

## 业务边界

- 配方档案决定“这是什么配方”。
- 配方原料决定“生产可能用哪些物料、怎么算用量”。
- 内部变量决定“哪些中间计算结果需要复用，避免公式重复写复杂取值”。
- 业务配置项决定“客户下单能选什么、选了之后带出哪些生产物料”。
- 配方模拟决定“按一组样例订单是否能算出合理 BOM”。
- 配方审核决定“当前草稿是否冻结成可被后续报价、订单引用的版本”。
- 审核记录决定“当前配方提交、审核、驳回等关键动作如何追溯”，数据来自 `pc_formula_version` 中的提交和审核快照字段。
- 配方工作台决定“同一个配方如何继续修订、撤回和查看历史快照”，列表页保留工作台和模拟快捷入口，不继续堆复杂配置动作。

## 状态口径

配方主档使用单状态：

| 状态 | 含义 |
| --- | --- |
| `DRAFT` | 草稿，可编辑、可删除、可配置原料和选项 |
| `PENDING_REVIEW` | 待审核，不可普通编辑 |
| `REJECTED` | 驳回，可重新编辑并校验 |
| `EFFECTIVE` | 生效，审核通过后进入；不可普通编辑，可停用；未被真实业务引用前可撤回为草稿继续调整 |
| `STOPPED` | 停用，不再重新启用；继续使用时新建或复制草稿重新审核 |

## 版本口径

- 草稿数据保存在配方主档、原料、内部变量、业务配置项、限制规则和用量规则表中。
- 审核通过时生成 `pc_formula_version` 冻结快照。
- 后续报价、订单和生产快照只应引用审核通过的配方版本，不直接引用草稿。
- 配方审核列表只展示 `pc_formula_version.version_status = PENDING_REVIEW` 的待审核快照。
- 版本快照由 `ProductFormulaSnapshotJson` 统一输出，只保留业务字段；草稿行审计字段不进入快照。已有历史快照中的审计字段由读取方兼容忽略。
- 审核详情和版本快照共用冻结快照物料明细展示口径：按属性分组排序和合并，`用量汇总` 展示固定值、默认规则、条件规则和公式摘要，不展示规则数量占位。
- 审核记录不等同于普通修改记录；审核记录由 `pc_formula_version` 的提交人、提交时间、审核人、审核时间、审核状态和驳回原因生成，用于追溯提交审核、审核通过、驳回等审核流水。
- 生效配方需要继续调整时，优先在同一个配方下“开始修订”，系统从当前生效版本快照恢复草稿数据，并把草稿版本号设为下一正式版本预览，例如 `V2 草稿`。
- `PENDING_REVIEW` 可以撤回为草稿；`EFFECTIVE` 在当前版本未被销售产品、价格设置或未来真实 SKU、订单、订单快照引用前可以撤回为草稿。已被真实业务引用时永久阻断撤回，只能开始修订生成下一版本。
- 撤回生效版本会保留原版本快照记录，并清空当前生效版本指针；重新提交审核通过后，会在同一个配方下固化新的正式版本。
- 复制配方用于创建另一个新配方草稿，必须录入新的配方编号和名称；它不是日常修订方式，也不复用原配方编号。
