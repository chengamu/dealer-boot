# 配方管理操作手册

## 录入顺序

1. 在基础信息完成产品分类、产品字典、单位、厂家、物料类型、物料属性和物料管理。
2. 在配方档案新增配方，填写编号、名称、产品分类、产品类型和尺寸范围。
3. 在配方原料选入生产可能用到的物料，维护生产备注和用量规则。
4. 在业务配置项维护客户下单时能选择的配置项、选项值、关联生产物料和限制规则。
5. 在配方模拟输入样例宽高和选项，确认 BOM 与用量结果。
6. 校验通过后提交审核。
7. 审核通过后生成版本快照，供后续报价、订单和生产引用。

## 配方档案

配方档案只维护配方主信息，不维护物料明细。

必填信息：

- 配方编号
- 配方名称
- 产品分类
- 产品类型
- 尺寸范围

产品类型来自产品字典，不在配方里手工造值。

## 配方原料

配方原料是技术科确认生产清单的核心入口。

页面顶部统计用于快速判断当前配方原料配置是否完整：

| 指标 | 含义 |
| --- | --- |
| 面料/铝材/系统/配件/零件包/包装 | 当前原料池中各属性分组的物料行数 |
| 总数 | 当前原料池物料总行数 |
| 未设 | 未完成用量配置的物料行数，鼠标悬停展示完整含义“未设用量” |
| 规则 | 当前业务配置项中的限制规则数量，不代表校验失败 |

### 物料来源

- 只能从物料管理选入已维护物料。
- 不在配方原料里新增基础物料。
- 物料类型、属性分组、物料中文名称、物料英文名称、型号、规格、主单位等来自基础信息。

### 原料字段

| 字段 | 用法 |
| --- | --- |
| 属性分组 | 物料所属大类，例如面料、铝材、系统、配件 |
| 物料类型 | 物料小类，例如遮光面料、电机、下杆 |
| 物料编码/中文名称/英文名称/规格 | 用于技术科确认是否选对物料 |
| 单位 | 展示物料主单位 |
| 损耗率 | 在物料池行内维护，默认按百分比录入；用量设置里不重复维护 |
| 生产必带 | 命中配置时生产必须包含；这是生产属性，不代表客户可见 |
| 用量汇总 | 展示固定值或公式规则摘要；未配置时展示未设置 |
| 生产备注 | 给生产端看的说明，可直接在物料池行内维护 |
| 排序 | 控制原料池展示顺序，先按属性分组，再按排序和物料编码排列 |

### 用量设置

用量规则从原料行的“用量”进入。物料池 grid 只做材料确认、损耗率、生产备注和排序，不在主表格里展开复杂公式。

当前用量方式：

| 用量方式 | 用法 |
| --- | --- |
| 固定值 | 始终固定数量的物料，只维护一个固定用量 |
| 公式规则 | 按默认规则或表达式条件维护多条公式规则 |

打开多行“批量用量设置”时，抽屉顶部展示当前选择的物料，切换物料后维护对应物料自己的用量规则。

固定值示例：

- 安装码固定 `2.00` 个。
- 包装袋固定 `1.00` 个。

公式规则示例：

- 默认规则：公式为 `1`
- 表达式条件：`订单宽(in) > 120` 时公式为 `2`
- 表达式条件：`系统 = 电机` 时使用电机相关物料公式

当前公式规则不支持把 `if/case` 写进一个公式里。分支优先放到多条规则的“使用条件”中，每个公式只表达一条计算方式。

用量设置里的“用量结果”只用于提示当前公式是否校验通过，不展示最终生产数量。最终数量、损耗后的用量和 BOM 结果在配方模拟中查看。

损耗率不改变公式本身。模拟时先按固定值或公式规则计算基础用量，再按物料池行内维护的损耗率做加成。

### 条件和公式

公式编辑器支持中文变量录入，并转换成内部变量保存。下单端录入美国单位 `in`，系统自动派生 `cm` 和 `m²` 变量：

| 中文变量 | 内部变量 | 当前使用说明 |
| --- | --- | --- |
| 订单宽(in) | `orderWidthIn` | 模拟页输入字段 |
| 订单长(in) | `orderLengthIn` | 模拟页输入字段 |
| 订单宽(cm) | `orderWidthCm` | `订单宽(in) * 2.54` |
| 订单长(cm) | `orderLengthCm` | `订单长(in) * 2.54` |
| 订单面积(m²) | `orderAreaM2` | `订单宽(cm) * 订单长(cm) / 10000` |

条件表达式还支持两类业务条件：

- 业务配置项选择值：系统会把选中的配置项值放入 `option_配置项编码` 变量中，例如可表达“系统 = 电机”。
- 选项关联物料属性：来源于当前订单选中的配置项值所关联的具体物料属性，例如“面料厚度 + 副面料厚度 > 12”或“罩壳重量 > 2.5”。这些属性必须先在基础信息的物料属性中定义，并在物料资料里填具体值。

原料池只提供可被选入和关联的物料范围，不直接作为下单条件。按面料、铝材、系统等差异做扣减或限制时，应优先通过配置项选择值和该值关联物料的结构化属性表达。

### 内部变量池

内部变量用于把一段会反复使用的计算结果先命名，再在用量公式里引用。

典型例子：

- `铝材扣减`：不同系统或面料类型下扣减值不同。
- `走珠数量`：按订单宽(cm)计算并取整。
- `安装码数量`：按订单宽(cm)或安装方式计算。

内部变量跟随当前配方，不做全局共享。后续可以从已有配方复制变量池到当前配方，再按当前产品调整。

公式选择器中的使用方式：

- 左侧编写当前公式。
- 右侧展示当前配方内部变量池，只展示“变量”和“取值摘要”。
- 双击内部变量行可插入公式。
- 底部固定展示订单变量快捷按钮，例如订单宽(in)、订单长(in)、订单宽(cm)、订单长(cm)、订单面积(m²)。

变量采用“变量 + 取值规则”模型：

| 内容 | 用法 |
| --- | --- |
| 变量名称 | 技术科可读名称，例如铝材扣减 |
| 变量编码 | 技术科识别和搜索用编码；保存公式时系统改用隐藏稳定引用 |
| 默认规则 | 没有命中其它条件时使用 |
| 取值条件 | 命中条件时覆盖默认值 |
| 取值方式 | 固定值或公式值 |

变量公式支持订单变量和前面已定义的内部变量，也支持 `四舍五入()`、`向上取整()`、`向下取整()`。保存时系统转换为 `round()`、`ceil()`、`floor()`。

用量公式外层摘要继续展示技术科输入口径，例如 `订单宽(cm) - 铝材扣减`；内部表达式保存为 `orderWidthCm - var_隐藏变量引用`。

内部变量的取值条件只用于决定变量取值，不用于决定某个物料是否进入生产清单。物料或用量规则是否生效，必须维护在用量规则的“使用条件”中。

推荐拆法：

- 扣减值、走珠数量、安装码数量这类公共计算参数，放内部变量。
- “系统 = 拉珠 时才使用这条物料”这类生效判断，放用量规则使用条件。
- 不要把同一个业务分支同时写在变量取值条件和用量使用条件里；若确实需要组合，变量未命中取值条件时会走默认规则。

## 业务配置项

业务配置项决定客户下单时能选什么，以及选中后生产端自动带出什么。

业务配置项页面分两块：

- 限制规则：维护超宽、超重或特殊组合的禁用/提醒规则。
- 选项树：维护下单可选项、选项值，以及选中某个值后是否继续展开下级选项。

示例结构：

```text
系统
├─ 拉珠
│  └─ 拉珠型号
├─ 电机
│  ├─ 电机型号
│  └─ 遥控型号
└─ 无拉
```

再例如：

```text
包布
├─ 不包
└─ 包
   └─ 面料
```

这里“包布 = 不包”只是一个纯选择，不需要关联物料；“包布 = 包”下面继续展开“面料”时，面料选项值再从原料池里的面料物料导入。

配置项说明：

- 根配置项默认展示，例如系统。
- 子配置项通过父选项值控制展示，例如选择系统 = 电机后才展示电机型号和遥控型号。
- 配置项和值都维护中文名和英文名；中文用于技术科和工厂识别，英文用于美国客户下单展示。
- 选项值可以关联生产物料，例如选择遥控型号后带出对应遥控器或网关。
- 来源物料池的末级选项值必须关联生产物料；布尔、手工或产品字典来源的选项值可以作为纯选择值，例如“包布 = 不包”不需要关联物料。有下级配置项的父级值可以只负责展开后续选择。
- 删除配置项或选项值前，必须先清空限制规则、用量规则、内部变量取值规则和关联物料中的引用。
- 如果后端提示引用存在，不要改编码绕过；应回到对应规则清理引用后再删除。

### 配置项来源

配置项有四种来源：

| 来源 | 适用场景 | 操作方式 |
| --- | --- | --- |
| 来源物料池 | 面料、罩壳、下杆等直接从当前配方原料池选物料 | 先在配方原料选入物料，再在配置项里选择物料分组，使用“导入选项值”批量带入 |
| 产品字典 | 已在产品字典维护的业务枚举 | 在配置项里选择该来源，选项值按当前页面维护 |
| 布尔值 | 是/否、包/不包、静音/不静音 | 直接新增选项值，例如“包”“不包” |
| 手工 | 系统、控制方式、遥控型号等技术科手工维护的选择 | 直接新增选项值，可按需要关联原料池物料 |

新增配置项默认是手工来源、单选。技术科再按实际业务改来源类型、选择模式和名称。

### 选项树维护

- “新增一级”用于新增根配置项，例如系统、面料、包布。
- “新增下级”必须先选中一个值节点，例如选中“系统 = 电机”后新增“电机型号”。
- 配置项和值节点都可以上移、下移，用于维护下单展示顺序。
- 删除前系统会检查下级配置项、默认值和限制规则等明显引用；保存时后端还会检查用量规则、内部变量取值规则和关联物料引用。
- 不要通过修改编码来绕过引用检查。编码只是保存后的业务标识，不是前端编辑态唯一身份。

### 选项值和关联物料

- 来源物料池的配置项通过“导入选项值”从当前配方原料池批量选入。导入后，选项值会自动关联对应生产物料。
- 手工、布尔值和产品字典来源的配置项通过“新增选项值”维护。
- 选项值的中文名给技术科和工厂看，英文名给美国客户下单看。
- “关联物料”只允许选择当前配方原料池里的物料。若物料不在原料池，先回到配方原料选入。
- 一个选项值可以关联多个物料，例如选择某个电机系统后同时带出电机、遥控器和安装配件。
- 纯选择值可以没有关联物料，例如“不包”“不带遥控”；它们只影响后续分支或规则判断。

限制规则放在业务配置项上方，作为全局风险规则维护。限制条件用公式编辑器录入，例如订单宽(in)过大、订单长(in)过大、某个配置项选择了拉珠或电机；触发动作可以作用于整单，也可以作用于某个配置项值。

限制规则只用于禁用或提醒，例如订单宽度过大时禁用拉珠。正常分支展示不要放到限制规则里。

## 配方模拟

配方模拟用于审核前检查。

当前模拟页用于录入一组样例订单：

- 订单宽(in)
- 订单长(in)
- 数量，默认 `1`
- 房间或备注，用于识别这组样例
- 客户选择的配置项值

订单宽(cm)、订单长(cm)和订单面积(m²)由系统在公式计算时自动派生，不需要技术科或下单用户重复录入。

配置项会按业务配置项的显示条件动态展示：根配置项默认展示，子配置项在父选项值命中后展示。可见且必填的配置项必须选择；隐藏配置项不会参与本次模拟。

运行模拟时，系统会：

1. 检查订单宽、订单长和数量。
2. 检查尺寸是否落在配方档案范围内。
3. 检查可见必填配置项是否已选。
4. 检查已选值是否仍是当前配方启用值。
5. 带出生产必带物料和选项值关联物料。
6. 计算内部变量、用量规则、单件金额和数量合计金额。
7. 检查限制规则，例如宽度过大时禁用拉珠。

模拟结果应能看出：

- 哪些物料被带出
- 每个物料的用量结果
- 命中的用量规则摘要
- 单件金额和数量合计金额
- 是否命中限制规则
- 是否存在缺默认规则、缺公式、公式无效等问题

“运行模拟”只返回本次结果；“确认模拟通过”会把结果写入配方的模拟校验状态。提交审核前，配方原料、业务配置项和配方模拟都必须校验通过。

## 配方审核

提交审核前必须先校验通过。配方审核页只展示待审核版本快照，不展示草稿和已生效版本。

审核列表可按以下条件查询：

- 配方编号
- 配方名称
- 版本号
- 提交人
- 校验状态

审核列表展示提交审核时冻结的配方编号、配方名称、版本号、状态、提交人、提交时间和校验状态。选中一行后可以执行：

| 操作 | 用法 |
| --- | --- |
| 详情 | 进入审核详情页，查看待审核版本快照中的配方信息和物料明细 |
| 审核通过 | 确认当前快照可生效，系统将配方状态改为生效并冻结版本 |
| 驳回 | 填写驳回原因，配方回到可编辑状态 |
| 审核记录 | 查看当前配方的提交审核、审核通过、驳回等变更流水 |

审核详情页用于复核本次提交的版本快照。详情页展示配方主信息、物料明细、用量规则数量、选项带出物料数量和审核记录摘要。

审核通过后：

- 配方状态进入生效。
- 系统生成版本号和版本快照。
- 草稿后续修改不影响已审核版本。
- 后续报价、订单和生产快照应引用审核通过的版本快照。

驳回后：

- 配方回到可编辑状态。
- 技术科按驳回原因修改后重新校验和提交。

停用后：

- 配方不再重新启用。
- 继续使用时复制或新建草稿重新审核。
