# 配方管理操作手册

## 录入顺序

1. 在基础信息完成产品分类、产品字典、单位、厂家、物料类型、物料属性和物料管理。
2. 在配方档案新增配方，填写编号、名称、产品分类、产品类型和尺寸范围。
3. 在配方原料选入生产可能用到的物料，维护生产备注和用量规则。
4. 在业务配置项维护客户下单时能选择的配置项、选项值、关联生产物料和异常限制。
5. 在配方模拟输入样例宽高和选项，确认 BOM 与用量结果。
6. 校验通过后提交审核。
7. 审核通过后生成版本快照，供后续报价、订单和生产引用。

## 配方档案

配方档案只维护配方主信息，不维护物料明细。

必填信息：

- 配方编号
- 配方名称
- 产品分类
- 产品类型
- 尺寸范围

产品类型来自产品字典，不在配方里手工造值。

## 配方原料

配方原料是技术科确认生产清单的核心入口。

### 物料来源

- 只能从物料管理选入已维护物料。
- 不在配方原料里新增基础物料。
- 物料类型、属性分组、物料中文名称、物料英文名称、型号、规格、主单位等来自基础信息。

### 原料字段

| 字段 | 用法 |
| --- | --- |
| 属性分组 | 物料所属大类，例如面料、铝材、系统、配件 |
| 物料类型 | 物料小类，例如遮光面料、电机、下杆 |
| 物料编码/中文名称/英文名称/规格 | 用于技术科确认是否选对物料 |
| 单位 | 展示物料主单位 |
| 生产必带 | 命中配置时生产必须包含；这是生产属性，不代表客户可见 |
| 用量汇总 | 展示固定值或公式规则摘要 |
| 生产备注 | 给生产端看的说明 |

### 用量设置

用量规则从原料行的“用量设置”进入。物料池 grid 只做材料确认，不在主表格里展开复杂公式。

当前用量方式：

| 用量方式 | 用法 |
| --- | --- |
| 固定值 | 始终固定数量的物料，只维护一个固定用量 |
| 公式规则 | 按默认规则或表达式条件维护多条公式规则 |

固定值示例：

- 安装码固定 `2.00` 个。
- 包装袋固定 `1.00` 个。

公式规则示例：

- 默认规则：公式为 `1`
- 表达式条件：`订单宽(in) > 120` 时公式为 `2`
- 表达式条件：`系统 = 电机` 时使用电机相关物料公式

当前公式规则不支持把 `if/case` 写进一个公式里。分支优先放到多条规则的“使用条件”中，每个公式只表达一条计算方式。

### 条件和公式

公式编辑器支持中文变量录入，并转换成内部变量保存。下单端录入美国单位 `in`，系统自动派生 `cm` 和 `m²` 变量：

| 中文变量 | 内部变量 | 当前使用说明 |
| --- | --- | --- |
| 订单宽(in) | `orderWidthIn` | 模拟页输入字段 |
| 订单长(in) | `orderLengthIn` | 模拟页输入字段 |
| 订单宽(cm) | `orderWidthCm` | `订单宽(in) * 2.54` |
| 订单长(cm) | `orderLengthCm` | `订单长(in) * 2.54` |
| 订单面积(m²) | `orderAreaM2` | `订单宽(cm) * 订单长(cm) / 10000` |

条件表达式还支持两类业务条件：

- 业务配置项选择值：系统会把选中的配置项值放入 `option_配置项编码` 变量中，例如可表达“系统 = 电机”。
- 原料池条件：来源于当前配方原料池，不从全系统物料随意取值；当前界面支持按属性分组下的物料类型构造条件，例如“面料.物料类型 = 遮光面料”或“系统.物料类型 = 电机”。

原料池条件用于表达“当前配方原料池中某一类物料怎么处理”。按面料、铝材、系统等差异做扣减时，应优先使用当前原料池聚合出的属性分组和物料类型。

### 内部变量池

内部变量用于把一段会反复使用的计算结果先命名，再在用量公式里引用。

典型例子：

- `铝材扣减`：不同系统或面料类型下扣减值不同。
- `走珠数量`：按订单宽(cm)计算并取整。
- `安装码数量`：按订单宽(cm)或安装方式计算。

内部变量跟随当前配方，不做全局共享。后续可以从已有配方复制变量池到当前配方，再按当前产品调整。

公式选择器中的使用方式：

- 左侧编写当前公式。
- 右侧展示当前配方内部变量池，只展示“变量”和“取值摘要”。
- 双击内部变量行可插入公式。
- 底部固定展示订单变量快捷按钮，例如订单宽(in)、订单长(in)、订单宽(cm)、订单长(cm)、订单面积(m²)。

变量采用“变量 + 取值规则”模型：

| 内容 | 用法 |
| --- | --- |
| 变量名称 | 技术科可读名称，例如铝材扣减 |
| 变量编码 | 系统内部编码，保存公式时变为 `var_变量编码` |
| 默认规则 | 没有命中其它条件时使用 |
| 取值条件 | 命中条件时覆盖默认值 |
| 取值方式 | 固定值或公式值 |

变量公式支持订单变量和前面已定义的内部变量，也支持 `四舍五入()`、`向上取整()`、`向下取整()`。保存时系统转换为 `round()`、`ceil()`、`floor()`。

用量公式外层摘要继续展示技术科输入口径，例如 `订单宽(cm) - 铝材扣减`；内部表达式保存为 `orderWidthCm - var_ALU_DEDUCT`。

内部变量的取值条件只用于决定变量取值，不用于决定某个物料是否进入生产清单。物料或用量规则是否生效，必须维护在用量规则的“使用条件”中。

推荐拆法：

- 扣减值、走珠数量、安装码数量这类公共计算参数，放内部变量。
- “系统 = 拉珠 时才使用这条物料”这类生效判断，放用量规则使用条件。
- 不要把同一个业务分支同时写在变量取值条件和用量使用条件里；若确实需要组合，变量未命中取值条件时会走默认规则。

## 业务配置项

业务配置项决定客户下单时能选什么，以及选中后生产端自动带出什么。

示例结构：

```text
系统
├─ 拉珠
│  └─ 拉珠型号
├─ 电机
│  ├─ 电机型号
│  └─ 遥控型号
└─ 无拉
```

配置项说明：

- 根配置项默认展示，例如系统。
- 子配置项通过父选项值控制展示，例如选择系统 = 电机后才展示电机型号和遥控型号。
- 配置项和值都维护中文名和英文名；中文用于技术科和工厂识别，英文用于美国客户下单展示。
- 选项值可以关联生产物料，例如选择遥控型号后带出对应遥控器或网关。
- 最末级选项值必须关联生产物料；有下级配置项的父级值可以只负责展开后续选择。

异常限制放在业务配置项上方，作为全局风险规则维护。限制条件用公式编辑器录入，例如订单宽(in)过大、订单长(in)过大、某个配置项选择了拉珠或电机；触发动作可以作用于整单，也可以作用于某个配置项值。

异常限制只用于禁用或提醒，例如订单宽度过大时禁用拉珠。正常分支展示不要放到异常限制里。

## 配方模拟

配方模拟用于审核前检查。

当前模拟页只需要录入：

- 订单宽(in)
- 订单长(in)
- 客户选择的配置项值

订单宽(cm)、订单长(cm)和订单面积(m²)由系统在公式计算时自动派生，不需要技术科或下单用户重复录入。

模拟结果应能看出：

- 哪些物料被带出
- 每个物料的用量结果
- 是否命中异常限制
- 是否存在缺默认规则、缺公式、公式无效等问题

## 配方审核

提交审核前必须先校验通过。

审核通过后：

- 配方状态进入生效。
- 系统生成版本号和版本快照。
- 草稿后续修改不影响已审核版本。

驳回后：

- 配方回到可编辑状态。
- 技术科按驳回原因修改后重新校验和提交。

停用后：

- 配方不再重新启用。
- 继续使用时复制或新建草稿重新审核。
