# 配方管理体验问题清单

本文只记录当前仍需关注的配方管理体验问题。已修复并确认的问题直接移除。

## 配方原料

- 物料池 grid 应保持“技术科确认材料”的定位，避免把计算单位、规则名称等用量编辑字段前置干扰浏览。
- 生产必带需要在 tooltip 或帮助文案中解释清楚，避免误解为客户可见字段。
- 单位展示应优先展示中文名称；用量设置里不应重复让用户选择当前物料的单位。
- 物料选择弹窗必须支持属性分组、物料类型、物料编码、物料名称等过滤，并默认只选启用物料。
- 用量设置弹窗应尽量减少横向滚动，规则名称等可自动生成的字段不应强迫用户录入。
- 物料池工具栏应尽量保持一行：检索条件靠左，批量操作靠右，减少纵向空间占用。

## 条件与公式

- 条件编辑器需要继续收敛“默认规则、表达式条件、条件选项值”的展示关系，避免用户理解成三套互相割裂的规则体系。
- 表达式条件可以表达 `系统 = 电机` 这类业务配置项条件，也可以表达 `面料.物料类型 = 遮光面料` 这类原料池条件；界面需要明确配置项、属性分组和值的来源，避免用户手写内部编码。
- 当前代码已支持订单宽、订单高、订单长、订单面积、订单重量等变量；实际下单端当前主要输入宽高，重量、面积、订单长的业务含义需要继续确认后再引导用户使用。
- 按“面料类型、一类面料”做扣减时，应优先从当前配方原料池的属性分组和物料类型聚合；按面料系列做扣减时，需要确认面料系列是否进入物料快照或属性值，不能在界面里让用户猜。
- 公式规则必须能提示缺默认规则、重复条件、未知变量和非法表达式。

## 业务配置项

- 配置项树要清楚表达父子关系，长名称不能撑爆布局。
- 选项值关联生产物料应从原料池选择，不让用户手写系统编码。
- 异常限制只处理禁用或提醒，不承担正常分支显示逻辑。
- 正常分支应通过显示条件表达，例如选择系统 = 电机后才展示电机型号、遥控型号。

## 国际化与展示

- 页面不允许出现 `productCenter.xxx` 原始 i18n key。
- 中文环境不应中英同屏展示统计标题；英文环境按英文文案展示。
- 日期时间展示到分钟即可。
- 数量和用量默认展示 2 位小数。

## 校验与提交流程

- 提交审核前必须完成配方档案、配方原料、业务配置项和模拟相关校验。
- 校验失败原因必须能在页面上稳定看到，不能只弹一次提示。
- 审核通过生成的版本快照应可回看主档、原料、选项、限制和用量规则。
