# 配方管理体验问题清单

本文只记录当前仍需回归确认的体验问题。已固化为业务规则、接口契约或已确认修复的问题，直接移出本清单。

## 配方原料

- 需要用真实物料数据回归物料选择弹窗：属性分组、物料类型、物料编码、物料名称过滤是否够顺手，是否默认只选启用物料。
- 需要用真实物料数据回归配方原料顶部统计：属性分组、总数、未设、规则在常见屏幕宽度下是否一行展示，短文案是否容易理解。
- 需要用真实用量规则回归用量设置弹窗：横向滚动、规则摘要、公式输入和“用量结果”通过/失败提示是否影响技术科连续录入。
- 需要回归物料池行内损耗率和生产备注：确认损耗率仅展示和供后续 ERP 参考，不影响模拟用量，也不重复出现在用量设置里。

## 条件与公式

- 需要用真实“面料 + 副面料 + 罩壳”数据回归选项物料属性表达式，例如 `面料.厚度 + 副面料.厚度 > 12`、`罩壳.重量 > 2.5`。
- 需要回归配置项或选项值被限制规则、用量规则、内部变量取值规则、关联物料引用时，删除失败原因是否能明确指向引用位置。
- 需要回归条件编辑器的弹窗宽度和左右布局：物料属性、业务变量、订单变量的输入控件不能被挤压，也不能出现大面积无意义空白。
- 需要回归公式选择器在有内部变量和无内部变量两种场景下的布局：订单变量区、校验结果区和底部按钮必须一屏可见，右侧变量池只做查看和插入。

## 业务配置项

- 需要用真实多级选项树回归长名称、上移下移、新增下级和删除引用保护是否顺手。
- 需要用真实下单分支回归“系统 = 电机后展示电机型号、遥控型号”这类正常分支配置是否清楚。
- 需要用真实图文物料、普通下拉和 Help 内容回归配置项第二行设置：必输项、客户可见、展示方式和 Help 设置弹窗是否顺手。

## 配方模拟

- 需要用真实配方回归模拟页的配置项动态展示：父选项切换后，子配置项是否正确显示、默认值是否正确带出、隐藏项残留选择是否被清空或至少不参与 BOM、限制规则和物料属性上下文计算。
- 需要回归订单宽(in)、订单高(in)、数量和尺寸范围校验，确认提示能明确指出缺失或超范围原因。
- 需要回归生产必带物料、选项值关联物料和用量规则是否共同生成正确 BOM；金额固定两位，用量按单位精度保底两位并保留有效尾数。
- 需要回归限制规则命中后的展示，例如宽度过大时禁用拉珠，确认页面能稳定显示限制原因，而不是只弹一次 toast。
- 需要回归“运行模拟”和“确认模拟通过”的差异：运行只预览结果，确认通过才写入配方模拟校验状态。

## 国际化与展示

- 需要中英文环境各回归一次，确认页面不出现原始 i18n key，数量、用量和日期时间展示符合规则。

## 校验与提交流程

- 需要完整跑一次“配方原料 -> 业务配置项 -> 模拟 -> 校验 -> 提交审核 -> 审核通过”的回归，确认失败原因可稳定查看，版本快照可回看。
