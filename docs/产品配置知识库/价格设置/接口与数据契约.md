# 价格设置接口与数据契约

## 模块定位

价格设置维护“可售产品 + 生效配方版本”的完整物料销售定价。价格物料只能从配方版本快照同步，不允许脱离配方手工新增。

- 面料：通常按订单面积计价，例如 `50 * MAX(areaM2, 1)`。
- 铝材、系统、配件、零件包、包装、印刷品及后续新增物料分组：通常按配方试算用量计价，例如 `3.5 * usageQty`。
- 每个物料可维护一条默认规则和多条尺寸、面积或选项条件规则。
- 邮费独立维护，不进入价格设置；客户报价和销售订单按价格币种读取当前启用邮费模板，并冻结实际邮费快照。

## 页面与接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/product-pricing/price-settings/{saleProductId}` | 获取价格工作台聚合数据 |
| `PUT` | `/product-pricing/price-settings/{saleProductId}/generate-material-prices/{overwrite}` | 同步或覆盖重建当前配方版本的全部物料价格行 |
| `PUT` | `/product-pricing/price-settings/{saleProductId}/materials/{priceMaterialId}/rules` | 保存单个物料价格规则 |
| `PUT` | `/product-pricing/price-settings/{saleProductId}/materials/rules/batch` | 原子化批量保存同一物料分组的规则 |
| `PUT` | `/product-pricing/price-settings/{saleProductId}/validate` | 校验价格并回写就绪状态 |
| `POST` | `/product-pricing/price-settings/{saleProductId}/quote` | 按尺寸和选项试算物料价格，不包含邮费 |

页面路由：`/product-formula/price-settings?saleProductId={id}`。

## 聚合返回

| 字段 | 说明 |
| --- | --- |
| `saleProduct/setting` | 可售产品和当前版本价格头 |
| `priceMaterials` | 当前配方版本快照中的全部物料价格行 |
| `materialRules` | 按 `priceMaterialId` 归属的默认和条件规则 |
| `formulaMaterials/formulaOptions/formulaOptionValues/formulaOptionMaterials` | 条件编辑器和试算所需配方快照 |
| `materialGroupCounts` | 物料分组统计 |
| `issues` | 当前价格校验问题 |

快照读取会忽略创建人、更新时间、删除标志等审计字段，这些字段不参与同步、条件和计价。
工作台统计和价格表按快照中的实际物料分组动态展示，不把可定价范围写死为现有六类。

## 表结构

### `pc_price_setting`

- `price_setting_id`
- `sale_product_id/code/name`
- `formula_id/formula_version_id/formula_version_label`
- `currency_code`
- `validation_status/message/time`
- `status`：`DRAFT`、`READY`

同一可售产品的不同配方版本使用不同价格头，V2 不覆盖 V1。

### `pc_price_material`

- `price_material_id`
- `price_setting_id/sale_product_id/formula_version_id`
- `formula_material_id/material_id`
- `material_code/material_name_cn/spec_model_text/unit_code`
- `attribute_group_code/name`、`material_type_code/name`
- `sort_order`

物料身份优先使用不可变 `material_id`，编码和名称仅为版本展示快照。

### `pc_price_material_rule`

- `material_rule_id/price_material_id`
- `price_setting_id/sale_product_id/formula_version_id`
- `condition_type`：`DEFAULT`、`EXPRESSION`
- `condition_json`：结构化条件快照
- `condition_expression/text/key`
- `unit_price/price_formula`
- `default_rule_flag/sort_order/remark`

条件引用配方选项时，内部表达式和 `condition_json` 使用不可变 `optionRefKey/valueRefKey`；编码、名称只作为展示快照。公式支持 `unitPrice`、`usageQty`、`width`、`drop`、`widthCm`、`dropCm`、`areaM2`、`areaSqft` 及 `MAX/MIN/ROUND/CEIL/FLOOR`。

`unit_price` 使用 `numeric(18,4)`，页面允许录入并展示到 4 位。价格公式使用完整 `unitPrice` 和 `usageQty`，禁止在进入公式前压成两位；正式金额只在币种边界舍入。

## 同步与版本继承

- 新增可售产品时自动创建当前配方版本的 `DRAFT` 价格头；进入价格工作台时也会兜底创建缺失价格头，但物料价格行仍需执行同步后生成。
- 普通同步：新增快照缺失物料，删除已离开当前快照的物料，清理重复行，保留仍匹配规则。
- 覆盖重建：清除当前版本物料和规则后重建。
- 新版本首次同步：按 `material_id`，无法取得时按物料编码，从同一可售产品上一价格版本复制该物料的全部默认和条件规则。
- 继承条件规则时会按新版本重新解析稳定选项引用；新版本已删除的选项或选项值对应规则不会复制。
- 新增物料生成默认规则；面料默认 `unitPrice * MAX(areaM2, 1)`，其他物料默认 `unitPrice * usageQty`。
- 同步和批量保存按可售产品加锁并在单事务内完成。
- 查询建档、同步、单条保存、批量保存、校验和试算共享同一个价格锁名，并以 `saleProductId` 作为业务锁键，避免跨方法并发覆盖。

## 校验和试算

- 当前快照每个物料都必须存在且只能有一条默认规则。
- 同一物料条件键不能重复，条件和价格公式必须可解析。
- 公式引用 `unitPrice` 时单价必须大于 0；显式数字公式不强制单价字段。
- 条件引用必须能在当前版本快照中找到稳定选项和值。
- 试算先执行配方选项、限制、变量和用量规则，得到实际 BOM 与 `usageQty`，再逐物料匹配条件价格；未命中条件时使用默认规则。
- 只有价格校验通过后才能试算和启用可售产品。

## 可检索性与引用关系

- 稳定业务入口是可售产品编号、可售产品名称和配方版本；价格工作台本身不单独维护用户可编辑业务编码。
- 价格设置引用可售产品、生效配方版本、版本快照物料和稳定配置项引用；物料编码、名称及配置项中文名称只作为快照展示。
- 删除可售产品前检查其全部版本的物料价格行和价格规则；历史配方版本价格不能被新版本覆盖。
- 查询和详情必须按租户、删除标记和 `product:pricing:query` 权限过滤；页面入口为 `/product-formula/price-settings?saleProductId={id}`。
- 邮费模板与价格设置没有表级归属关系，当前价格试算接口不包含邮费。

## 权限

- `product:pricing:query`：查询工作台和试算。
- `product:pricing:edit`：同步物料、单个和批量保存规则。
- `product:pricing:validate`：执行价格校验。
