# 价格设置接口与数据契约

## 模块定位

价格设置维护某个可售产品绑定配方版本的销售定价规则。第一版只覆盖产品销售价格：

- 面料价格：系统从可售产品绑定的配方版本快照中同步生成面料清单；每个面料下维护默认价格规则和条件价格规则，主表展示价格汇总。

其它普通物料的销售价来源于物料基础信息，不在价格设置里重复维护。

## 页面路由

| 页面 | 路由 | 说明 |
| --- | --- | --- |
| 价格设置 | `/product-pricing/price-settings?saleProductId={id}` | 独立工作台，按可售产品加载价格规则 |

## 接口清单

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/product-pricing/price-settings/{saleProductId}` | 获取价格设置工作台聚合数据 |
| `PUT` | `/product-pricing/price-settings/{saleProductId}/generate-fabric-prices/{overwrite}` | 按当前绑定配方版本同步面料价格行；`overwrite=true` 时覆盖同版本现有价格并尝试复制上一版本可匹配价格 |
| `PUT` | `/product-pricing/price-settings/{saleProductId}/fabrics/{priceFabricId}/rules` | 保存单个面料的默认/条件价格规则 |
| `PUT` | `/product-pricing/price-settings/{saleProductId}/validate` | 校验价格设置并回写可售产品价格状态 |

## 工作台返回契约

`GET /product-pricing/price-settings/{saleProductId}` 返回当前可售产品、价格头表、面料清单、规则行、校验问题和配方快照数据。

| 字段 | 说明 |
| --- | --- |
| `saleProduct` | 当前可售产品，绑定一个生效配方版本 |
| `setting` | 当前价格设置头表 |
| `priceFabrics` | 当前可售产品绑定配方版本快照中的面料清单，只能由配方版本快照同步生成 |
| `fabricRules` | 已维护的面料价格规则，按 `priceFabricId` 归属到某个面料 |
| `formulaMaterials/formulaOptions/formulaOptionValues/formulaOptionMaterials` | 当前配方版本快照数据，供价格规则条件选择器使用 |
| `issues` | 当前校验问题 |
| `materialGroupCounts` | 当前配方版本快照中的原料分组统计 |

价格设置只读取配方版本快照中的业务字段。历史快照如果残留 `createById`、`createBy`、`createTime`、`updateBy`、`updateTime`、`delFlag` 等审计字段，后端读取时忽略，不展示给用户，也不参与条件判断、价格计算或同步配方面料。

### `pc_price_setting`

| 字段 | 说明 |
| --- | --- |
| `price_setting_id` | 主键 |
| `sale_product_id/code/name` | 可售产品快照 |
| `formula_id/formula_version_id/formula_version_label` | 可售产品绑定的生效配方版本 |
| `currency_code` | 币种，默认 `USD` |
| `validation_status/message/time` | 最近价格校验状态、消息和时间 |
| `status` | 当前价格设置状态：`DRAFT`、`READY` |

### `pc_price_fabric`

| 字段 | 说明 |
| --- | --- |
| `price_fabric_id` | 主键 |
| `price_setting_id/sale_product_id/formula_version_id` | 归属关系 |
| `material_id/material_code/material_name_cn` | 面料物料 |
| `unit_code` | 单位 |
| `sort_order` | 同步自配方快照的展示顺序 |

### `pc_price_fabric_rule`

| 字段 | 说明 |
| --- | --- |
| `fabric_rule_id` | 主键 |
| `price_fabric_id` | 所属面料价格清单行 |
| `price_setting_id/sale_product_id/formula_version_id` | 冗余归属关系，方便查询和校验 |
| `condition_type` | `DEFAULT` 默认规则，`EXPRESSION` 条件规则 |
| `condition_json` | 结构化条件快照；条件编辑器生成，内部优先使用 `optionRefKey/valueRefKey` |
| `condition_expression/condition_text/condition_key` | 条件表达式、展示文本和标准化键，例如 `width > 20 && drop > 30` 或 `option_OPT_SYSTEM == "VAL_MOTOR"`；页面展示为“系统 = 电机” |
| `price_mode` | 固定为 `FORMULA` |
| `unit_price` | 条件命中时使用的面料单价，两位小数展示 |
| `price_formula` | 价格公式，页面按实际单价展示和保存，例如 `50 * MAX(drop * 2.54 * width * 2.54 / 10000, 1)`；支持 `width`、`drop`、`areaM2`、`areaSqft` 和常用函数 |
| `default_rule_flag` | 是否默认价格规则；每个面料必须且只能有一条默认规则 |

## 业务规则

- 价格设置必须绑定可售产品。
- 可售产品绑定的配方版本必须是生效版本。
- 价格设置按 `sale_product_id + formula_version_id` 隔离；V1 价格不会被 V2 覆盖。
- 面料价格只能来自当前可售产品绑定的配方版本快照原料池，不从全局物料资料直接选择。
- 价格条件选择配方选项时必须使用配方快照里的 `optionRefKey/valueRefKey` 作为长期身份；编码和名称只做展示快照，避免改名导致历史价格规则失效。
- 点击“同步配方面料”时，系统按当前配方版本快照重建当前价格设置的面料清单；旧的无归属规则、重复面料行和不在当前快照里的脏行会清理掉，避免重复生成脏数据。
- 绑定新配方版本后，可生成新的版本价格；同面料的默认/条件价格规则可从上一版本复制，新增面料保持待设置。
- 可售产品启用后，价格规则冻结，不允许继续编辑。
- 配方快照中存在面料时，每个面料必须有默认价格规则。
- 条件价格用于表达不同尺寸、面积或选项条件下的不同单价；没有命中条件时使用默认规则。
- 批量设置价格必须先在面料价格主表勾选面料，再进入批量价格规则抽屉；保存时按所选面料分别写入规则，规则归属仍以 `price_fabric_id` 为准。
- 面料价格公式不能为空且必须通过表达式校验；只有公式仍引用单价变量时，单价字段才必须大于 0。
- 邮费不属于产品价格设置，也不影响产品价格就绪校验。下单试算读取当前启用邮费模板，订单提交时再冻结实际命中的邮费快照。
- 校验无错误时，可售产品 `price_status` 回写为 `READY`；存在问题时回写为 `WARNING`。
