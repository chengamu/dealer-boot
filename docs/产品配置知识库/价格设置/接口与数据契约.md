# 价格设置接口与数据契约

## 模块定位

价格设置维护某个可售产品绑定配方版本的销售定价规则。第一版只覆盖两个价格来源：

- 面料价格：系统从可售产品绑定的配方版本快照中生成面料价格矩阵，行是面料，列是订单选项组合，例如“柔式拉珠”“柔式电动”。
- 邮费公式：系统固定维护“不带电”和“带电”两条邮费公式，是否带电由订单选项中是否选择电机类系统判断。

其它普通物料的销售价来源于物料基础信息，不在价格设置里重复维护。

## 页面路由

| 页面 | 路由 | 说明 |
| --- | --- | --- |
| 价格设置 | `/product-pricing/price-settings?saleProductId={id}` | 独立工作台，按可售产品加载价格规则 |

## 接口清单

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/product-pricing/price-settings/{saleProductId}` | 获取价格设置工作台聚合数据 |
| `PUT` | `/product-pricing/price-settings/{saleProductId}/generate-fabric-prices/{overwrite}` | 按当前绑定配方版本生成面料价格矩阵；`overwrite=true` 时覆盖同版本现有价格并尝试复制上一版本可匹配价格 |
| `PUT` | `/product-pricing/price-settings/{saleProductId}/fabric-rules` | 保存面料价格规则 |
| `PUT` | `/product-pricing/price-settings/{saleProductId}/fee-rules` | 保存不带电/带电邮费公式 |
| `PUT` | `/product-pricing/price-settings/{saleProductId}/validate` | 校验价格设置并回写可售产品价格状态 |

## 工作台返回契约

`GET /product-pricing/price-settings/{saleProductId}` 返回当前可售产品、价格头表、规则行、校验问题和候选面料。

| 字段 | 说明 |
| --- | --- |
| `saleProduct` | 当前可售产品，绑定一个生效配方版本 |
| `setting` | 当前价格设置头表 |
| `fabricCandidates` | 当前可售产品绑定配方版本快照中的面料原料池，只包含 `attributeGroupCode = FABRIC` 的原料，并带出审核快照时的 `salesPrice/unitPrice` |
| `fabricPriceColumns` | 根据配方版本快照中的订单选项推导出的价格矩阵列，例如“柔式拉珠” |
| `fabricRules` | 已维护的面料价格矩阵行 |
| `feeRules` | 不带电/带电邮费公式；为空时后端会返回两条默认展示行 |
| `issues` | 当前校验问题 |
| `materialGroupCounts` | 当前配方版本快照中的原料分组统计 |

### `pc_price_setting`

| 字段 | 说明 |
| --- | --- |
| `price_setting_id` | 主键 |
| `sale_product_id/code/name` | 可售产品快照 |
| `formula_id/formula_version_id/formula_version_label` | 可售产品绑定的生效配方版本 |
| `currency_code` | 币种，默认 `USD` |
| `validation_status/message/time` | 最近价格校验状态、消息和时间 |
| `status` | 当前价格设置状态：`DRAFT`、`READY` |

### `pc_price_fabric_rule`

| 字段 | 说明 |
| --- | --- |
| `fabric_rule_id` | 主键 |
| `price_setting_id/sale_product_id/formula_version_id` | 归属关系 |
| `material_id/material_code/material_name_cn` | 面料物料 |
| `unit_code` | 单位 |
| `option_combination_key/name` | 选项组合键和展示名，例如 `STYLE=ROMAN;CONTROL=MOTORIZED` / `柔式电动` |
| `price_mode` | 固定为 `FORMULA` |
| `base_price` | 当前面料在该选项组合下的单价，两位小数展示 |
| `area_formula` | 价格公式，支持 `unitPrice`、`width`、`drop` 和常用函数 |
| `min_bill_area` | 保留字段，第一版 UI 不展示 |
| `loss_rate` | 保留字段，第一版 UI 不展示 |
| `status` | 保留字段，第一版固定 `ENABLED`，UI 不展示启停 |

### `pc_price_fee_rule`

| 字段 | 说明 |
| --- | --- |
| `fee_rule_id` | 主键 |
| `fee_code/fee_name` | `MANUAL` 不带电邮费、`MOTORIZED` 带电邮费 |
| `fee_category` | 固定为 `SHIPPING`，UI 不展示 |
| `trigger_condition` | 固定为 `motorized = false/true`，UI 不展示自由录入 |
| `fee_mode` | 固定为 `FORMULA` |
| `fee_amount` | 保留字段，第一版 UI 不展示 |
| `formula_text` | 邮费公式 |
| `status` | 保留字段，第一版固定 `ENABLED`，UI 不展示启停 |

## 业务规则

- 价格设置必须绑定可售产品。
- 可售产品绑定的配方版本必须是生效版本。
- 价格设置按 `sale_product_id + formula_version_id` 隔离；V1 价格不会被 V2 覆盖。
- 面料价格只能来自当前可售产品绑定的配方版本快照原料池，不从全局物料资料直接选择。
- 点击“生成当前版本价格”时，系统按当前配方版本快照生成面料价格矩阵。
- 绑定新配方版本后，可生成新的版本价格；同面料、同选项组合的价格可从上一版本复制，新增组合保持待设置。
- 可售产品启用后，价格规则冻结，不允许继续编辑。
- 配方快照中存在面料时，每个面料和选项组合都必须有价格行。
- 面料单价必须大于 0，面料价格公式不能为空。
- 必须维护不带电和带电两条邮费公式。
- 校验无错误时，可售产品 `price_status` 回写为 `READY`；存在问题时回写为 `WARNING`。

## 权限

| 权限 | 说明 |
| --- | --- |
| `product:pricing:query` | 查询价格设置工作台 |
| `product:pricing:edit` | 生成和保存面料价格、保存邮费公式 |
| `product:pricing:validate` | 执行价格校验 |
