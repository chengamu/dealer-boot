# AI 代办能力规范

本文定义后续在基础信息、产品工程设计、产品配置中心中接入 AI 聊天框时，功能文档需要提供的结构化信息。

AI 代办知识库采用双层结构，不为每个功能新增独立 AI 文档：

- `操作手册.md` 写业务化说明：用户说法、追问场景、保存前确认摘要。
- `接口与数据契约.md` 写工具化契约：调用接口、权限、槽位、payload 边界、禁止暴露字段、失败提示。

AI 代办不是绕过页面直接写库，而是把自然语言转换为可确认、可校验、可审计的业务操作。

## 知识消费分层

`docs/产品配置知识库` 是 AI 代办的唯一业务知识源，但不同能力消费不同粒度的知识：

- 全局助手需要一个短小的功能导航索引，用来从任意页面识别用户意图、定位菜单、判断目标路由和进入正确页面。
- Page Agent 页面操作只消费当前路由的精简页面指令包。
- 页面指令包从 `操作手册.md` 和 `接口与数据契约.md` 提炼，不直接把完整接口契约、数据库字段或原始 payload 喂给前端 Agent。
- 客户知识问答、跨页面资料检索和大文档问答后续由独立 AI Gateway / 知识检索服务承载，不把全量知识库打进前端 bundle。
- Java 业务后端负责权限、业务校验、保存结果和审计；Page Agent 负责当前页面 DOM 操作和确认前引导。

全局导航索引只包含：

```text
功能名称
菜单路径
前端路由
3 到 5 个关键词
一句消歧规则
```

全局导航索引不能放必填槽位、字段规则、接口规则、保存流程或失败提示；这些内容必须放在对应页面指令包里。后续模块增多后，完整功能地图和权限过滤应迁移到 AI Gateway 的能力查询工具。

页面指令包至少包含：

```text
页面用途
支持意图
必填槽位
可推断字段
高风险动作
禁止暴露字段
失败提示口径
```

页面指令包必须保持短小，只服务当前页面操作。接口路径、内部 ID、数据库字段、租户字段、审计字段和完整请求示例仍保留在 `接口与数据契约.md`，不面向业务用户输出。

页面指令包必须支持当前前端语言。当前前端支持 `zh_CN` 和 `en_US`，新增页面指令时必须同时提供中文和英文版本；Page Agent 的工作语言应跟随当前界面语言。

## 基本原则

- AI 只能调用已有后端接口，不直接写数据库。
- AI 不能让用户填写内部 ID、JSON、技术字段。
- AI 必须先补齐必填信息，再提交。
- AI 对高风险操作必须让用户确认。
- AI 返回给用户的是业务摘要，不是接口 payload。
- AI 失败时必须说明业务原因和下一步动作。

## 意图定义

每个支持 AI 代办的功能，需要在对应文档里定义意图。

```text
意图名称：
用户说法：
适用页面：
调用接口：
权限要求：
```

示例：

```text
意图名称：新增物料
用户说法：帮我新增一个电机物料
适用页面：基础信息 / 物料管理
调用接口：POST /product-capability/materials
权限要求：product:base:add
```

## 槽位定义

槽位分为四类：

- 必填槽位：缺失时必须追问。
- 可选槽位：用户提供就保存，未提供用默认值或留空。
- 可推断槽位：可以根据上下文、字典或默认规则推断。
- 禁止槽位：用户不应该直接填写的内部字段。

示例：

```text
必填槽位：
- materialCode
- materialNameCn
- materialType
- spec
- unitCode

可推断槽位：
- status = DISABLED
- materialType snapshot fields from pc_material_type
- attributeGroup snapshot fields from pc_material_type_group

禁止槽位：
- id
- createBy
- createTime
- updateBy
- updateTime
```

## 保存前确认

新增、批量新增、删除、停用、发布、覆盖类操作必须保存前确认。

确认内容必须是业务摘要：

```text
将新增物料：
- 编码：MOTOR_25
- 名称：25 管电机
- 类型：电机系统
- 工程单位：PCS
是否确认保存？
```

## 错误处理

错误提示要从技术错误翻译成业务动作。

```text
错误：materialCode already exists
提示：物料编码已存在，请换一个编码，或者打开现有物料继续编辑。
```

每个功能的接口契约必须列出失败时的可读提示；不能只列后端错误 key。

## 禁止事项

- 不允许 AI 猜测价格、采购、库存、生产工艺结果。
- 不允许 AI 自动发布工程方案。
- 不允许 AI 自动删除核心主数据。
- 不允许 AI 直接返回数据库字段让用户理解。
- 不允许 AI 以“我已经保存”为结论，除非接口真实返回成功。
