# i18n Playbook

仅在 i18n 规划、实现或验证时按需加载。

## 规则

- 遵守 `AGENTS.md` 和 `.ai/CONTEXT.md` 中的 i18n 规则。
- 新增系统文案优先进入项目约定的单源 locale 文件，再由脚本同步到前后端运行资源。
- 前后端运行时资源通常是同步产物，不要手动分叉维护多套翻译。
- 系统字典翻译应有稳定 key 规则；不要在页面里散落拼接或硬编码翻译。
- 国家、币种、语言等标准数据优先按标准资料建模；客户业务数据原样存储，不生成 i18n key。
- AI 翻译只允许在开发/构建期运行，运行时不得依赖 AI。

## 检查项

- i18n 校验应检查 key 完整性、空值、占位符一致性和必要的字典覆盖。
- 迁移旧文案时先归入分类：系统文案、系统字典、标准数据、业务数据。
- 不确定是否涉及 SQL、字典结构或旧 i18n 源清理时，先回到 `/plan`。
