# UI Execution Contract: 共享产品能力中心上线体验收口

创建日期：2026-06-07

本文是本 Wave 的 UI 执行合同。所有前端实现、代码审计和浏览器验收必须先读本文，再读 `docs/产品配置中心/产品能力界面设计稿.md`。

目的：防止 AMU Wave Scheduler 把连续设计拆碎，导致只完成局部组件、丢失主信息架构、快捷操作或体验策略。

## 1. 单一真相

本 Wave 的 UI 主合同按优先级读取：

1. `docs/产品配置中心/共享产品能力中心开发入口.md`
2. `docs/产品配置中心/产品能力界面设计稿.md`
3. 本文件 `ui-execution-contract.md`
4. `.ai/changes/20260607-product-capability-usability-wave/check.md`

冲突时按上面顺序处理；如果本文件和设计文档冲突，不允许自行猜测，必须记录 `blocked` 或回到 `/plan` 修订。

## 2. 执行方法不可拆散

5 个自定义页面必须按同一条链路执行，不能拆成互不相干的局部任务：

```text
读取设计文档
  -> 锁定现有 admin-ui 风格
  -> 拆 H5 / PNG 为 Vue 页面结构、代码渲染区域、资产计划和验收点
  -> Vue / Element Plus / SCSS 组件化实现
  -> design-taste 审查是否变成普通 CRUD 或失去录入效率
  -> 内部浏览器真实地址验收
```

技能边界：

| 步骤 | 工具 / Skill | 产出 | 禁止 |
| --- | --- | --- | --- |
| 风格锁定 | `visual-replica` | 当前后台颜色、间距、卡片、表格、抽屉、按钮风格真值 | 重做 Layout、Sidebar、Navbar、TagsView |
| 结构拆解 | `image2-ui-skill` | Vue 区块、代码渲染区、资产区、截图验收清单 | 把 H5 / PNG 当生产代码或截图贴图 |
| 实现 | `frontend-design` + 项目现有模式 | Vue / Element Plus / SCSS 页面和组件 | 独立应用、Tailwind CDN、外链图片、硬编码文案 |
| 审查 | `design-taste-frontend` | 视觉和交互问题清单 | 只看好不好看，不看录入效率 |
| 验收 | 内部浏览器 | 真实菜单、真实登录态、真实库、console/network、移动端证据 | 用编译通过替代体验验收 |

## 3. 不可拆散的连续体验

这些是跨页面连续体验，不能被任务拆分破坏：

| 连续体验 | 起点 | 终点 | 不可丢失要求 |
| --- | --- | --- | --- |
| 缺口处理 | 工作台待处理流 / 优先队列 | 缺口待办 / 目标编辑页 | 点击后必须带入筛选或定位目标，不让用户重新搜索 |
| 配置录入 | 配置模板三栏编辑 | Live checks / Dealer preview / BOM preview | 边录边看检查和预览，不能拆成多个孤立页面 |
| 价格试算 | 价格中心矩阵编辑 | 右侧试算器 / 报价预览 | 改价格后可立即试算，试算调用后端价格引擎 |
| 发布闸门 | 测试发布检查 | 缺口任务 / 发布包 / 同步状态 | BLOCKER 不能发布，WARNING 二次确认，发布包信息发布前可见 |
| 销售消费 | 销售只读总览 | Quick Quote / 发布包 / 资料动作 | 页面只读，销售可见字段由后端控制，Quick Quote 能进入报价预览 |

如果某个任务只完成单页样式，却破坏这些跨页路径，不能标记 `done`。

## 4. 5 个自定义页执行矩阵

### 4.1 工作台

主结构：

- 页面顶部：标题、说明、数据更新时间、刷新。
- 今日待处理流：发布阻断、资料缺失、价格缺口、待审核快照、同步异常。
- 产品配置进度看板：产品、系列、版本、完整度、缺失项、价格状态、资料状态、发布状态、负责人、更新时间、下一步动作。
- 右侧效率面板：优先处理队列、一键快捷操作、最近同步事件。
- 底部分析区：资料缺失 TOP、价格缺口 TOP、发布/同步动态。

必须组件化：

- `TaskFlowCard` 或等价组件。
- `ProductProgressTable` 或等价组件。
- `PriorityQueuePanel` 或等价组件。
- `QuickActionGrid` 或等价组件。
- `RecentSyncEvents` 或等价组件。

快捷操作：

- 待处理卡片点击后筛选进度看板。
- 优先队列点击后直达目标对象或缺口。
- 行内下一步动作优先展示继续配置、补资料、匹配价格、发布检查。
- 快捷按钮必须按权限过滤。

Done 禁止：

- 只做 KPI 卡片，没有处理入口。
- 同步异常只展示数字，不能进入同步日志。
- 工作台和缺口/发布/资料之间不能跳转。

### 4.2 配置模板

主结构：

- 左侧：产品模型、销售变体、模板版本、问题组树、完成状态。
- 中间：问题 / 答案 / 规则录入，保留表格密度。
- 右侧：Live checks、Dealer preview、BOM preview、AI suggestions。
- 顶部：保存草稿、复制问题组、批量粘贴、预览销售视图、发布入口。

必须组件化：

- `QuestionGroupNavigator` 或等价组件。
- `QuestionOptionEditor` 或等价组件。
- `LiveCheckPanel` 或等价组件。
- `DealerPreviewPanel` 或等价组件。
- `BomPreviewPanel` 或等价组件。
- `BatchPasteDrawer`、`ComponentBindingDrawer`、`MediaBindingDrawer` 或等价抽屉。

快捷操作：

- 保存草稿不离开当前问题组。
- 批量粘贴先解析预览，再确认写入。
- 绑定组件 / 资料使用右侧抽屉。
- 问题行支持复制、删除、启停、排序或明确禁用态。
- BLOCKER 点击定位到对应问题或答案。

Done 禁止：

- 把问题、答案、规则拆成互不联动的普通 CRUD。
- 右侧检查、预览、BOM 消失或只做静态占位。
- 批量录入只留下 TODO 或假按钮。

### 4.3 价格中心

主结构：

- 顶部方案信息：方案名称、编码、产品系列、币种、定价模式、生效状态、生效时间。
- 主编辑区：矩阵价格、价格规则、加价项、折扣策略。
- 矩阵编辑器：宽度/高度维度、单元格价格、公式或无效组合。
- 右侧试算面板：配置条件、重新计算、价格结果、命中规则、生成报价预览。
- 底部规则说明：当前单元格公式、适用范围、引用物料、固定加价。

必须组件化：

- `PricePlanHeaderForm` 或等价组件。
- `PriceMatrixEditor` 或等价组件。
- `PriceTesterPanel` 或等价组件。
- `PriceHitDetails` 或等价组件。
- `FormulaRuleSummary` 或等价组件。

快捷操作：

- 矩阵支持横向滚动、单元格选中、批量填充或明确预留且禁用态说明。
- 改价后右侧试算器可一键重新计算。
- 试算结果展示基础价、系数、组件加价、折扣、最终价。
- 试算结果可以进入报价预览。
- 发布价格前能查看影响产品或明确禁用态。

Done 禁止：

- 前端自行复制最终价格计算规则。
- 只有普通价格列表，没有矩阵 + 试算同屏反馈。
- 金额、币种、小数位硬编码。

### 4.4 测试发布

主结构：

- 顶部：重新检查、检查时间、发布动作。
- 检查概览：PASS、WARNING、BLOCKER、PENDING。
- 检查项总览：配置完整性、价格命中、媒体完整性、组件绑定、BOM、快照、审批、同步。
- 阻断任务：优先级、任务、模块、描述、创建人、创建时间、修复入口。
- 右侧发布包预览：产品图、编码、产品线、地区、版本、hash、生效时间、包内容。
- 同步状态：事件总线、最后同步时间、同步延迟、失败事件数。

必须组件化：

- `PublishStatusCards` 或等价组件。
- `CheckItemTable` 或等价组件。
- `BlockerTaskTable` 或等价组件。
- `PublishPackagePreview` 或等价组件。
- `SyncStatusCard` 或等价组件。

快捷操作：

- 一键重跑检查。
- BLOCKER 可转缺口任务或定位修复入口。
- 检查项可查看详情、下载 BOM、预览发布内容、查看审批。
- BLOCKER 存在时发布按钮禁用。
- WARNING 存在时发布前二次确认。
- 发布成功后展示 outbox 写入和同步状态。

Done 禁止：

- PASS / WARNING / BLOCKER / PENDING 颜色语义弱化。
- 发布包 hash、版本、生效时间发布前不可见。
- 发布按钮是纯前端假按钮。

### 4.5 销售只读总览

主结构：

- 顶部发布摘要：主图、产品名、版本、编码、类别、系列、生效时间、最后同步。
- Quick Quote：基于当前已发布配置进入报价。
- 快捷资料：发布包详情、对外文档、规格书、复制链接。
- 能力摘要：配置完整性、价格状态、客户资料、BOM 摘要。
- 销售限制：区域、客户类型、渠道、MOQ、交期、特殊限制。
- 版本历史：版本、发布时间、内容、发布人、详情。

必须组件化：

- `PublishedProductHeader` 或等价组件。
- `QuickQuotePanel` 或等价组件。
- `ReadonlyAbilityCards` 或等价组件。
- `SalesRestrictionBlock` 或等价组件。
- `VersionHistoryTimeline` 或等价组件。

快捷操作：

- Quick Quote 进入报价预览。
- 查看配置详情、价格详情、全部媒体、规格书。
- 复制产品链接或在无权限/无数据时显示明确禁用态。

Done 禁止：

- 页面出现新增、编辑、删除、保存草稿等维护动作。
- 展示内部字段给销售角色。
- 图片和资料不区分客户可见性。

## 5. 普通 Grid/List 和历史页合同

普通页面不走自定义 dashboard，不用 imagegen。它们必须执行项目标准：

```text
app-container
  -> el-form inline 搜索
  -> toolbar + right-toolbar
  -> el-table / tree-table
  -> pagination
  -> el-drawer 表单 / 详情 / 引用检查
```

必须覆盖页面：

- 产品分类、物料管理、辅材管理、资料资产、资料绑定。
- 产品模型、销售变体、问题组模板。
- 报价预览、审核审批、缺口待办、发布包、同步日志、导入中心。
- 系统用户、商家用户、菜单、角色、参数、字典、岗位、部门、OSS、公告、操作日志详情、商家审核、商家资料。

统一要求：

- 搜索区默认不超过两行，复杂筛选放更多筛选。
- 抽屉底部操作区 sticky。
- 字典下拉 `filterable`。
- 业务对象选择支持远程搜索或选择器。
- 时间使用 `formatUtc()`。
- 状态使用标签或 `DictTag`，不硬编码文案。
- 长表单用右侧抽屉，短确认才用小弹窗。

## 6. Agent 拆分规则

为避免 AMU 把设计拆碎，本 Wave 执行必须遵守：

- `PCC-UW-F01` 作为 5 个自定义页的设计主线任务，不能再按页面拆给不同 Agent 并行实现。
- `PCC-UW-F02` 只做普通页和历史页标准化，不允许改 5 个自定义页主结构。
- `PCC-UW-F03` 只做流程页操作收口，不允许重做 5 个自定义页视觉。
- `PCC-UW-B01` 不做前端布局，只保证后端 API、权限、数据和发布检查收敛。
- `PCC-UW-V01` 必须按本文逐项验收，不只检查 URL 是否能打开。
- 需要趋势、分布、命中情况等可视化时允许使用项目已有 `echarts`，但必须按需引入、组件封装，且只展示后端结果，不替代后端业务判定。

任何 Agent 如果发现自己的任务需要修改别的任务主结构，必须停止并记录 `blocked`，不能自行扩大范围。

## 7. Done Gate

以下任意一项未满足，`PCC-UW-F01` 和本 Wave 不能标记 `done`：

- 5 个自定义页主结构没有按本文落到 Vue 页面。
- 页面 SFC 没有拆出高密度业务组件。
- 关键快捷操作缺失且没有明确禁用态和 TODO。
- 浏览器验收未覆盖真实菜单、真实登录态、真实开发库。
- console/network 有未解释错误。
- 移动宽度 390px 存在关键按钮遮挡或横向溢出。
- 新增文案没有进入 `i18n/locales/en_US.json`。
- 权限按钮没有对齐 `sys_menu.perms` / `v-hasPermi` / `@SaCheckPermission`。
