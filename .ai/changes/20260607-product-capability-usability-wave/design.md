# Design: 共享产品能力中心上线体验收口 Wave

## 1. 设计原则

- 本 Wave 只收口上线体验，不扩大业务边界。
- UI 精修是 P0，不是 P1/P2。
- 执行必须以 `docs/产品配置中心/产品能力界面设计稿.md` 为 UI 主合同，重点遵守第 2.3 设计技能流转、第 2.4 还原策略、第 5 章 Vue 实现结构、第 5.5 快捷操作和第 6 章减少录入负担。
- 普通页使用标准 grid/list + right-toolbar + drawer。
- 自定义页只在内容区自定义，必须接入现有 Layout、菜单、权限、请求、i18n、UTC。
- H5 静态页和 PNG 效果图只作为设计输入；正式交付必须是 Vue 组件化重构和体验优化，不是复制 H5/PNG。
- 复杂区块组件化，页面 SFC 只做编排。
- 所有真实判断来自后端，前端不复制配置、价格、BOM、发布检查规则。

## 1.1 UI 执行方法

本 Wave 前端执行必须按设计文档的教程走：

1. `visual-replica`：分析现有 `admin-ui` 首页、系统管理标准页和产品能力当前页面，沉淀风格真值，不能重做侧边栏、顶栏、TagsView。
2. `image2-ui-skill`：把 5 个 H5 静态页和 PNG 主参考拆成 Vue 结构、代码渲染区域、资产区域和截图验收清单；文字、表格、按钮、表单必须代码渲染。
3. `frontend-design`：在风格真值和设计文档约束下实现 Vue / Element Plus / SCSS 页面。
4. `design-taste-frontend`：做反模板化审查，检查页面是否退化成普通 CRUD、是否信息层级混乱、是否牺牲录入效率。
5. 内部浏览器：用真实菜单、真实登录态、真实开发库验证视觉、滚动、底部按钮、接口、错误态和移动端。

## 3. 后端和数据收口

- 报价预览、价格试算、发布检查、审核、缺口、发布包、同步日志、导入预览使用真实 API。
- 发布检查结果需要避免历史噪音；执行时优先评估批次号、最新批次过滤或幂等覆盖。
- 如需 SQL，只允许 PostgreSQL 受控脚本，包含 preflight、execute、post-check。
- 权限码必须与 `sys_menu.perms` 和前端 `v-hasPermi` 一致。

## 4. 前端收口

- 复用现有 Element Plus、SvgIcon、项目列表/抽屉风格。
- 5 个自定义页按设计文档拆分可复用 Vue 组件，页面负责数据编排。
- 组件拆分按设计文档第 5.3 / 5.4 执行：工作台拆任务流、优先队列、快捷操作、同步事件；配置页拆问题组导航、问题答案编辑、Live Check、Dealer Preview、BOM Preview；价格页拆矩阵编辑器、试算器、命中明细；发布页拆状态卡、发布包预览、阻断任务、同步状态；销售页拆发布头、只读能力卡、Quick Quote、限制信息和版本历史。
- 字典下拉 `filterable`；业务对象选择使用远程搜索或选择器。
- 空态、错误态、loading、禁用态、无权限态有明确视觉。
- 长页面底部动作保持可见，移动端不遮挡。

## 4.1 页面关键交互不能丢

- 工作台：待处理流卡片可筛选看板，优先队列可直达目标，快捷操作只展示有权限按钮。
- 配置模板：批量粘贴、组件绑定、资料绑定用抽屉；BLOCKER 点击能定位问题或答案；右侧检查/预览/BOM 同屏反馈。
- 价格中心：矩阵支持横向滚动、单元格选中、批量填充；右侧试算调用后端价格引擎并展示命中明细。
- 测试发布：BLOCKER 禁止发布，WARNING 二次确认，发布包 hash、版本、生效时间、包含内容在发布前可见。
- 销售只读：页面只读，Quick Quote 进入报价预览，资料和媒体只展示后端允许销售可见的数据。

## 5. 验收设计

验收分三层：

1. Static Review：权限、SQL、事务、DTO/VO、XSS、重复提交、i18n、UTC。
2. Build/Test：Java 编译、Vue typecheck/build、i18n sync/check、codegraph sync、git diff check。
3. Runtime Validation：内部浏览器真实 `192.*` 地址、真实登录态、真实开发库。

Runtime Validation 必须覆盖：

- 19 个产品能力二级页面刷新 URL 不 404。
- 5 个自定义页主结构、底部、快捷操作、关键 API。
- 报价预览、审核通过/拒绝、发布检查、同步重试、导入预览至少关键路径。
- 移动宽度 390px 抽查，不横向溢出、不遮挡关键按钮。
