# i18n Playbook

仅在 i18n 规划、实现或验证时按需加载。

## 规则

- 遵守 `AGENTS.md` 和 `.ai/CONTEXT.md` 中的 i18n 规则。
- 新增系统文案默认只写 `i18n/locales/en_US.json`，再用 `i18n:translate` 补 `zh_CN.json`，人工校对后提交。
- 前后端运行时 JSON 是 `i18n:sync` 产物，不手动改 `admin-ui/public/i18n/*.json` 或 `bocoo-admin/src/main/resources/i18n/*.json`。
- 系统字典翻译 key 由规则生成：`dict.<dictType>.<value>`；字典表不再新增手写 `i18n_key` 依赖。
- 国家、币种、语言按标准数据建模；客户业务数据原样存储，不生成 i18n key。
- `sys_i18n_message` 已废弃；新功能不要继续写入或依赖它。
- AI 翻译只在开发/构建期运行，运行时不得调用 AI。

## 检查项

- `i18n:validate` 应检查 key 完整性、空值、占位符一致性和字典覆盖。
- 迁移旧文案时先归入分类：系统文案、系统字典、标准数据、业务数据。
- 不确定是否涉及 SQL、字典结构或 `sys_i18n_message` 清理时，先回到 `/plan`。
